# 毛玻璃效果技术文档 (GlassEffect)

## 1. 概述

毛玻璃效果（GlassEffect）用于在弹窗（Dialog）打开时，对底层 UI 进行**模糊截图**作为遮罩背景，形成常见的「背景模糊 + 弹窗前置」的视觉交互效果。

效果通过 `CS.GlassEffect`（C# 侧实现）的 `BlurFrame` 方法触发 UI 管线截图，由 `DialogBlockView` 中的 `glass_effect` 组件负责显示与动画。

## 4. 关键数据字段说明

### 4.1 `DialogParam`（`dialog_mgr.ts:686-697`）

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `isBgVisible` | `boolean` | `true` | 控制弹窗是否需要毛玻璃背景遮罩 |
| `animationType` | `number` | `0` | -1=无动画, 0=仅DoFade, 1=DoFade+DoPanelBounce, 2=DoFade+DoPanelScale |
| `tag` | `string` | `""` | 标签，用于批量关闭 |
| `musicType` | `number` | `1` | 打开音效类型 |
| `parent` | `Transform` | `layoutDialogTrans` | 弹窗父节点 |

> **注意**：`HeroTalentUnlockPopupUI.Create()` 内部创建的是 `new DialogParam()`，未设置 `isBgVisible`，因此取默认值 `true`，即**启用毛玻璃**。若不需要毛玻璃，需显式设置 `dialogParam.isBgVisible = false`。

### 4.2 `DialogItem`（`dialog_mgr.ts:11-220`）

| 字段 | 类型 | 说明 |
|------|------|------|
| `isBgVisible` | `boolean` | 当前弹窗是否需要毛玻璃 |
| `hasAddedGlass` | `boolean` | **是否已为此弹窗执行了截图**（用于 Pop 时减计数） |
| `animationType` | `number` | 弹窗动画类型 |

### 4.3 `CS.GlassEffect` 全局属性

| 属性 | 说明 |
|------|------|
| `GlassEffect.g_glassCount` | **全局毛玻璃引用计数**，最大值限制为 2。每次 Push 截图时 +1，Close/Pop 时 -1 |
| `GlassEffect.s_update` | 是否允许毛玻璃持续更新。截图时设为 `false`，关闭后设为 `true` |
| `GlassEffect.Pipeline.UI` | UI 截图管线枚举值 |
| `GlassEffect.BlurFrame(pipeline, force?)` | 执行单次模糊截图。第二个参数 `force` 用于切屏时强制重建 |

## 5. 核心流程详解

### 5.1 Push（弹窗入栈）— `dialog_mgr.ts:360-448`

```typescript
// Step 1: 停止切屏效果（避免截到过渡动画）
TriggerMgr.Trigger(E_TriggerType.need_stop_latter_switch_map_effect)

// Step 2: 判断是否需要截图
let add_glass = false
if (dialogParam.isBgVisible && CS.GlassEffect.g_glassCount < 2) {
    CS.GlassEffect.s_update = false          // 暂停毛玻璃实时更新
    CS.GlassEffect.BlurFrame(CS.GlassEffect.Pipeline.UI) // 执行截图
    add_glass = true
}

// Step 3: 将标记存入 DialogItem
item.hasAddedGlass = add_glass
```

**截图条件**（同时满足）：
1. `dialogParam.isBgVisible === true`（弹窗需要毛玻璃背景）
2. `GlassEffect.g_glassCount < 2`（全局引用计数未达上限）

### 5.2 Pop（弹窗出栈）— `dialog_mgr.ts:121-165`

```typescript
// Close() 中：
if (this.hasAddedGlass) {
    CS.GlassEffect.g_glassCount--  // 引用计数减1
    if (CS.GlassEffect.g_glassCount < 0) {
        CS.GlassEffect.g_glassCount = 0
    }
}
```

同时 `CloseAll()`（dialog_mgr.ts:539-555）会直接重置：
```typescript
CS.GlassEffect.g_glassCount = 0
```

### 5.3 ResetStatus（毛玻璃组件状态刷新）— `dialog_block_view.ts:80-171`

每次弹窗数量/层级变化时调用，逻辑如下：

```
是否还有弹窗需要毛玻璃背景 (IsBgVisible)?
  ├─ YES: 启用 glass_effect 组件
  │   ├─ 如果之前是 disabled → enabled（首次启用）
  │   └─ 如果已经是 enabled → ActiveEffect()（刷新模糊效果）
  │
  └─ NO: 禁用 glass_effect 组件
      └─ GlassEffect.s_update = true（恢复毛玻璃实时更新）
```

### 5.4 淡入淡出动画 — `dialog_block_view.ts:181-207`

```
DoBgTween(arg):
  arg = 1:  淡入, glass_effect.DOFade(0 → 1, 0.2s)
  arg = -1: 淡出, glass_effect.DOFade(1 → 0, 0.2s)，完成后隐藏 bgImage
  arg = 0:  无动画
```

触发时机：
- `arg = 1`：`isBgVisibleInMgr = true` 且 `bgImage` 当前非激活
- `arg = -1`：`isBgVisibleInMgr = false` 且 `bgImage` 当前激活
- `arg = 0`：状态无变化

### 5.5 切屏重建 — `dialog_mgr.ts:259-268`

```typescript
TriggerMgr.AddTrigger(E_TriggerType.on_screen_size_changed, () => {
    if (this.currentItem && this.currentItem.isBgVisible && CS.GlassEffect.g_glassCount > 0) {
        // 先隐藏弹窗和 Block
        this.currentItem.transform.gameObject.SetActive(false)
        DialogBlockView.Ins?.gameObject?.SetActive(false)
        
        // 强制重建截图
        CS.GlassEffect.s_update = false
        CS.GlassEffect.BlurFrame(CS.GlassEffect.Pipeline.UI, true)
        
        // 恢复显示
        DialogBlockView.Ins?.gameObject?.SetActive(true)
        this.currentItem.transform.gameObject.SetActive(true)
    }
})
```

切屏（分辨率变化）时，旧截图失效，需要重新截图。

## 7. 使用流程（给业务开发者的接入指南）

### 7.1 添加毛玻璃效果（标准做法）

需要毛玻璃效果的 UI，必须**通过弹窗管理系统创建**，而不是直接 `UtilAsset.AutoCreateInstanceAsync`。核心改动只有 **Create** 和 **Close** 两处。

**改造前（无毛玻璃）：**

```typescript
static async Create(): Promise<SomeUI> {
    if (!Context.IsValid(SomeUI.static_ctx)) {
        let ctx = Context.Create(Context.RootScene)
        SomeUI.static_ctx = ctx
        let obj = await UtilAsset.AutoCreateInstanceAsync(   // ← 直接实例化
            ctx,
            "ui_fs/ui_prefabs/some.prefab",
            GameInitSystemIns.layoutUITrans)
        if (ctx.IsCancel()) return null
        SomeUI.Ins.ctx = ctx
    }
    return SomeUI.Ins
}

static Close() {
    SomeUI.static_ctx?.Cancel()
    XT.DestroyImmediate(SomeUI.Ins.gameObject)               // ← 直接销毁
}
```

**改造后（有毛玻璃）：**

```typescript
import { DialogParam } from "../../../xkit/ui/dialog_mgr"
import { UIManager } from "../../../xkit/ui/UIManager"

export class SomeUI implements IBehaviour {
    dialogId: number                                           // ← 新增 dialogId 字段

    static async Create(): Promise<SomeUI> {
        if (!Context.IsValid(SomeUI.static_ctx)) {
            let ctx = Context.Create(Context.RootScene)
            SomeUI.static_ctx = ctx
            let dialogParam = new DialogParam()                // ← 创建 DialogParam
            dialogParam.parent = GameInitSystemIns.layoutUITrans
            let id = await UIManager.Ins.CreateDialogUIAsync(  // ← 走弹窗管理系统
                ctx,
                "ui_fs/ui_prefabs/some.prefab",
                dialogParam,
                (obj) => {
                    SomeUI.Ins.ctx = ctx
                }
            )
            if (ctx.IsCancel()) return null
            SomeUI.Ins.dialogId = id
        }
        return SomeUI.Ins
    }

    static Close() {
        SomeUI.static_ctx?.Cancel()
        if (SomeUI.Ins) {
            UIManager.Ins.CloseByIdAsync(SomeUI.Ins.dialogId)  // ← 通知 DialogMgr 关闭
        }
    }
}
```

**关键点：**
1. 新增 `dialogId: number` 字段，用于关闭时通知 `DialogMgr` 释放毛玻璃引用计数
2. `Create()` 中：`UtilAsset.AutoCreateInstanceAsync` → `UIManager.Ins.CreateDialogUIAsync`
3. `Close()` 中：`XT.DestroyImmediate` → `UIManager.Ins.CloseByIdAsync(dialogId)`
4. `DialogParam` 使用 `new DialogParam()`，默认 `isBgVisible = true`，自动启用毛玻璃

### 7.2 关闭毛玻璃效果

如果业务 UI 不需要毛玻璃，在创建 `DialogParam` 后显式设置：

```typescript
let dialogParam = new DialogParam()
dialogParam.isBgVisible = false   // ← 关闭毛玻璃
```

4. 关闭时 `CloseByIdAsync` 触发 Pop 逻辑，`g_glassCount--`，毛玻璃淡出

## 8. 注意事项

### 8.1 引用计数上限
- `GlassEffect.g_glassCount` 最大值为 **2**
- 超过 2 个弹窗同时需要毛玻璃时，后续弹窗不再触发截图（复用已有的模糊截图）
- 每个弹窗 Pop 时通过 `hasAddedGlass` 标记来正确减计数

### 8.2 与切屏效果的互斥
- 毛玻璃截图会截到切屏（switch_map）的过渡动画
- 因此 Push 前先通过 `TriggerMgr.Trigger(E_TriggerType.need_stop_latter_switch_map_effect)` 停止切屏效果

### 8.3 切屏时重建
- 注册了 `on_screen_size_changed` 事件监听
- 分辨率变化后旧截图无效，需要**隐藏弹窗 → 重新截图 → 恢复弹窗**

### 8.4 两次 `s_update = false`
在同一个 Push 流程中，`dialog_mgr.ts` 的 Push/PushAsync 中设置了一次 `s_update = false`，`dialog_block_view.ts` 的 `ResetStatus` 中又存在一次设置。实际第一次截图时已设为 `false`，第二次在 `ResetStatus` 中通过 `glass_effect.ActiveEffect()` 刷新但不再重复截图。

### 8.5 改造要求：必须走 Dialog 管理系统
毛玻璃效果依赖 `DialogMgr` 的 Push/Pop 生命周期管理。如果业务 UI 直接使用 `UtilAsset.AutoCreateInstanceAsync` + `XT.DestroyImmediate` 创建/销毁，则完全绕过了毛玻璃系统。**任何需要毛玻璃的 UI 都必须改造成通过 `UIManager.CreateDialogUIAsync` + `CloseByIdAsync` 创建和销毁**。
