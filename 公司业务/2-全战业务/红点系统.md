# 红点系统

## 概述

红点系统基于**树形数据结构**，以非叶子节点作为监听单元，叶子节点存储红点值，通过**脏标记-延迟触发**机制驱动 UI 更新。

## 核心机制

### 1. 节点路径

路径用 `.` 分隔层级，例如 `"Main.Growth.DrawCard"`。

- **非叶子节点**（如 `"Main.Growth.DrawCard"`）：可聚合子节点值，可添加监听
- **叶子节点**（如 `"Main.Growth.DrawCard.BigPool"`）：`isLeafNode=true`，`valueSelf` 必须为 1

所有非叶子节点在 `RedPointMgr.Init()` 时预创建到树中（值为 0）。

### 2. 值聚合

节点的 `value` 由 `valueSelf + valueInSubNode` 组成：

```
Main.Growth.DrawCard         value = 2       ← 两个子节点各贡献 1
├── BigPool                   valueSelf = 1  ← 常驻池可抽
└── NewPool                   valueSelf = 1  ← 新池可抽
    └── 1001                  valueSelf = 1  ← 具体新池（可选层级）
```

当新增叶子节点（`AddNode(true, path, 1)`），从该节点向上遍历所有祖先，每个祖的 `valueInSubNode += 1`。

### 3. 脏标记延迟触发

```
值变化 → 所有祖先加入 dirtySet
                    ↓
RedPointMgr.Update()（每0.1s LateUpdate）
                    ↓
遍历 dirtySet → 对每个节点调用 TriggerListener()
                    ↓
遍历节点的 listenerList → 逐一执行回调
```

### 4. 监听注册

```typescript
// 注册监听 + 立即用当前值调用一次回调（用于 UI 初始刷新）
Global.RedMgr.AddListenerWithInvoke(
    fullPath,          // 非叶子节点路径，如 "Main.Growth.DrawCard"
    (node) => {        // 回调
        RedPointItemController.RefreshRedPointItem(this.RedDrawCard, node.value)
    },
    null               // target
)
```

**规则**：
- 只允许对**非叶子节点**添加监听
- 监听回调会在节点值变化时被触发
- 注册时立即调用一次，获取当前值完成 UI 初始化

## RedDrawCard 完整数据流示例

```
业务触发：CardDrawRedController.CreateRedNodeForBigPool()
    ↓
Global.RedMgr.AddNode(true, "Main.Growth.DrawCard.BigPool", 1)
    ↓
RedPointTree.AddNode()
    ├─ 创建叶子节点 BigPool（valueSelf=1）
    └─ 向上遍历：DrawCard、Growth、Main
        ├─ 每个祖先的 valueInSubNode += 1
        └─ 每个祖先加入 dirtySet
    ↓
RedPointMgr.Update()（下一帧，LateUpdate 0.1s周期）
    ├─ 遍历 dirtySet
    └─ DrawCard.TriggerListener() → 执行回调
    ↓
StartRed() 注册的回调：
    if (this.RedDrawCard) {
        RedPointItemController.RefreshRedPointItem(this.RedDrawCard, node.value)
    }
    ↓
RedPoint.Refresh(active=true, text=0)
    ├─ gameObject.SetActive(true)  → 红点显示
    └─ text.text = ""              → 不显示数字
```

## RedDrawCard1 实现模式

`RedDrawCard1` 与 `RedDrawCard` 是预制体上两个独立的红点 RectTransform，监听相同的 `"Main.Growth.DrawCard"` 节点。实现方式是在 `StartRed()` 中复制 `RedDrawCard` 的监听逻辑：

```typescript
redPointListenerList.push(Global.RedMgr.AddListenerWithInvoke(RedPointNonLeafPathConst.DrawCard, (node: RedPointTreeNode) => {
    if (this.RedDrawCard1) {
        RedPointItemController.RefreshRedPointItem(this.RedDrawCard1, node.value)
    }
}, null))
