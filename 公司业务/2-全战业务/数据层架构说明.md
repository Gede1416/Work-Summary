# 全战客户端 — 数据层架构说明

> 编写时间：2026-05-07  
> 核心文件：`src/generator/excel/init.js`（配置表入口）、`src/system/role/Global.ts`（数据管理中枢）

---

## 零、前置知识

| 项目 | 说明 |
|------|------|
| **技术栈** | TypeScript + Unity (JSBehaviour) + protobuf + MobX |
| **代码路径约定** | `src/generator/excel/` → 配置表 | `src/system/role/user_data/` → 用户数据类 | `src/system/***/` → 各业务模块（含管理器、UI） |
| **配置表入口** | `src/generator/excel/init.js`，导出全局对象 `cfg` |
| **数据中枢** | `src/system/role/Global.ts`，静态类，通过 `Global.xxx` 全局访问 |
| **角色基类** | `src/system/role/Role.ts`，持有 `userData`（用户数据）、`mgrMod`（管理器模块）|

---

## 一、配置表体系（init.js）

### 1.1 作用

- **自动生成的代码**（首行注：`Code generated by ExcelTool. DO NOT EDIT.`）
- 将 200+ 张 Excel 配置表文件统一 `require()` 并导出为 `cfg` 对象
- 提供初始化方法 `cfg.INIT()` 和异步版 `cfg.INIT_ASYNC()`
- 支持多语言：通过 `cfg.text` 和 `cfg.text_i18n` 实现国际化

### 1.2 每张配置表的通用接口

| 方法 | 说明 |
|------|------|
| `cfg.xxx.get(id)` | 按 ID 查单条记录 |
| `cfg.xxx.getData()` | 取全表数组 |
| `cfg.xxx.has(id)` | 判断是否存在 |
| `cfg.xxx.size` | 总条目数 |
| `cfg.xxx.for_each(fn)` | 遍历所有条目 |

> - `cfg.property.get(5535).Value` — 新手保护期后的技能返还率

> - `cfg.property.get(1310).Value` — 部队移动速度

## 二、全局数据中枢（Global.ts）

### 2.1 作用

- 一个**静态类**（所有成员都是 `static`），全局唯一入口
- 通过 `Role.Ins` 获取当前玩家的角色实例
- 分为三大块：**基础信息**、**用户数据**（UserData）、**管理器模块**（Mgr）

### 2.2 初始化与销毁

```
Role.Ins 创建
  ├── userData 创建（new UserData()）
  │     ├── 各类用户数据对象（UserHero / UserTeam / UserItem ...）
  │     └── 网络消息注册
  ├── mgrMod 创建（new MgrModule()）
  │     ├── 各类管理器（HeroMgr / BattleMgr / MapMgr ...）
  │     └── 初始化、注册消息处理器
  ├── shareData 创建（新玩家信息缓存）
  └── 登录流程 → OnRoleBeforeLogin → OnRoleAfterLogin

登出时 → 全部销毁，下次登录重新创建
```

> **规律总结**：每个「用户数据类」（UserXxx）往往对应一个「管理器」（XxxMgr）。数据类负责存储和本地计算，管理器负责网络通信、消息处理、业务逻辑编排。
