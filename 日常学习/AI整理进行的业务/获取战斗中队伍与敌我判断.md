## 二、获取"战斗中"的队伍

### 2.1 快速入口

| 方法/属性 | 所属类 | 返回值 | 说明 |
|-----------|--------|--------|------|
| `IsPlayerHasTeamInBattle` | `UserTeam` | `boolean` | 是否有任意队伍在战斗中（非世界地图），UI 层快速判断 |

### 2.2 UserTeam 层面方法链

```typescript
// ── A: 获取所有存活队伍 ──
const aliveTeams = Global.UserTeam.GetAliveTeams();

// ── B: 获取当前地图的所有队伍 ──
const currentMapTeams = Global.UserTeam.GetInCurrentMapTeams();

// ── C: 按创建类型过滤队伍 ──
const normalTeams = Global.UserTeam.GetTeamsByCreateType(CreateType.Normal);

// ── D: 获取选中队伍 ──
const selectedTeams = Global.UserTeam.GetSelectedTeams();
```

### 2.3 UserTeamItem 层面判断（核心）

| 属性 | 返回 | 说明 |
|------|------|------|
| `IsBattling` | `boolean` | **是否在战斗中**（攻城器械时额外检查器械状态） |
| `DispatchState` | `UserTeamDispatchState` | 派遣状态，值为 `Battle` 即表示战斗 |
| `IsAlive` | `boolean` | 是否存活（`soldier_num > 0`） |
| `IsInCurrentMap` | `boolean` | 是否在当前地图 |
| `AutoBattle` | `boolean` | 是否自动战斗（`auto_battle_type != -1`） |

```typescript
// 获取所有正在战斗的队伍
const battlingTeams = Global.UserTeam.teamList.filter(team => team.IsBattling);

// 获取存活且在战斗的队伍
const aliveBattling = Global.UserTeam.GetAliveTeams().filter(team => team.IsBattling);

// 按派遣状态过滤
const battleStateTeams = Global.UserTeam.teamList.filter(
  team => team.DispatchState === UserTeamDispatchState.Battle
);

// 获取当前地图中正在战斗的队伍
const currentBattle = Global.UserTeam.GetInCurrentMapTeams().filter(
  team => team.IsBattling
);
```

### 2.4 UserTeamGroupItem 层面（队伍组状态机）

`GetDispatchState()` 状态机中返回 `Battle` 的条件：

```
在战场地图（!IsInWorldMap）
  └─ 有队伍正在战斗（遍历队伍检查 IsBattling）
      └─ 返回 Battle
  └─ 否则
      └─ 返回 CombatReadiness（战备状态）
```

```typescript
// 获取所有战斗中的队伍组
const battlingGroups = Global.UserTeam.teamGroupList.filter(
  group => group.GetDispatchState() === UserTeamDispatchState.Battle
);

// 或更细粒度：检查队伍组内有哪些队伍在战斗
const group = someTeamGroup;
// group.IsOutCity          → 是否出城
// group.LocationMapUUID    → 所在位置地图UUID
// group.TotalCurrentSoldierNum → 所有队伍当前兵力总和
// group.TotalMaxSoldierNum     → 所有队伍最大兵力总和
```

## 四、判断敌我

`UserTeam` 类只管理**玩家自己的队伍**，不提供主动的敌我判断方法。敌我判断需要依赖 `userUUID` 及外部模块。

### 4.1 核心判断依据

| 数据 | 来源 | 说明 |
|------|------|------|
| `team.teamData.user_uuid` | `UserTeamItem.teamData` | **队伍归属的玩家 UUID**（最原始） |
| `team.userUUID` | `UserTeamItem` 计算属性 | 同上，便捷访问 |
| `Global.UserHero` 或其他全局模块 | 全局单例 | 获取**当前玩家自身的 UUID** |

### 4.2 基础判断逻辑

```typescript
// 当前玩家 UUID（来源需根据实际项目确定）
const selfUUID = Global.UserHero.heroData?.user_uuid;

// ── 判断是否己方队伍 ──
function isOwnTeam(team: UserTeamItem): boolean {
  return team.userUUID === selfUUID;
  // 或 team.teamData.user_uuid === selfUUID
}

// ── 判断是否敌方队伍 ──
function isEnemyTeam(team: UserTeamItem): boolean {
  // 情况1：不是自己
  if (team.userUUID === selfUUID) return false;

  // 情况2：排除同盟（需 UserGuild）
  // const isAlly = Global.UserGuild?.isAllied?.(team.userUUID);
  // if (isAlly) return false;

  return true; // 非己方、非同盟 → 敌人
}
```

### 4.3 通过地图单位判断（更准确）

在大世界战斗中，队伍的阵营标记通常存在于**地图单位数据**中，而非队伍数据本身。

| 模块/API | 用途 |

| `MapAPI` | 获取地图单位，检查单位阵营/关系 |
| `MapTeamData` | 队伍的地图数据（含坐标、阵营等） |
| `UserMap` | 地图层面的数据 |
| `UserGuild` | 同盟/联盟关系查询 |

```typescript
// 伪代码：通过地图 API 获取目标阵营
function getTeamCamp(teamUUID: Long): MapUnitCamp {
  // 从地图系统获取单位数据

  // return mapUnit?.camp;

// 伪代码：判断两个队伍是否敌对
function isHostile(teamA: UserTeamItem, teamB: UserTeamItem): boolean {
  // 同玩家 → 不敌对
  if (teamA.userUUID === teamB.userUUID) return false;

  // 同盟 → 不敌对
  // if (isAllied(teamA.userUUID, teamB.userUUID)) return false;

  // 阵营标记不同 → 敌对
```

## 七、总结

| 目标 | 关键方法 | 类 |
|------|----------|-----|
| 是否存在战斗中队伍 | `IsPlayerHasTeamInBattle` | `UserTeam` |
| 某队伍是否在战斗 | `IsBattling` / `DispatchState === Battle` | `UserTeamItem` |
| 队伍组是否在战斗 | `GetDispatchState() === Battle` | `UserTeamGroupItem` |
| 判断己方队伍 | 比较 `userUUID` | `UserTeamItem` |
| 判断敌方队伍 | `userUUID != self` + 排除同盟（`UserGuild`） | 跨模块 |
| 地图阵营判断 | `MapAPI` / `MapTeamData` | 地图系统 |
