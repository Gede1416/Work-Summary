# C++ 包管理与构建工具

## 详细描述

C++ 的包管理和构建系统历来是最碎片化的领域，但近年已有显著改善。理解现有的构建生态是开始任何 C++ 项目的前提。

**CMake** 是 C++ 世界事实上的构建系统标准。它生成平台原生的构建文件（Unix Makefile、Ninja、Visual Studio 解决方案、Xcode 项目），而不是直接编译代码。现代 CMake（3.12+）推荐声明式风格：`target_sources`/`target_include_directories`/`target_link_libraries`，替代全局的 `include_directories` 和 `link_libraries`。

**vcpkg**（微软）和 **Conan**（社区）是两大主流包管理器：
- **vcpkg**：源码构建，所有包从源码编译，与 CMake 和 MSBuild 深度集成。`vcpkg.json` 声明依赖，`vcpkg install` 自动安装。支持 triplet（x64-windows、x64-linux、arm64-osx 等）控制目标平台。
- **Conan**：支持预编译包和源码构建，有更灵活的版本管理。通过 `conanfile.py` 或 `conanfile.txt` 配置依赖。适合复杂的企业级项目。

**C++20 Module** 将彻底改变构建方式——模块预编译后直接导入，不再需要头文件展开。但目前在 CMake 中的支持仍在实验阶段，各编译器（MSVC、Clang、GCC）的实现程度不同。

**Ninja** 是一个专注于速度的构建文件执行器，常与 CMake 搭配（`cmake -G Ninja`）。它比 Make 更快，特别适合增量构建。

**其他工具**：`ccache` 缓存编译结果加速重编译；`CPM.cmake`（CMake 的轻量级包管理器，基于 FetchContent）；`Bazel`（Google 的构建工具，支持大型 monorepo）；`xmake`（国产新兴构建工具，Lua 语法配置）。

## 代码实例

```cmake
# ========== 1. 现代 CMake 项目示例 ==========
# CMakeLists.txt

cmake_minimum_required(VERSION 3.20)
project(MyCppProject VERSION 1.0.0 LANGUAGES CXX)

# 设置 C++ 标准
set(CMAKE_CXX_STANDARD 20)
set(CMAKE_CXX_STANDARD_REQUIRED ON)
set(CMAKE_CXX_EXTENSIONS OFF)

# 查找外部依赖
find_package(fmt CONFIG REQUIRED)     # {fmt} 格式化库
find_package(spdlog CONFIG REQUIRED)  # spdlog 日志库

# 库目标
add_library(my_lib STATIC
    src/lib/calculator.cpp
    src/lib/parser.cpp
)

# 声明式：附加于目标的属性
target_include_directories(my_lib PUBLIC
    $<BUILD_INTERFACE:${CMAKE_CURRENT_SOURCE_DIR}/include>
    $<INSTALL_INTERFACE:include>
)

target_link_libraries(my_lib PUBLIC
    fmt::fmt
    spdlog::spdlog
)

target_compile_features(my_lib PUBLIC cxx_std_20)

# 可执行目标
add_executable(my_app src/main.cpp)
target_link_libraries(my_app PRIVATE my_lib)

# ========== 2. vcpkg 清单文件 ==========
# vcpkg.json
```
{
    "name": "my-cpp-project",
    "version": "1.0.0",
    "dependencies": [
        "fmt",
        "spdlog",
        "nlohmann-json",
        "boost-asio"
    ]
}
```

# 在 CMakeLists.txt 中使用 vcpkg 的包
# find_package(nlohmann_json CONFIG REQUIRED)
# find_package(Boost REQUIRED COMPONENTS system)
```

## 推荐书籍

1. **《Professional CMake: A Practical Guide》（第14版）** — Craig Scott
   CMake 的权威指南，从基础到高级（generator expressions、install、testing、CPack 打包）。

2. **《Modern CMake for C++》** — Rafal Swidzinski
   以现代 CMake 的最佳实践为主线，讲解 target-based 构建方法。

3. **vcpkg 官方文档**（在线，免费，docs.microsoft.com/vcpkg）
   覆盖 triplet 配置、自定义 triplet、overlay ports、binary caching 等。
