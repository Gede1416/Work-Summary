# C++ 内存管理

## 详细描述

C++ 的内存管理是它区别于大多数高级语言的核心特征——你不必为所有对象支付垃圾回收的开销，但你必须精确控制每个对象的生命周期。这不是限制，而是 C++ 实现极致性能的关键。

**栈与堆分配**：局部变量自动分配在栈上，作用域结束自动销毁，速度快但空间有限。`new` 在堆上分配内存，返回指针，需要手动 `delete` 释放。现代 C++ 强烈建议避免裸 `new/delete`，改用智能指针和 RAII 容器。

**RAII**（Resource Acquisition Is Initialization）是 C++ 最重要的设计习语。资源（内存、文件句柄、锁、socket）在构造函数中获取，在析构函数中自动释放。RAII 保证了异常安全——无论函数如何退出（正常返回或异常），栈展开都会调用析构函数。这是 C++ 不需要 `finally` 块的原因。

**智能指针**（C++11）是 RAII 在内存管理上的标准化体现：
- `unique_ptr<T>`：独占所有权，不可拷贝，可移动。零开销（与裸指针相同大小）。首选智能指针。
- `shared_ptr<T>`：共享所有权，引用计数。需要额外的控制块分配。用于共享生命周期场景。
- `weak_ptr<T>`：不参与引用计数，可从 shared_ptr 构造。用于打破循环引用和缓存模式。
- `make_unique`/`make_shared`：推荐的创建方式，异常安全且更高效（`make_shared` 一次分配对象和控制块）。

**移动语义**与内存管理紧密相关——通过"窃取"资源而非深拷贝，避免不必要的内存分配和释放。移动后源对象必须处于"合法但未指定"状态，通常将指针置为 `nullptr`。

**allocator** 是 STL 容器的模板参数，允许自定义内存分配策略。默认的 `std::allocator` 使用 `::operator new`，你可以替换为内存池分配器（`std::pmr::memory_resource`，C++17）或自定义策略。

**对象生命周期**的理解是避免悬垂指针（dangling pointer）和 use-after-free 的关键。C++ 保证：构造顺序（基类→成员→构造函数体），析构顺序（析构函数体→成员→基类），静态对象在 `main` 前后构造/析构（注意静态初始化顺序问题）。

## 代码实例

```cpp
#include <iostream>
#include <memory>
#include <vector>
#include <cassert>

// ========== 1. RAII 模式 ==========
class FileHandle {
    FILE* file_;
public:
    explicit FileHandle(const char* path, const char* mode)
        : file_(fopen(path, mode)) {
        if (!file_) throw std::runtime_error("Cannot open file");
        std::cout << "File opened\n";
    }

    ~FileHandle() {
        if (file_) {
            fclose(file_);
            std::cout << "File closed\n";
        }
    }

    // 禁止拷贝，允许移动
    FileHandle(const FileHandle&) = delete;
    FileHandle& operator=(const FileHandle&) = delete;

    FileHandle(FileHandle&& other) noexcept : file_(other.file_) {
        other.file_ = nullptr;
    }

    FileHandle& operator=(FileHandle&& other) noexcept {
        if (this != &other) {
            if (file_) fclose(file_);
            file_ = other.file_;
            other.file_ = nullptr;
        }
        return *this;
    }

    FILE* get() const { return file_; }
    void write(const std::string& data) {
        fputs(data.c_str(), file_);
    }
};

// ========== 2. 智能指针 ==========
struct Widget {
    int id;
    explicit Widget(int i) : id(i) { 
        std::cout << "Widget " << id << " created\n"; 
    }
    ~Widget() { 
        std::cout << "Widget " << id << " destroyed\n"; 
    }
    void doWork() { std::cout << "Widget " << id << " working\n"; }
};

void smart_pointer_examples() {
    // unique_ptr：独占所有权
    {
        auto w1 = std::make_unique<Widget>(1);
        w1->doWork();

        // auto w2 = w1;  // 错误！不能拷贝
        auto w2 = std::move(w1);  // OK：转移所有权
        assert(w1 == nullptr);    // w1 现在为空
        w2->doWork();
    }  // w2 离开作用域，Widget 1 自动销毁

    // shared_ptr：共享所有权
    {
        auto s1 = std::make_shared<Widget>(2);
        std::cout << "Use count: " << s1.use_count() << "\n";  // 1
        {
            auto s2 = s1;  // 共享所有权
            std::cout << "Use count: " << s1.use_count() << "\n";  // 2
            auto s3 = s1;  // 3
        }
        std::cout << "Use count: " << s1.use_count() << "\n";  // 1
    }  // 最后一个 shared_ptr 销毁，Widget 2 销毁

    // weak_ptr：打破循环引用
    struct Node {
        std::string name;
        std::shared_ptr<Node> next;      // 强引用
        std::weak_ptr<Node> prev;        // 弱引用，避免循环
        ~Node() { std::cout << "Node " << name << " destroyed\n"; }
    };

    auto n1 = std::make_shared<Node>();
    n1->name = "First";
    auto n2 = std::make_shared<Node>();
    n2->name = "Second";
    
    n1->next = n2;
    n2->prev = n1;  // weak_ptr 不增加引用计数
    
    // n1 和 n2 都能正常销毁（没有循环引用）
}

// ========== 3. 自定义内存池 ==========
template <typename T, size_t BlockSize = 256>
class PoolAllocator {
    struct Block {
        alignas(T) char data[sizeof(T) * BlockSize];
        bool used[BlockSize] = {};
    };
    std::vector<Block> blocks_;

public:
    using value_type = T;

    T* allocate(size_t n) {
        assert(n == 1);  // 简化：一次只分配一个
        
        for (auto& block : blocks_) {
            for (size_t i = 0; i < BlockSize; ++i) {
                if (!block.used[i]) {
                    block.used[i] = true;
                    return reinterpret_cast<T*>(&block.data[i * sizeof(T)]);
                }
            }
        }
        
        // 所有块满了，分配新块
        blocks_.emplace_back();
        auto& newBlock = blocks_.back();
        newBlock.used[0] = true;
        return reinterpret_cast<T*>(newBlock.data);
    }

    void deallocate(T* ptr, size_t n) {
        for (auto& block : blocks_) {
            auto begin = reinterpret_cast<T*>(block.data);
            auto end = begin + BlockSize;
            if (ptr >= begin && ptr < end) {
                auto index = ptr - begin;
                block.used[index] = false;
                return;
            }
        }
    }
};

// ========== 4. 移动语义与资源转移 ==========
class DynamicArray {
    size_t size_;
    int* data_;
public:
    explicit DynamicArray(size_t size)
        : size_(size), data_(new int[size]) {
        std::fill(data_, data_ + size_, 0);
        std::cout << "Array of size " << size_ << " allocated\n";
    }

    ~DynamicArray() {
        delete[] data_;
        std::cout << "Array of size " << size_ << " freed\n";
    }

    // 移动构造函数
    DynamicArray(DynamicArray&& other) noexcept
        : size_(other.size_), data_(other.data_) {
        other.size_ = 0;
        other.data_ = nullptr;
    }

    // 拷贝构造函数
    DynamicArray(const DynamicArray& other)
        : size_(other.size_), data_(new int[other.size_]) {
        std::copy(other.data_, other.data_ + size_, data_);
        std::cout << "Array of size " << size_ << " copied\n";
    }
};

int main() {
    // RAII 文件操作
    try {
        FileHandle file("test.txt", "w");
        file.write("Hello, RAII!\n");
        // 即使这里抛异常，文件也会被正确关闭
    } catch (const std::exception& e) {
        std::cerr << "Error: " << e.what() << "\n";
    }

    smart_pointer_examples();

    // 移动语义
    DynamicArray a1(1000);           // 分配
    DynamicArray a2 = std::move(a1);  // 移动，无新分配
    
    return 0;
}
```

## 推荐书籍

1. **《Effective Modern C++》** — Scott Meyers
   条款 18-22 详细讲解了智能指针的所有权语义和使用场景，包括 `make_unique`/`make_shared`、自定义删除器、weak_ptr 用法。

2. **《C++ Concurrency in Action》（第2版）** — Anthony Williams
   虽然主要讲并发，但其中对 `shared_ptr` 和 `weak_ptr` 在多线程场景下原子引用计数的分析非常深入。

3. **《The C++ Programming Language》（第4版）** — Bjarne Stroustrup
   第 13 章（内存管理）和第 34 章（allocator）提供了 RAII 的理论基础和标准库内存管理设施。

4. **《Optimized C++》** — Kurt Guntheroth
   从性能优化角度讲解内存管理：缓存友好、内存池、避免碎片化，以及如何用测量工具验证分配开销。
