# C++ 异步与并发编程

## 详细描述

C++ 从 C++11 开始拥有了标准化的线程和并发支持，彻底告别了依赖平台 API 的时代。现代 C++ 提供了从底层原子操作到高层协程的完整并发工具箱。

**std::thread** 是最基础的并发单元——代表一个操作系统线程。构造时传入可调用对象和参数，线程立即开始执行。必须在 `thread` 对象析构前调用 `join()`（等待完成）或 `detach()`（分离线程），否则程序调用 `std::terminate`。C++20 的 `std::jthread` 在析构时自动 join，并支持协作式取消（stop_token）。

**互斥锁与同步**：`std::mutex` 提供基本的互斥访问，配合 `std::lock_guard`（RAII，作用域内自动加锁/解锁）和 `std::unique_lock`（更灵活，可延迟加锁、移动、配合条件变量）。`std::shared_mutex`（C++17）支持读写锁——多个读者同时访问，写者独占。`std::scoped_lock`（C++17）可以同时锁定多个 mutex 而无需担心死锁（使用内置的死锁避免算法）。

**std::atomic** 提供无锁的原子操作。对于简单的共享变量（计数器、标志位），原子操作比互斥锁更高效。`memory_order` 允许控制内存序：`relaxed`（仅保证原子性）、`acquire/release`（建立 happens-before 关系）、`seq_cst`（全局一致性，默认）。

**条件变量**（`std::condition_variable`）实现了线程间的通知机制。生产者-消费者模型是其经典用例：消费者在条件变量上 wait（释放锁并阻塞），生产者完成工作后 notify，消费者被唤醒并重新获取锁继续执行。

**future/promise 与 async**：`std::async` 启动一个异步任务并返回 `std::future`，通过 `future.get()` 获取结果（阻塞等待）。`std::promise` 在生产者端设置结果，关联的 `std::future` 在消费者端获取。这是异步任务的一次性通信机制。

**C++20 协程**是 C++ 异步编程的革命性变化。协程函数（包含 `co_await`、`co_yield`、`co_return` 之一）可以在执行中暂停并恢复，而不会阻塞线程。`co_await` 暂停等待，`co_yield` 产生值，`co_return` 返回最终结果。协程的核心优势在于可以写出看起来同步的代码，却以异步方式执行——一个网络请求等待期间，线程可以执行其他协程。注意：C++20 提供的是低级协程基础设施，业务代码一般基于第三方库（如 cppcoro）或自己实现的协程类型。

// ========== 1. 基础线程 ==========
void hello_thread(int id) {
    std::cout << "Thread " << id << ": Hello from "
              << std::this_thread::get_id() << "\n";
}

void thread_basics() {
    std::jthread t1(hello_thread, 1);
    std::jthread t2(hello_thread, 2);
    // jthread 自动 join，无需手动管理
}

// ========== 2. mutex 与锁 ==========
class ThreadSafeCounter {
    mutable std::mutex mutex_;
    int value_ = 0;
public:
    void increment() {
        std::lock_guard lock(mutex_);  // RAII 锁
        ++value_;
    }

    int get() const {
        std::lock_guard lock(mutex_);
        return value_;
    }
};

// shared_mutex 读写锁
class ThreadSafeCache {
    mutable std::shared_mutex rw_mutex_;
    std::unordered_map<std::string, std::string> cache_;

public:
    std::string get(const std::string& key) const {
        std::shared_lock lock(rw_mutex_);  // 共享读锁
        auto it = cache_.find(key);
        return it != cache_.end() ? it->second : "";
    }

    void set(const std::string& key, std::string value) {
        std::unique_lock lock(rw_mutex_);  // 独占写锁
        cache_[key] = std::move(value);
    }
};

// ========== 3. 原子操作 ==========
class SpinLock {
    std::atomic_flag flag_ = ATOMIC_FLAG_INIT;
public:
    void lock() {
        while (flag_.test_and_set(std::memory_order_acquire)) {
            // 自旋等待
        }
    }
    void unlock() {
        flag_.clear(std::memory_order_release);
    }
};

// 无锁计数器
std::atomic<long> global_counter{0};

void atomic_worker() {
    for (int i = 0; i < 10000; ++i) {
        global_counter.fetch_add(1, std::memory_order_relaxed);
    }
}

// ========== 4. 条件变量（生产者-消费者） ==========
template <typename T>
class BoundedQueue {
    std::queue<T> queue_;
    std::mutex mutex_;
    std::condition_variable not_empty_;
    std::condition_variable not_full_;
    size_t max_size_;

public:
    explicit BoundedQueue(size_t max_size) : max_size_(max_size) {}

    void push(T item) {
        std::unique_lock lock(mutex_);
        not_full_.wait(lock, [this] { return queue_.size() < max_size_; });
        queue_.push(std::move(item));
        not_empty_.notify_one();
    }

    T pop() {
        std::unique_lock lock(mutex_);
        not_empty_.wait(lock, [this] { return !queue_.empty(); });
        T item = std::move(queue_.front());
        queue_.pop();
        not_full_.notify_one();
        return item;
    }
};

// ========== 5. future/promise 与 async ==========
long fibonacci(int n) {
    if (n <= 1) return n;
    return fibonacci(n - 1) + fibonacci(n - 2);
}

void future_example() {
    // std::async 启动异步任务
    std::future<long> f1 = std::async(std::launch::async, fibonacci, 40);
    std::future<long> f2 = std::async(std::launch::async, fibonacci, 35);
    
    // 等待并获取结果
    std::cout << "Fib(40) = " << f1.get() << "\n";
    std::cout << "Fib(35) = " << f2.get() << "\n";
    
    // promise 模式
    std::promise<std::string> prom;
    std::future<std::string> fut = prom.get_future();
    
    std::jthread worker([&prom] {
        std::this_thread::sleep_for(std::chrono::milliseconds(100));
        prom.set_value("Task completed!");
    });
    
    // 带超时的等待
    auto status = fut.wait_for(std::chrono::seconds(5));
    if (status == std::future_status::ready) {
        std::cout << "Result: " << fut.get() << "\n";
    } else {
        std::cout << "Timeout!\n";
    }
}

// ========== 7. 线程池 ==========
class ThreadPool {
    std::vector<std::jthread> workers_;
    BoundedQueue<std::function<void()>> tasks_;
    std::atomic<bool> running_{true};

public:
    explicit ThreadPool(size_t num_threads, size_t queue_size = 256)
        : tasks_(queue_size) {
        for (size_t i = 0; i < num_threads; ++i) {
            workers_.emplace_back([this] {
                while (running_.load(std::memory_order_acquire)) {
                    auto task = tasks_.pop();
                    task();
                }
            });
        }
    }

    void submit(std::function<void()> task) {
        tasks_.push(std::move(task));
    }

    ~ThreadPool() {
        running_.store(false, std::memory_order_release);
        // jthread 自动 join
    }
};

int main() {
    thread_basics();
    future_example();
    
    ThreadPool pool(4);
    for (int i = 0; i < 10; ++i) {
        pool.submit([i] {
            std::cout << "Task " << i << " on thread "
                      << std::this_thread::get_id() << "\n";
        });
    }
    
    std::cout << "Global counter: " << global_counter << "\n";
    return 0;
}

## 推荐书籍

1. **《C++ Concurrency in Action》（第2版）** — Anthony Williams
   C++ 并发编程的权威圣经，从 thread/mutex/atomic 基础到无锁数据结构和协程，涵盖所有并发主题。

2. **《C++20 - The Complete Guide》** — Nicolai M. Josuttis
   第 17 章详细讲解 C++20 协程——`co_await`、`co_yield`、`co_return` 以及协程类型的基础设施。

3. **《Concurrency with Modern C++》** — Rainer Grimm
   以清晰示例和图表讲解现代 C++ 并发的实战模式，包括并行算法、任务系统和并发设计模式。
