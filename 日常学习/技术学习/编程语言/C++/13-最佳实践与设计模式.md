# C++ 最佳实践与设计模式

## 详细描述

C++ 的最佳实践围绕几个核心原则展开：**RAII（资源获取即初始化）**、**零成本抽象**和**类型安全**。现代 C++ 的设计哲学是：在编译期发现尽可能多的错误，运行时做到零开销。

**C++ Core Guidelines** 是 Bjarne Stroustrup 和 Herb Sutter 维护的最佳实践集合。核心规则：I.x（接口）、F.x（函数）、C.x（类）、P.x（资源管理）、ES.x（表达式与语句）、T.x（模板）。

**RAII** 是 C++ 最重要的设计模式。资源在构造函数中获取，析构函数中释放——无论作用域如何退出，资源都被正确释放。这适用于内存、文件、锁、socket、数据库连接等所有资源。

**避免裸 new/delete**：`new` 和 `delete` 应该仅出现在低层资源管理代码中（如 unique_ptr 的实现）。应用代码使用 `make_unique`/`make_shared`、容器和 RAII。

**SOLID 原则适配**：C++ 中的接口通过抽象类（纯虚函数）表达；依赖注入通过构造函数或模板参数实现；开放封闭原则在 C++ 中更倾向于用策略模板而非运行时多态。

**现代 C++ 核心准则**：使用 `auto` 简化类型、优先使用 `{}` 初始化语法、使用 `nullptr` 而非 `NULL`、使用 `override` 和 `final`、使用 range-for 而非传统 for 循环、使用 `std::optional` 替代 sentinel 值。

## 代码实例

```cpp
#include <memory>
#include <iostream>

// ========== RAII 锁守卫 ==========
template <typename Mutex>
class LockGuard {
    Mutex& m_;
public:
    explicit LockGuard(Mutex& m) : m_(m) { m_.lock(); }
    ~LockGuard() { m_.unlock(); }
    LockGuard(const LockGuard&) = delete;
};

// ========== 策略模板（替代运行时多态） ==========
template <typename SortStrategy>
class Sorter {
    SortStrategy strategy_;
public:
    template <typename Container>
    void sort(Container& c) { strategy_(c); }
};

// ========== 工厂模式 + RAII ==========
class Resource {
public:
    virtual ~Resource() = default;
    virtual void use() = 0;
    static std::unique_ptr<Resource> create(const std::string& type);
};

// ========== pImpl 模式（减少编译依赖） ==========
class Widget {
    struct Impl;
    std::unique_ptr<Impl> pImpl_;
public:
    Widget();
    ~Widget();
    Widget(Widget&&) noexcept;
    void doWork();
};
```

## 推荐书籍

1. **《C++ Core Guidelines》** — Bjarne Stroustrup / Herb Sutter（在线免费）
   C++ 社区的最佳实践集中体现，每个规则都有动机、示例和例外说明。
2. **《Effective Modern C++》** — Scott Meyers
   42 个条款覆盖了现代 C++ 最关键的实践：类型推导、智能指针、移动语义、lambda。
3. **《C++ Software Design》** — Klaus Iglberger
   从设计模式角度讲解 C++ 的 SOLID 原则适配、策略模式、visitor 模式等现代实现。
