# C++ 错误处理

## 详细描述

C++ 的错误处理策略是多层次的：从经典的异常机制，到现代的错误码和 `std::expected`，再到编译期的 `static_assert`，不同场景选择不同工具。

**异常（Exception）**：`try/catch/throw` 是 C++ 标准化的错误传播机制。当 `throw` 抛出异常时，运行时沿调用栈向上展开（Stack Unwinding），在展开过程中调用所有局部对象的析构函数（RAII 保证资源正确释放），直到找到匹配的 `catch` 块。异常的代价主要在 throw 路径上（遍历栈、调用析构函数），正常路径（try 块内不含 throw）在现代编译器上几乎零开销。

**noexcept**（C++11）标记函数不会抛出异常，既作为文档承诺，也允许编译器进行优化（如移动 vs 拷贝的选择）。`noexcept` 函数中如果确实抛出了异常，程序会调用 `std::terminate()` 直接终止。`noexcept(expr)` 可以在编译期根据表达式结果决定是否 noexcept。

**错误码**是 C++ 中更传统的错误处理方式：函数返回表示成功/失败的整数值或枚举，通过输出参数或全局 `errno` 获取错误信息。虽然繁琐，但在禁用异常的环境中（嵌入式、游戏）是唯一选择。

**std::expected\<T, E\>**（C++23）是函数式错误处理在 C++ 中的标准化。它要么包含期望值（类型为 T），要么包含错误（类型为 E）。与异常不同，`expected` 强制调用者处理错误——你必须检查 `has_value()` 或使用 `value_or()`、`and_then()` 等一元操作。模式匹配（C++26）将使其更优雅。

**断言**分为运行时断言（`assert`）和编译时断言（`static_assert`）。`assert` 在 Debug 模式下检查，Release 模式下通常被移除；`static_assert` 在编译期检查条件，失败时产生友好的错误信息。

**错误处理哲学**：现代 C++ 推荐区分不同类型的失败——编程错误（逻辑 bug、前置条件违规）用断言；可恢复的运行时错误用异常或 `expected`；不可恢复的错误用 `std::terminate` 或 `std::abort`。

## 代码实例

```cpp
#include <iostream>
#include <expected>
#include <string>
#include <system_error>
#include <cassert>
#include <format>

// ========== 1. 异常基础 ==========
class DatabaseError : public std::runtime_error {
public:
    explicit DatabaseError(const std::string& msg)
        : std::runtime_error(msg) {}
};

class ConnectionError : public DatabaseError {
public:
    explicit ConnectionError(const std::string& host)
        : DatabaseError(std::format("Cannot connect to {}", host)) {}
};

class Database {
    std::string host_;
    bool connected_ = false;
public:
    explicit Database(std::string host) : host_(std::move(host)) {}

    void connect() {
        // 模拟连接逻辑
        if (host_.empty()) {
            throw ConnectionError("(empty host)");
        }
        if (host_ == "invalid") {
            throw ConnectionError(host_);
        }
        connected_ = true;
    }

    std::string query(const std::string& sql) {
        if (!connected_) {
            throw DatabaseError("Not connected");
        }
        if (sql.empty()) {
            throw std::invalid_argument("SQL cannot be empty");
        }
        return "result for: " + sql;
    }
};

void exception_example() {
    Database db("localhost");
    
    try {
        db.connect();
        auto result = db.query("SELECT * FROM users");
        std::cout << result << "\n";
    } catch (const ConnectionError& e) {
        std::cerr << "Connection failed: " << e.what() << "\n";
        // 重试逻辑或降级处理
    } catch (const DatabaseError& e) {
        std::cerr << "Database error: " << e.what() << "\n";
    } catch (const std::exception& e) {
        std::cerr << "Unexpected error: " << e.what() << "\n";
    }
    
    // 栈展开期间资源安全释放（RAII）
    // 所有局部对象的析构函数被调用
}

// ========== 2. noexcept ==========
class Widget {
    std::string name_;
public:
    explicit Widget(std::string name) : name_(std::move(name)) {}
    
    // noexcept 保证此移动操作不抛异常
    Widget(Widget&& other) noexcept : name_(std::move(other.name_)) {}
    
    Widget& operator=(Widget&& other) noexcept {
        name_ = std::move(other.name_);
        return *this;
    }
    
    const std::string& name() const noexcept { return name_; }
};

// 条件 noexcept
template <typename T>
void safe_swap(T& a, T& b) noexcept(
    std::is_nothrow_move_constructible_v<T> &&
    std::is_nothrow_move_assignable_v<T>
) {
    T temp = std::move(a);
    a = std::move(b);
    b = std::move(temp);
}

// ========== 3. std::expected（C++23） ==========
enum class MathError {
    DivisionByZero,
    Overflow,
    NegativeRoot,
};

std::expected<double, MathError> safe_divide(double a, double b) {
    if (b == 0.0) return std::unexpected(MathError::DivisionByZero);
    return a / b;
}

std::expected<double, MathError> safe_sqrt(double x) {
    if (x < 0) return std::unexpected(MathError::NegativeRoot);
    return std::sqrt(x);
}

// 组合 expected
std::expected<double, MathError> calculate(double a, double b, double c) {
    // and_then：如果成功则继续处理
    return safe_divide(a, b)
        .and_then([c](double result) { return safe_sqrt(result + c); })
        .transform([](double val) { return val * 2.0; })  // 成功时映射
        .or_else([](MathError e) -> std::expected<double, MathError> {
            // 从错误中恢复
            if (e == MathError::DivisionByZero) return 0.0;
            return std::unexpected(e);
        });
}

void expected_example() {
    auto result = calculate(10, 2, 6);  // 10/2 + 6 = 11, sqrt(11)*2
    if (result) {
        std::cout << "Result: " << *result << "\n";
    } else {
        std::cout << "Error occurred\n";
    }
    
    // value_or：提供默认值
    auto div_result = safe_divide(5, 0);
    double val = div_result.value_or(0.0);  // 返回 0.0
    
    // 模式匹配风格检查
    div_result.and_then([](double v) {
        std::cout << "Division succeeded: " << v << "\n";
        return std::expected<double, MathError>(v);
    });
}

// ========== 4. 断言 ==========
template <typename T>
T clamp(T value, T min, T max) {
    // 编译时断言：类型必须是算术类型
    static_assert(std::is_arithmetic_v<T>,
                  "clamp requires an arithmetic type");
    
    // 编译时断言：min <= max
    // （在 constexpr 上下文中检查）
    
    // 运行时断言（仅 Debug 模式）
    assert(min <= max && "min must be <= max");
    
    if (value < min) return min;
    if (value > max) return max;
    return value;
}

// ========== 5. 错误码模式 ==========
enum class FileError {
    NotFound = 1,
    PermissionDenied = 2,
    DiskFull = 3,
};

struct FileResult {
    FileError error;
    std::string context;
    
    explicit operator bool() const {
        return static_cast<int>(error) == 0;
    }
};

FileResult read_file(const std::string& path, std::string& content) {
    // 模拟文件读取
    if (path.empty()) {
        return {FileError::NotFound, "Path is empty"};
    }
    content = "file contents here";
    return {FileError{}, ""};  // 0 表示成功
}

int main() {
    exception_example();
    expected_example();
    
    auto val1 = clamp(5, 0, 10);     // OK
    auto val2 = clamp(3.14, 0.0, 1.0);  // OK
    // auto val3 = clamp("hello", "a", "z");  // static_assert 错误
    
    return 0;
}
```

## 推荐书籍

1. **《C++ Crash Course》** — Josh Lospinoso
   清晰讲解了异常、断言和 `std::expected` 在现代 C++ 中的最佳实践，包括 noexcept 的设计指导。

2. **《Effective Modern C++》** — Scott Meyers
   条款 14（noexcept）深入分析了 noexcept 的语义和移动构造的交互，是不可不知的实践。

3. **《C++ Software Design》** — Klaus Iglberger
   第 4 章从设计层面讨论错误处理策略——何时用异常、何时用错误码、如何构建统一的错误处理接口。
