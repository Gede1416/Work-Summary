# C++ 函数与作用域

## 详细描述

C++ 的函数系统层次丰富，从传统的自由函数到现代的 lambda 表达式，再到编译期求值的 constexpr 函数，构成了多层次的抽象能力。

**函数重载**是 C++ 静态多态的基础——同一作用域内函数名相同但参数列表不同即构成重载。重载决议（Overload Resolution）是编译器选择最匹配函数的过程，涉及实参类型到形参的精确匹配、提升、转换等多个级别。理解重载决议规则对避免"歧义调用"错误至关重要。

**默认参数**允许为函数参数提供默认值，调用时可省略。默认参数必须从右向左连续出现，且声明和定义中只能出现一次。注意默认参数的求值发生在调用点而非声明点，这可能导致微妙的生命周期问题。

**Lambda 表达式**（C++11）是函数式编程在 C++ 中的核心体现。基本语法为 `[capture](params) -> ret { body }`。捕获列表 `[]` 决定 lambda 如何访问外部变量：`[=]` 值捕获、`[&]` 引用捕获、`[this]` 捕获 this 指针、C++14 引入的 `[x = expr]` 初始化捕获。无捕获的 lambda 可以隐式转换为函数指针。泛型 lambda（C++14）使用 `auto` 参数，模板 lambda（C++20）可以直接使用模板参数列表。

**std::function** 是一个多态函数包装器，可以持有任何可调用对象（函数、lambda、函数对象、bind 结果），以类型擦除的方式提供统一接口。性能敏感的场景应当优先使用模板或 lambda。

**constexpr 函数**（C++11，C++14 放宽限制，C++20 允许 `try-catch` 和 `dynamic_cast`）可以在编译期求值。结合 `constexpr` 变量，可以在编译期完成复杂计算，实现零运行时开销。

**命名空间**解决了大型项目中的命名冲突。`using` 声明引入特定名称，`using` 指令引入整个命名空间（后者在头文件中应避免）。内联命名空间（C++11）支持版本管理，匿名命名空间提供文件作用域的内部链接。

**作用域**规则：块作用域、函数作用域、类作用域、命名空间作用域、全局作用域。名字查找遵循从内到外的规则。ADL（Argument Dependent Lookup，Koenig lookup）使得调用运算符重载时会在实参所在命名空间中查找。

## 代码实例

```cpp
#include <iostream>
#include <functional>
#include <vector>
#include <algorithm>
#include <memory>

// ========== 1. 函数重载与默认参数 ==========
namespace Calculator {
    // 重载：不同类型参数
    int add(int a, int b) { return a + b; }
    double add(double a, double b) { return a + b; }
    
    // 默认参数
    double power(double base, int exp = 2) {
        double result = 1;
        for (int i = 0; i < exp; ++i) result *= base;
        return result;
    }
}

// ========== 2. Lambda 表达式 ==========
void lambda_examples() {
    int multiplier = 10;
    int sum = 0;
    
    // 基础 lambda
    auto greet = []() { return "Hello"; };
    
    // 值捕获：[=]
    auto multiply = [=](int x) { return x * multiplier; };
    
    // 引用捕获：[&]，可以修改外部变量
    auto add_to_sum = [&](int x) { sum += x; };
    
    // 初始化捕获（C++14）
    auto ptr = std::make_unique<int>(42);
    auto owner = [p = std::move(ptr)]() { return *p; };
    // ptr 现在为 nullptr
    
    // 泛型 lambda（C++14）
    auto generic_add = [](auto a, auto b) { return a + b; };
    std::cout << generic_add(1, 2) << "\n";       // 3
    std::cout << generic_add(1.5, 2.7) << "\n";   // 4.2
    
    // 模板 lambda（C++20）
    auto template_lambda = []<typename T>(std::vector<T> const& vec) {
        return vec.size();
    };
    
    // mutable lambda：允许修改值捕获的副本
    int counter = 0;
    auto incrementer = [counter]() mutable { return ++counter; };
    std::cout << incrementer() << "\n"; // 1
    std::cout << incrementer() << "\n"; // 2
    std::cout << counter << "\n";       // 0（原始变量未改变）
}

// ========== 3. std::function ==========
class TaskScheduler {
    std::vector<std::function<void()>> tasks_;
public:
    void schedule(std::function<void()> task) {
        tasks_.push_back(std::move(task));
    }
    
    void run_all() {
        for (auto& task : tasks_) task();
    }
};

void function_examples() {
    TaskScheduler scheduler;
    
    // 存储普通函数
    scheduler.schedule([]() { std::cout << "Task 1\n"; });
    
    // 存储带状态的 lambda
    int progress = 0;
    scheduler.schedule([&progress]() {
        progress += 25;
        std::cout << "Progress: " << progress << "%\n";
    });
    
    scheduler.run_all();
}

// ========== 4. constexpr 函数 ==========
constexpr int fib(int n) {
    if (n <= 1) return n;
    return fib(n - 1) + fib(n - 2);
}

constexpr int fib10 = fib(10);  // 编译期求值 = 55

// C++20：编译期 std::vector（在 constexpr 上下文中）
constexpr auto make_squares(int n) {
    std::vector<int> result;
    for (int i = 1; i <= n; ++i) result.push_back(i * i);
    return result;
}
constexpr auto squares = make_squares(5);  // {1, 4, 9, 16, 25}

// ========== 5. 命名空间与作用域 ==========
namespace Lib {
    namespace v1 {
        void process() { std::cout << "version 1\n"; }
    }
    
    // 内联命名空间：默认版本（C++11）
    inline namespace v2 {
        void process() { std::cout << "version 2 (default)\n"; }
    }
}

void namespace_examples() {
    Lib::process();       // 默认调用 v2
    Lib::v1::process();   // 显式调用 v1
    
    using Lib::v1::process;  // using 声明
    process();               // 现在调用 v1
}

// ========== ADL（参数依赖查找） ==========
namespace Audio {
    struct Note { int pitch; };
    void play(Note n) { std::cout << "Playing note: " << n.pitch << "\n"; }
}

void adl_example() {
    Audio::Note n{60};
    play(n);  // ADL：在 Audio 命名空间中找到 play，无需写 Audio::play(n)
}

int main() {
    using namespace Calculator;
    std::cout << add(1, 2) << "\n";       // int 版本
    std::cout << add(1.5, 2.5) << "\n";   // double 版本
    std::cout << power(3.0) << "\n";      // 默认参数，平方
    std::cout << power(2.0, 10) << "\n";  // 1024
    
    lambda_examples();
    function_examples();
    namespace_examples();
    adl_example();
    
    return 0;
}
```

## 推荐书籍

1. **《C++ Lambda Story》** — Bartłomiej Filipek
   专门讲 C++ lambda 表达式的书，从 C++11 到 C++20，涵盖捕获、泛型 lambda、性能、与 STL 算法的配合等。

2. **《C++17 - The Complete Guide》** — Nicolai M. Josuttis
   其中有专门章节讲解 constexpr 函数在 C++17 中的扩展、折叠表达式等编译期特性。

3. **《Effective Modern C++》** — Scott Meyers
   条款 31-34 详细讨论了 lambda 的捕获、初始化捕获、完美转发等，是理解 lambda 的最佳参考。
