# TypeScript 面向对象与模块化

## 详细描述

TypeScript 在 JavaScript 的原型继承之上，提供了一套熟悉的类语法和强类型的面向对象抽象，同时保留了 JS 的灵活性。它的模块系统以 ES 模块为基础，辅以声明文件机制，为大型项目提供了良好的组织能力。

**class** 是 TS 中最核心的 OOP 抽象。它支持 `extends` 继承、`implements` 接口实现、`abstract` 抽象类和抽象方法。与 JS 一样，TS 的类在运行时仍然是基于原型的，但编译器提供了静态类型检查和更清晰的语法。访问修饰符有 `public`（默认）、`protected`、`private`（编译时检查，运行时仍然可访问），以及 TS 特有的 `readonly` 属性修饰符。构造函数参数前加修饰符可自动声明并初始化成员（参数属性）。

**interface** 只存在于编译时，定义类型的"形状"（shape）。interface 可以继承多个其他 interface，class 可以实现多个 interface。与 C# 不同，TS interface 支持可选属性（`?`）、只读属性（`readonly`）、索引签名、函数调用签名。

**type 别名**与 interface 有很多重叠，但 type 更灵活——支持联合/交叉/映射类型，而 interface 支持声明合并。惯例是：描述对象形状用 interface，需要组合/转换类型用 type。

**抽象类**不能直接实例化，可以包含抽象方法（子类必须实现）和具体方法。与 interface 的关键区别：抽象类可以有运行时实现，生成实际的 JS 代码。

**装饰器**（Decorators）是 TS 的实验性和传统特性，用于元编程——可以修改类、方法、属性、参数的元数据和行为。`reflect-metadata` 库可以在运行时访问装饰器存储的类型元数据。NestJS、Angular 等框架大量使用装饰器进行依赖注入和路由定义。

**ES 模块与 CommonJS**：TS 支持这两种主流模块系统。ES 模块使用 `import/export`，输出的是只读的绑定引用（Live Bindings）；CommonJS 使用 `require/module.exports`，输出的是值的副本。`tsconfig.json` 中的 `module` 选项控制编译目标（`esnext`、`commonjs`、`nodenext` 等），`moduleResolution` 控制模块查找策略。

**声明文件**（`.d.ts`）是 TS 生态的关键——为没有类型的 JS 库提供类型定义。通过 `declare` 关键字声明变量、函数、类、模块的类型，不产生运行时代码。`@types/*` 包（DefinitelyTyped 社区）维护着数千个库的类型定义。

**namespace**（内部模块）现在主要用于声明文件中和跨文件组织类型定义。在现代 TS 项目中，ES 模块已经取代了 namespace 的大部分用途。声明合并（同名 interface/namespace 自动合并）在扩展第三方类型时很有用。

## 代码实例

```typescript
// ========== 1. Class、继承与修饰符 ==========

// 接口：仅类型层面
interface Identifiable {
    id: string;
}

interface Auditable {
    createdAt: Date;
    updatedAt: Date;
    markUpdated(): void;
}

// 抽象类：有运行时实现
abstract class Entity implements Identifiable, Auditable {
    // 参数属性：自动声明并赋值
    constructor(
        public readonly id: string,
        public readonly createdAt: Date = new Date(),
        protected _updatedAt: Date = new Date(),
    ) {}

    get updatedAt(): Date {
        return this._updatedAt;
    }

    // 抽象方法：子类必须实现
    abstract describe(): string;

    // 具体方法：子类继承
    markUpdated(): void {
        this._updatedAt = new Date();
    }
}

// 具体类
class Customer extends Entity {
    constructor(
        id: string,
        public readonly name: string,
        public readonly email: string,
    ) {
        super(id);
    }

    describe(): string {
        return `Customer ${this.name} <${this.email}>`;
    }

    // 子类特有方法
    sendPromotion(product: string): void {
        console.log(`Sending promotion for ${product} to ${this.email}`);
    }
}

// 实现多个接口
interface Flyable {
    fly(): void;
}

interface Swimmable {
    swim(): void;
}

class Duck extends Entity implements Flyable, Swimmable {
    constructor(id: string, public readonly name: string) {
        super(id);
    }

    describe(): string {
        return `Duck: ${this.name}`;
    }

    fly(): void {
        console.log(`${this.name} flies`);
    }

    swim(): void {
        console.log(`${this.name} swims`);
    }
}

// ========== 2. Interface vs Type ==========

interface User {
    name: string;
    age: number;
}

// interface 扩展
interface AdminUser extends User {
    role: "admin";
    permissions: string[];
}

// type 别名：联合/交叉
type Role = "admin" | "user" | "guest";
type Permission = "read" | "write" | "delete";
type RolePermissions = Record<Role, Permission[]>;

// interface 与 type 互操作
type UserOrId = User | { id: number };

interface ExtendedUser extends User {
    tags: string[];
}

// ========== 3. 装饰器 ==========
function LogMethod(
    target: any,
    propertyKey: string,
    descriptor: PropertyDescriptor,
) {
    const original = descriptor.value;
    descriptor.value = function (...args: any[]) {
        console.log(`Calling ${propertyKey}(${args.join(", ")})`);
        const result = original.apply(this, args);
        console.log(`${propertyKey} returned ${result}`);
        return result;
    };
}

function Validate(target: any, propertyKey: string, index: number) {
    const existingValidators: number[] =
        Reflect.getOwnMetadata("validators", target, propertyKey) ?? [];
    existingValidators.push(index);
    Reflect.defineMetadata("validators", existingValidators, target, propertyKey);
}

class Calculator {
    @LogMethod
    add(@Validate a: number, @Validate b: number): number {
        return a + b;
    }
}

// ========== 4. ES 模块系统 ==========

// --- shapes.ts ---
export interface Shape {
    area(): number;
    perimeter(): number;
}

export class Circle implements Shape {
    constructor(private radius: number) {}
    area(): number { return Math.PI * this.radius ** 2; }
    perimeter(): number { return 2 * Math.PI * this.radius; }
}

export class Rectangle implements Shape {
    constructor(private width: number, private height: number) {}
    area(): number { return this.width * this.height; }
    perimeter(): number { return 2 * (this.width + this.height); }
}

// 默认导出
export default function createShape(type: "circle" | "rectangle", ...args: number[]): Shape {
    if (type === "circle") return new Circle(args[0]);
    return new Rectangle(args[0], args[1]);
}

// --- app.ts ---
// import createShape, { Circle, Rectangle, Shape } from "./shapes";
// export type { Shape } from "./shapes";  // 仅类型导出

// ========== 5. 声明文件与声明合并 ==========

// 全局声明（在 .d.ts 文件中）
declare global {
    interface Window {
        myAppConfig: {
            apiUrl: string;
            debug: boolean;
        };
    }
}

// 声明合并：扩展已有 interface
interface Array<T> {
    last(): T | undefined;
}

// 实现扩展的方法
Array.prototype.last = function <T>(this: T[]): T | undefined {
    return this[this.length - 1];
};

console.log([1, 2, 3].last()); // 3

// namespace 声明合并（用于 DefinitelyTyped 场景）
declare namespace Express {
    interface Request {
        user?: { id: string; role: string };
    }
}

// ========== 6. 依赖注入模式 ==========
interface Logger {
    log(message: string): void;
}

class ConsoleLogger implements Logger {
    log(message: string): void {
        console.log(`[${new Date().toISOString()}] ${message}`);
    }
}

class UserService {
    constructor(private logger: Logger) {}
    
    createUser(name: string): void {
        this.logger.log(`Creating user: ${name}`);
        // ...业务逻辑
    }
}

// 手动 DI 容器
const logger = new ConsoleLogger();
const userService = new UserService(logger);
```

## 推荐书籍

1. **《Programming TypeScript》** — Boris Cherny
   第 5 章（Classes and Interfaces）和第 9 章（Namespaces.Modules），系统讲解 TS 的 OOP 和模块化。

2. **《Effective TypeScript》** — Dan Vanderkam
   条款 30-37 涵盖 interface vs type、声明合并、模块系统的实践建议。

3. **《TypeScript Design Patterns》** — Vilic Vane
   详细介绍 TS 中的经典设计模式实现，包括装饰器、工厂、观察者模式在 TS 中的应用。
