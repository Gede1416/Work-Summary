# TypeScript 异步与并发编程

## 详细描述

TypeScript 的异步模型基于 JavaScript 的**事件循环**——一个单线程、非阻塞的 I/O 模型。理解事件循环、Promise 和 Worker Threads 是编写高效 TS 应用的基础。

**事件循环**（Event Loop）是 JS/TS 运行时的核心机制。它不断检查调用栈是否为空，然后将任务队列（Task Queue）中的任务推入执行。任务分为宏任务（Macro-task：setTimeout、setInterval、I/O）和微任务（Micro-task：Promise.then/catch/finally、queueMicrotask、MutationObserver）。每轮事件循环先清空微任务队列，再取一个宏任务执行——理解这个顺序对于避免竞态条件至关重要。

**Promise** 是 TS 异步编程的基石。Promise 有三种状态：pending → fulfilled（resolved）或 rejected。`.then()` 注册成功回调，`.catch()` 注册失败回调，`.finally()` 注册无论结果都执行的回调。`Promise.all()` 等待所有 Promise 完成（一个失败整体失败），`Promise.allSettled()` 等待所有完成但不管成功/失败，`Promise.race()` 返回第一个完成的，`Promise.any()` 返回第一个成功的。

**async/await** 是 Promise 的语法糖，让异步代码看起来像同步代码。`async` 函数始终返回 Promise，`await` 暂停函数执行直到 Promise 完成。与 C# 不同，JS/TS 中没有 SynchronizationContext 的概念——await 后的代码总是在当前"事件循环任务"的微任务中执行。

**Worker Threads**（Node.js）和 **Web Workers**（浏览器）实现了真正的并行——在不同线程中运行 JS 代码。Worker 通过 `postMessage` 和 `onmessage` 与主线程通信，使用结构化克隆或 Transferable 对象传输数据。Worker 之间不共享内存（除非使用 SharedArrayBuffer）。

**Atomics 与 SharedArrayBuffer** 提供了低级别的共享内存和多线程同步原语。`Atomics.wait()` 和 `Atomics.notify()` 实现类似条件变量的机制，`Atomics.add()`、`Atomics.compareExchange()` 实现原子操作。这是实现自定义锁、无锁数据结构和高性能并行计算的基础。

**微任务 vs 宏任务**的调度选择影响关键路径的性能。`queueMicrotask()` 将任务插入微任务队列（同一轮循环立即执行），`setTimeout(fn, 0)` 插入宏任务队列（下一轮循环执行）。Promise 回调总是在微任务中，而用户交互、网络事件在宏任务中。

// ========== 1. 事件循环与任务队列 ==========

function demonstrateEventLoop(): void {
    console.log("1: 同步代码开始");

    setTimeout(() => console.log("2: 宏任务 - setTimeout"), 0);

    Promise.resolve().then(() => console.log("3: 微任务 - Promise.then"));

    queueMicrotask(() => console.log("4: 微任务 - queueMicrotask"));

    console.log("5: 同步代码结束");
}

// 输出顺序: 1, 5, 3, 4, 2
// 原因：同步代码 → 清空微任务 → 取一个宏任务

// ========== 2. Promise 组合 ==========

async function fetchUserProfile(userId: string): Promise<UserProfile> {
    const [user, posts, followers] = await Promise.all([
        fetch(`/api/users/${userId}`).then(r => r.json()),
        fetch(`/api/users/${userId}/posts`).then(r => r.json()),
        fetch(`/api/users/${userId}/followers`).then(r => r.json()),
    ]);
    return { user, posts, followers };
}

// 部分失败不影响其他
async function fetchAllSafe(urls: string[]): Promise<Result<string, Error>[]> {
    return Promise.allSettled(
        urls.map(url => fetch(url).then(r => r.text())),
    ).then(results =>
        results.map(r =>
            r.status === "fulfilled"
                ? { ok: true as const, value: r.value }
                : { ok: false as const, error: r.reason as Error }
        ),
    );
}

// 超时竞赛
async function fetchWithTimeout(url: string, timeoutMs: number): Promise<Response> {
    const controller = new AbortController();
    const timeout = setTimeout(() => controller.abort(), timeoutMs);

    try {
        return await fetch(url, { signal: controller.signal });
    } finally {
        clearTimeout(timeout);
    }
}

// ========== 3. async/await 高级模式 ==========

// 限制并发数量
async function mapWithConcurrency<T, R>(
    items: T[],
    fn: (item: T) => Promise<R>,
    concurrency: number = 5,
): Promise<R[]> {
    const results: R[] = new Array(items.length);
    let index = 0;

    async function worker(): Promise<void> {
        while (index < items.length) {
            const currentIndex = index++;
            results[currentIndex] = await fn(items[currentIndex]);
        }
    }

    const workers = Array.from({ length: concurrency }, () => worker());
    await Promise.all(workers);
    return results;
}

// 使用：一次最多 3 个并发请求
const urls = Array.from({ length: 100 }, (_, i) => `/api/item/${i}`);
const data = await mapWithConcurrency(urls, async (url) => {
    const res = await fetch(url);
    return res.json();
}, 3);

// 重试模式
async function retry<T>(
    fn: () => Promise<T>,
    options: { maxRetries?: number; delayMs?: number; backoff?: number } = {},
): Promise<T> {
    const { maxRetries = 3, delayMs = 1000, backoff = 2 } = options;

    for (let attempt = 0; attempt <= maxRetries; attempt++) {
        try {
            return await fn();
        } catch (error) {
            if (attempt === maxRetries) throw error;
            const delay = delayMs * Math.pow(backoff, attempt);
            console.warn(`Attempt ${attempt + 1} failed, retrying in ${delay}ms`);
            await new Promise(resolve => setTimeout(resolve, delay));
        }
    }
    throw new Error("Unreachable");
}

// ========== 6. AbortController 取消异步操作 ==========

class CancellableTask {
    private controller = new AbortController();

    async executeAsync(): Promise<void> {
        try {
            const response = await fetch("/api/long-running", {
                signal: this.controller.signal,
            });
            const data = await response.json();
            console.log("Data:", data);
        } catch (error) {
            if (error instanceof DOMException && error.name === "AbortError") {
                console.log("Task was cancelled");
                return;
            }
            throw error;
        }
    }

    cancel(): void {
        this.controller.abort();
    }
}

// 取消令牌模式（类似 C# CancellationToken）
class CancellationToken {
    private _isCancelled = false;
    private listeners: Array<() => void> = [];

    get isCancelled(): boolean {
        return this._isCancelled;
    }

    cancel(): void {
        this._isCancelled = true;
        this.listeners.forEach(fn => fn());
        this.listeners = [];
    }

    register(fn: () => void): void {
        if (this._isCancelled) {
            fn();
        } else {
            this.listeners.push(fn);
        }
    }

    throwIfCancelled(): void {
        if (this._isCancelled) throw new Error("Operation cancelled");
    }
}

## 推荐书籍

1. **《Node.js Design Patterns》（第3版）** — Mario Casciaro / Luciano Mammino
   第 3 章（Callbacks and Events）、第 5 章（Asynchronous Patterns）、第 11 章（Advanced Recipes）是 Node.js 异步编程的系统教材。

2. **《Effective TypeScript》** — Dan Vanderkam
   条款 46 讨论了 Promise 的类型安全使用，包括 async/await 与 Promise.all 的类型推断。

3. **《High Performance Browser Networking》** — Ilya Grigorik
   第 1-2 章深入讲解了浏览器的事件循环机制，对理解 JS/TS 的异步模型至关重要。

4. **《JavaScript: The New Toys》** — T.J. Crowder
   讲解 ES2015-2020 的 Promise、async/await、Worker、SharedArrayBuffer 等新特性。
