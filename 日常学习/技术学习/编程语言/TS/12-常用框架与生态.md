# TypeScript 常用框架与生态

## 详细描述

TypeScript 的框架生态是当今最活跃的技术生态之一，覆盖前端、后端、全栈、工具链等各个领域。

**前端框架**：
- **React + TS**：最流行的 UI 库，通过 `@types/react` 提供完整的类型支持。React 18 的 Concurrent Features、Hooks 与 TS 泛型结合产生强大的类型推断。`React.FC`、`useState<T>`、`useRef<T>` 都有完整的类型推导。
- **Vue 3 + TS**：Vue 3 完全用 TS 重写，Composition API 与 TS 配合非常自然。`defineProps<T>()` 和 `defineEmits<T>()` 提供类型安全的组件接口。
- **Angular**：最早且最深入的 TS 采用者，整个框架重度依赖装饰器和类型系统。

**后端框架**：
- **NestJS**：受 Angular 启发的企业级 Node.js 框架，模块化架构、依赖注入、装饰器。与 Express/Fastify 集成，内置 GraphQL、WebSocket、微服务支持。
- **Next.js**（React 全栈框架）：App Router、Server Components、API Routes，全面支持 TS。React Server Components 的异步组件模型是近年最具影响力的范式变革。
- **tRPC**：端到端类型安全的 API 框架。在服务端定义函数签名，客户端自动获得类型推断——无需声明文件、无需代码生成。

**数据库与 ORM**：
- **Prisma**：类型安全的 ORM，通过 Schema 文件生成 TS 类型和查询客户端。迁移管理、关系查询、原始 SQL 支持。
- **Drizzle**：新兴的 TypeScript ORM，无代码生成，基于类型推断，像写 SQL 一样写查询。

**运行时环境**：
- **Deno**：Node.js 创始人打造的新一代 JS/TS 运行时，原生支持 TS（无需编译配置）、安全权限模型、内置工具链（formatter、linter、test runner）。
- **Bun**：基于 JavaScriptCore 的极速运行时，内置打包器、包管理器、测试框架。

## 推荐书籍

1. **《Modern Full-Stack React with TypeScript》** — 多本在线资源
   覆盖 Next.js 14+、Server Components、tRPC、Prisma 的全栈开发最佳实践。
2. **《NestJS: A Progressive Node.js Framework》** — Greg Magolan
   NestJS 核心成员的权威指导，包括微服务和 GraphQL 的深入教程。
3. **《Learning React》（第2版）** — Alex Banks / Eve Porcello
   虽然主要讲 React，但所有示例都用现代 JS 编写，加上 TS 类型很容易适配。
