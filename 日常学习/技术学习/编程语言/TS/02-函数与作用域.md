# TypeScript 函数与作用域

## 详细描述

TypeScript 的函数系统在 JavaScript 的灵活性之上添加了类型约束，使函数签名成为可靠的契约。理解 TS 函数特性是编写类型安全代码的基础。

**函数声明与表达式**：函数声明会被提升（hoisting），可以在定义前调用；函数表达式则不会。TS 中函数参数和返回值都可以添加类型注解，编译器会进行类型检查。可选参数用 `?` 标记，默认参数直接赋值，剩余参数通过 `...args: Type[]` 收集。

**箭头函数**（`=>`）不仅语法简洁，更有独特的 `this` 绑定行为——箭头函数没有自己的 `this`，而是捕获定义时所在作用域的 `this`。这一特性使其成为回调和 React 组件方法中的首选写法。TS 中箭头函数的类型推断通常比普通函数更好。

**this 类型**（TypeScript 特有）可以在类和接口的方法返回类型中使用，表示"调用者自身的类型"。这对于流畅接口（fluent interface）和继承场景非常有用——链式调用时每个方法返回正确的子类类型。

**参数解构**结合类型注解，可以安全地对对象和数组参数进行解构：`function greet({ name, age }: { name: string; age: number })`。既保持了简洁的解构语法，又获得完整的类型检查。

**闭包**是 JS/TS 的核心概念——内部函数可以访问外部函数的变量，即使外部函数已经返回。这在模块模式、柯里化、事件处理中广泛应用。闭包可能导致内存泄漏（被捕获的变量无法被 GC 回收），需谨慎管理。

**模块作用域**是 TS/ES 模块的默认行为——每个文件有自己的作用域，不会污染全局。通过 `export` 暴露接口，`import` 引入依赖。TS 还支持 `export type` 和 `import type` 用于纯类型的导入导出。

**全局声明合并**（Declaration Merging）是 TS 特有的能力——同名 interface 会自动合并，同名 namespace 也会合并。这让用户可以扩展第三方库的类型定义，也可以在不同文件中向同一个 interface 添加属性。

**剩余参数与展开**：`...args` 在函数参数中收集多余实参为一个数组，`...array` 在调用时将数组展开为独立实参。TS 为两者提供了完整的类型推断。

**函数重载**在 TS 中以声明多个函数签名然后提供单一实现的方式表现。前几个是重载签名（overload signatures），最后一个是实现签名（implementation signature），编译器根据调用参数的个数和类型选择对应的重载签名进行类型检查。

## 代码实例

```typescript
// ========== 1. 函数声明与表达式 ==========

// 函数声明（会提升）
function add(a: number, b: number): number {
    return a + b;
}

// 函数表达式（不会提升）
const multiply = function(a: number, b: number): number {
    return a * b;
};

// 可选参数
function greet(name: string, title?: string): string {
    return title ? `${title} ${name}` : name;
}

// 默认参数
function createUser(name: string, age: number = 18, admin: boolean = false) {
    return { name, age, admin };
}

// 剩余参数
function sum(...numbers: number[]): number {
    return numbers.reduce((acc, n) => acc + n, 0);
}
sum(1, 2, 3, 4, 5); // 15

// ========== 2. 箭头函数与 this ==========

class EventHandler {
    private count = 0;

    // 箭头函数：自动绑定 this
    handle = () => {
        this.count++;
        console.log(this.count);
    };
}

// this 类型（流畅接口模式）
class FluentBuilder {
    private data: Record<string, unknown> = {};

    set(key: string, value: unknown): this {
        this.data[key] = value;
        return this;
    }

    build(): Record<string, unknown> {
        return { ...this.data };
    }
}

class ExtendedBuilder extends FluentBuilder {
    log(): this {
        console.log(this);
        return this;
    }
}

const builder = new ExtendedBuilder();
builder.set("a", 1).log().build(); // this 类型保证链式调用的类型安全

// ========== 3. 参数解构 ==========

// 对象解构 + 类型注解
function printUser({
    name,
    age,
    email,
}: {
    name: string;
    age: number;
    email?: string;
}): void {
    console.log(`${name}, ${age}岁`);
    if (email) console.log(`邮箱: ${email}`);
}

// 配合接口更清晰
interface Config {
    host: string;
    port: number;
    ssl?: boolean;
    retries?: number;
}

function connect({ host, port, ssl = false, retries = 3 }: Config): void {
    const protocol = ssl ? "https" : "http";
    console.log(`连接 ${protocol}://${host}:${port}, 重试${retries}次`);
}

// 数组解构
function firstAndRest<T>([first, ...rest]: T[]): [T, T[]] {
    return [first, rest];
}

// ========== 4. 函数重载（TS 风格） ==========

// 重载签名
function stringOrNumber(value: string): string[];
function stringOrNumber(value: number): number[];
// 实现签名
function stringOrNumber(value: string | number): string[] | number[] {
    if (typeof value === "string") {
        return value.split("");
    }
    return [value];
}

const chars = stringOrNumber("hello");  // string[]
const nums = stringOrNumber(42);        // number[]

// 带条件类型的重载
function process<T>(input: T): T extends string ? number : string;
function process(input: string | number): number | string {
    if (typeof input === "string") return input.length;
    return String(input);
}

const n: number = process("hello"); // 返回 number
const s: string = process(42);      // 返回 string

// ========== 5. 闭包与模块模式 ==========

// 经典闭包：计数器
function createCounter(initial: number = 0) {
    let count = initial;
    return {
        increment: () => ++count,
        decrement: () => --count,
        get value() { return count; },
    };
}

const counter = createCounter(10);
counter.increment(); // 11
counter.increment(); // 12
console.log(counter.value); // 12

// 柯里化
function curry<T extends unknown[], U extends unknown[], R>(
    fn: (...args: [...T, ...U]) => R,
    ...fixedArgs: T
): (...args: U) => R {
    return (...rest) => fn(...fixedArgs, ...rest);
}

const multiply2 = (a: number, b: number, c: number) => a * b * c;
const multiplyBy2 = curry(multiply2, 2);
const result = multiplyBy2(3, 4); // 24

// ========== 6. this 参数（TS 特有） ==========

// 声明 this 的类型（假的参数，编译后移除）
interface Card {
    suit: string;
    rank: number;
}

function describeCard(this: Card): string {
    return `${this.rank} of ${this.suit}`;
}

const card: Card = { suit: "hearts", rank: 10 };
// describeCard(); // ❌ 错误：this 上下文不匹配
const boundDescribe = describeCard.bind(card);
console.log(boundDescribe()); // "10 of hearts"

// noImplicitThis 必须声明 this 类型
function handleClick(this: HTMLElement, event: Event) {
    console.log(this.id);
}
```

## 推荐书籍

1. **《Programming TypeScript》** — Boris Cherny
   第 4 章专门讲函数，从函数类型、this 类型、重载到泛型函数，解释清晰，示例实用。

2. **《Effective TypeScript》** — Dan Vanderkam
   条款 14-20 涵盖函数相关的实践：参数类型、重载、剩余参数、this 类型，帮助写出更安全的函数。

3. **《TypeScript Cookbook》** — Stefan Baumgartner
   包含大量函数式编程的 TS 实践：柯里化、函数组合、高阶函数类型等。
