# TypeScript 内存管理

## 详细描述

TypeScript 本身不管理内存——它编译为 JavaScript，完全依赖 JS 运行时的垃圾回收器。但了解底层的内存管理机制对于编写内存高效的 Node.js/Deno 应用和避免前端内存泄漏至关重要。

**JS 运行时垃圾回收**：现代 JS 引擎（V8、SpiderMonkey、JavaScriptCore）都使用分代标记-清除（Mark-and-Sweep）GC。新对象分配在"新生代"（New Space），存活的对象被提升到"老生代"（Old Space）。标记阶段从根对象（全局对象、局部变量）开始递归追踪所有可达对象，清除阶段回收不可达对象的内存。

**V8 内存结构**（Node.js/Deno 的关键知识）：
- **New Space**：新生代，大小通常 1-8 MB，使用 Scavenge 算法（Cheney 复制）快速回收
- **Old Space**：老生代，存活较久的对象，使用标记-清除-整理（Mark-Sweep-Compact）
- **Large Object Space**：大对象直接分配的区域
- **Code Space**：JIT 编译的代码存储区

**WeakMap 和 WeakSet**（ES2015）是避免内存泄漏的重要工具。它们的"弱引用"特性意味着：如果没有其他强引用指向键/值，GC 可以回收这些条目。典型场景包括关联元数据到 DOM 元素或第三方对象，而不会阻止这些对象被回收。

**内存泄漏的常见原因**：
- **全局变量**：未用 `let`/`const` 声明的变量泄漏到全局
- **遗忘的定时器/事件监听器**：`setInterval` 和 `addEventListener` 保持对回调的引用，组件销毁时需清理
- **闭包陷阱**：大对象被小闭包意外捕获，导致整个作用域链无法被回收
- **分离的 DOM 节点**：JavaScript 持有了已从 DOM 树移除的节点引用

**手动解除引用**：虽然不需要（也不能）手动释放内存，但可以通过将变量赋值为 `null` 来主动断开引用链，让 GC 更早回收。

**FinalizationRegistry**（ES2021）允许在对象被垃圾回收时执行回调，但官方建议仅在万不得已时使用——GC 时间和顺序不可预测。

## 代码实例

```typescript
// ========== 1. WeakMap 用例：关联元数据 ==========

// 场景：为 DOM 元素或任意对象关联额外数据，不影响 GC
const elementMetadata = new WeakMap<object, {
    clickCount: number;
    lastAccess: number;
}>();

class Tracker {
    track(element: object): void {
        if (!elementMetadata.has(element)) {
            elementMetadata.set(element, {
                clickCount: 0,
                lastAccess: Date.now(),
            });
        }
    }

    click(element: object): void {
        const meta = elementMetadata.get(element);
        if (meta) {
            meta.clickCount++;
            meta.lastAccess = Date.now();
        }
    }

    getStats(element: object) {
        return elementMetadata.get(element);
    }
}

// 当元素被移除且无其他引用时，WeakMap 中的条目自动被回收
// 如果用 Map，条目将永远保留（内存泄漏）

// ========== 2. 避免闭包内存泄漏 ==========

// ❌ 错误：重量级对象被闭包捕获
function createLeakyHandler() {
    // 假设这是一个大数据集
    const largeData = new Array(1000000).fill({ complex: "data" });

    return function handleClick(event: Event) {
        // 即使只用了一个小属性，整个 largeData 都不会被回收
        console.log("Clicked at", event.clientX);
    };
}

// ✅ 正确：只捕获需要的部分
function createCleanHandler(prefix: string) {
    return function handleClick(event: Event) {
        console.log(prefix, "Clicked at", event.clientX);
    };
}

// ========== 3. 事件监听器管理 ==========

class LazyComponent {
    private cleanupFns: Array<() => void> = [];
    private element: HTMLElement;

    constructor(container: HTMLElement) {
        this.element = container;
        this.setup();
    }

    private setup(): void {
        // AbortController 管理一组监听器
        const controller = new AbortController();
        const { signal } = controller;

        this.element.addEventListener("click", this.handleClick, { signal });
        this.element.addEventListener("keydown", this.handleKeydown, { signal });

        // 注册清理
        this.cleanupFns.push(() => controller.abort());
    }

    private handleClick = (): void => {
        console.log("clicked");
    };

    private handleKeydown = (event: KeyboardEvent): void => {
        console.log("key:", event.key);
    };

    destroy(): void {
        // 一次性清理所有监听器和定时器
        this.cleanupFns.forEach(fn => fn());
        this.cleanupFns = [];
    }
}

// ========== 4. FinalizationRegistry（谨慎使用） ==========
const registry = new FinalizationRegistry((heldValue: string) => {
    console.log(`Object with value "${heldValue}" was garbage collected`);
});

class MonitoredObject {
    constructor(public readonly label: string) {
        registry.register(this, label, this);
    }
}

function demoFinalization(): void {
    let obj = new MonitoredObject("temp");
    // 当 obj 被 GC 回收时，registry 的回调被调用
    obj = null as any;  // 断开引用
    // GC 时机不确定，回调可能很久之后才触发
}

// ========== 5. 对象池模式（减少 GC 压力） ==========

class ObjectPool<T> {
    private pool: T[] = [];
    private factory: () => T;
    private reset: (item: T) => void;

    constructor(factory: () => T, reset: (item: T) => void, preAllocate = 0) {
        this.factory = factory;
        this.reset = reset;
        for (let i = 0; i < preAllocate; i++) {
            this.pool.push(factory());
        }
    }

    acquire(): T {
        return this.pool.pop() ?? this.factory();
    }

    release(item: T): void {
        this.reset(item);
        this.pool.push(item);
    }
}

// 使用示例：减少游戏中的临时向量分配
interface Vector3 { x: number; y: number; z: number; }

const vecPool = new ObjectPool<Vector3>(
    () => ({ x: 0, y: 0, z: 0 }),
    (v) => { v.x = 0; v.y = 0; v.z = 0; },
    100,  // 预分配
);

const temp = vecPool.acquire();
temp.x = 10;
// ... 使用 ...
vecPool.release(temp);  // 归还池中，避免被 GC

// ========== 6. 内存监控（Node.js） ==========
function monitorMemory(): void {
    const memoryUsage = process.memoryUsage();
    console.log({
        rss: `${(memoryUsage.rss / 1024 / 1024).toFixed(2)} MB`,       // 常驻集
        heapTotal: `${(memoryUsage.heapTotal / 1024 / 1024).toFixed(2)} MB`, // V8 堆总量
        heapUsed: `${(memoryUsage.heapUsed / 1024 / 1024).toFixed(2)} MB`,   // 已用堆
        external: `${(memoryUsage.external / 1024 / 1024).toFixed(2)} MB`,   // C++ 绑定内存
        arrayBuffers: `${(memoryUsage.arrayBuffers / 1024 / 1024).toFixed(2)} MB`,
    });
}

// 大数组对内存的影响
function demonstrateMemoryImpact(): void {
    // V8 对数字数组的优化（SMI / Double / Boxed）
    const smallInts = new Array(1000).fill(42);  // SMI 数组，高效
    const doubles = new Array(1000).fill(3.14);   // Double 数组
    
    // 稀疏数组 vs 密集数组
    const dense = [1, 2, 3, 4, 5];       // 密集，快速
    const sparse = [1, , 3, , 5];         // 稀疏，慢速
}
```

## 推荐书籍

1. **《JavaScript: The Good Parts》** — Douglas Crockford
   虽是 JS 书，但对理解 TS 运行时的内存语义至关重要，包括函数作用域、闭包和对象引用。

2. **《Node.js Design Patterns》（第3版）** — Mario Casciaro / Luciano Mammino
   第 4 章讲内存管理，第 11 章讲高性能模式，包含对象池、WeakMap 等 TS 相关的高级内存优化。

3. **《High Performance Browser Networking》** — Ilya Grigorik
   虽然主要讲网络，但第 5 章解释了浏览器和 V8 的内存管理，对 Web 和 Node.js 都适用。

4. **《Understanding V8's Bytecode》**（在线）— Franziska Hinkelmann
   深入 V8 内部，理解数组的 SMI/Double/Boxed 优化和 hidden class，是 TS 性能调优的底层知识。
