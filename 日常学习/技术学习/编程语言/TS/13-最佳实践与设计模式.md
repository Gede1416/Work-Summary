# TypeScript 最佳实践与设计模式

## 详细描述

TypeScript 的最佳实践核心是**类型导向开发**——利用类型系统从源头消除错误，让类型成为开发过程中的文档和约束。

**strict 模式是基线**：`strict: true` 启用所有严格检查——没有它，TypeScript 的价值大打折扣。`strictNullChecks`（null/undefined 独立于其他类型）、`noImplicitAny`（禁止隐式 any）、`strictFunctionTypes`（函数参数类型逆变检查）。

**避免 any**是 TS 开发的第一原则。`any` 是类型系统的毒药——一个 `any` 会感染整个调用链。ESLint `@typescript-eslint/no-explicit-any` 规则应设为 error。用 `unknown` 替代 `any`，调用者必须进行类型守卫。

**readonly 和 as const** 增强不可变性。`readonly` 修饰符和 `Readonly<T>` 映射类型防止意外修改；`as const` 将值收窄为精确的字面量类型，创建深度不可变的数据。

**联合类型领域建模**（Discriminated Union Pattern）是 TS 最强但被低估的设计模式。用 `{ kind: "circle"; radius: number } | { kind: "rectangle"; width: number; height: number }` 建模变体数据，编译器在 switch 的每个分支中自动收窄类型。

**依赖注入模式**在 TS 中有多种实现：基于接口的手动 DI（类似 C#）、装饰器 DI（NestJS 风格）、或函数式 DI（传入具体实现函数而非类）。TS 的 interface 只在编译时存在，DI 需要以具体 class 或 token 为键。

**优先使用 type 推断**：TS 编译器已经足够聪明——不要为每个变量显式标注类型。让类型流动起来，只在函数边界的参数和返回值上显式标注。

## 代码实例

```typescript
// ========== Discriminated Union 领域建模 ==========
type ApiResult<T> =
    | { status: "loading" }
    | { status: "success"; data: T }
    | { status: "error"; error: Error };

function renderResult<T>(result: ApiResult<T>): string {
    switch (result.status) {
        case "loading":
            return "Loading...";
        case "success":
            return `Data: ${JSON.stringify(result.data)}`;  // result 类型收窄
        case "error":
            return `Error: ${result.error.message}`;
    }
}

// ========== readonly + as const ==========
// 配置文件使用 as const 实现深度不可变
const APP_CONFIG = {
    api: { baseUrl: "https://api.example.com", timeout: 5000 },
    features: { darkMode: true, analytics: false },
} as const;

type Config = typeof APP_CONFIG;  // 推断为最精确的字面量类型
// APP_CONFIG.api.baseUrl = "other"; // 错误！只读

// ========== 函数式 Builder 模式 ==========
class QueryBuilder<T> {
    private conditions: string[] = [];
    
    where<K extends keyof T>(field: K, value: T[K]): this {
        this.conditions.push(`${String(field)} = ${JSON.stringify(value)}`);
        return this;
    }
    
    build(): string {
        return `SELECT * WHERE ${this.conditions.join(" AND ") || "1=1"}`;
    }
}

interface User { id: number; name: string; age: number; }
const query = new QueryBuilder<User>()
    .where("name", "Alice")  // 自动补全和类型检查
    .where("age", 30)
    .build();

// ========== 依赖注入（函数式风格） ==========
interface Logger { log: (msg: string) => void; }

const makeUserService = (logger: Logger) => ({
    createUser: (name: string) => {
        logger.log(`Creating user: ${name}`);
        return { id: Math.random(), name };
    },
});

const consoleLogger: Logger = { log: console.log };
const userService = makeUserService(consoleLogger);
```

## 推荐书籍

1. **《Effective TypeScript》** — Dan Vanderkam
   62 个条款涵盖 TS 的所有最佳实践：类型推导、any 的危害、严格模式、readonly、泛型约束。
2. **《Programming TypeScript》** — Boris Cherny
   第 10 章详细讨论了设计模式在 TS 中的实现：Factory、Builder、Adapter、Strategy 的类型安全版本。
3. **《Clean Architecture with TypeScript》** — 多本在线资源
   将 Clean Architecture 模式适配到 TS/Node.js 的实战指导。
