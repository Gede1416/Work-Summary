# TypeScript 包管理与构建工具

## 详细描述

TypeScript 的包管理和构建生态系统基于 JavaScript 生态的演化，拥有全球最大的包注册表和最活跃的工具链。

**npm / yarn / pnpm** 是三大主流包管理器：
- **npm**：Node.js 默认包管理器。`package.json` 声明依赖，`node_modules` 存放包，`package-lock.json` 锁定版本。workspaces 支持 monorepo。
- **yarn**：Facebook 出品，经典版（1.x）和 Berry（3.x+）差异大，`yarn.lock` 确定性锁定，Plug'n'Play（PnP）消除 `node_modules`。
- **pnpm**：基于内容寻址存储（Content-Addressable Storage），全局缓存节省磁盘空间，严格的 node_modules 结构防止幽灵依赖（phantom dependencies）。

**package.json** 是项目的核心配置文件。`dependencies`（生产依赖）、`devDependencies`（开发依赖）、`peerDependencies`（宿主依赖）、`scripts`（npm scripts）。`"type": "module"` 启用 ES 模块模式。

**tsconfig.json** 告诉 TypeScript 编译器如何处理项目。关键选项：
- `compilerOptions.target`：输出 JS 版本（`ES2022`）
- `compilerOptions.module`：模块系统（`ESNext`、`NodeNext`、`Preserve`）
- `compilerOptions.moduleResolution`：模块查找策略（`bundler`、`node`）
- `compilerOptions.strict`：启用所有严格检查（强烈建议）
- `compilerOptions.paths` 和 `baseUrl`：路径别名
- `include`/`exclude`：控制编译范围

**构建工具**：
- **tsc**：官方 TypeScript 编译器，最基础，只做类型检查和转译
- **esbuild**：Go 语言编写，极快（10-100× tsc），支持 TypeScript 和 JSX，用于打包和转译
- **swc**：Rust 语言编写，速度极快，专注于转译，常与 tsc 配合（swc 转译 + tsc 类型检查）
- **webpack**：最成熟的打包器，插件生态丰富，适合复杂项目
- **Vite**：基于 esbuild 和 Rollup，开发体验极好（即时的 HMR），已成为 Vue/React 项目首选
- **tsup**：基于 esbuild，零配置的 TS 库打包工具
- **tsx**：直接执行 TypeScript 文件，免去编译步骤，适合脚本和开发

## 代码实例

```json
// ========== package.json ==========
{
  "name": "my-typescript-app",
  "version": "1.0.0",
  "type": "module",
  "scripts": {
    "dev": "tsx watch src/index.ts",
    "build": "tsc --noEmit && esbuild src/index.ts --bundle --platform=node --outdir=dist",
    "start": "node dist/index.js",
    "test": "vitest",
    "lint": "eslint src/ && tsc --noEmit",
    "format": "prettier --write 'src/**/*.ts'"
  },
  "dependencies": {
    "express": "^4.18.0",
    "zod": "^3.23.0"
  },
  "devDependencies": {
    "typescript": "^5.4.0",
    "@types/express": "^4.17.0",
    "esbuild": "^0.21.0",
    "tsx": "^4.7.0",
    "vitest": "^1.6.0",
    "prettier": "^3.2.0",
    "eslint": "^8.57.0",
    "@typescript-eslint/parser": "^7.0.0"
  }
}
```

```json
// ========== tsconfig.json（现代化配置） ==========
{
  "compilerOptions": {
    /* 语言和环境 */
    "target": "ES2022",
    "lib": ["ES2022"],
    "module": "NodeNext",
    "moduleResolution": "NodeNext",

    /* 互操作性 */
    "esModuleInterop": true,
    "forceConsistentCasingInFileNames": true,
    "isolatedModules": true,
    "verbatimModuleSyntax": true,

    /* 类型检查 */
    "strict": true,
    "noUncheckedIndexedAccess": true,
    "noImplicitOverride": true,
    "exactOptionalPropertyTypes": false,

    /* 输出 */
    "outDir": "./dist",
    "rootDir": "./src",
    "declaration": true,
    "declarationMap": true,
    "sourceMap": true,

    /* 路径别名 */
    "baseUrl": ".",
    "paths": {
      "@/*": ["./src/*"],
      "@lib/*": ["./src/lib/*"],
      "@types/*": ["./src/types/*"]
    }
  },
  "include": ["src/**/*.ts"],
  "exclude": ["node_modules", "dist", "**/*.test.ts"]
}
```

```bash
# ========== 常用 CLI 命令 ==========

# 包管理
npm init -y
npm install typescript --save-dev
npm install express zod
npm install @types/express --save-dev

# pnpm 等价命令（推荐：更快、更省空间）
pnpm add express zod
pnpm add -D typescript @types/express

# TypeScript 编译
tsc                      # 编译整个项目
tsc --noEmit            # 只类型检查，不生成 JS
tsc --watch             # 监视模式
tsc --showConfig        # 显示完整的编译配置
tsc --version

# 执行 TypeScript
npx tsx src/index.ts    # 直接运行（tsx）
tsx watch src/index.ts  # 监视+运行

# 构建
esbuild src/index.ts --bundle --platform=node --outdir=dist
esbuild src/index.ts --bundle --minify --platform=node --outfile=dist/bundle.js

# 测试
npx vitest              # 运行测试
npx vitest --coverage   # 运行测试 + 覆盖率

# 代码质量
npx eslint src/
npx prettier --check src/**/*.ts
npx prettier --write src/**/*.ts
```

## 推荐书籍

1. **《Programming TypeScript》** — Boris Cherny
   附录 B 详细讲解了 tsconfig.json 的所有选项和项目配置。

2. **《esbuild》官方文档**（esbuild.github.io）
   学习 esbuild 的插件系统、内容类型（Content Types）和构建 API。

3. **《SurviveJS: Webpack》（在线免费）** — Juho Vepsalainen
   虽然 TypeScript 生态现在更倾向 esbuild/Vite，但对理解打包器原理仍是最好的参考资料。
