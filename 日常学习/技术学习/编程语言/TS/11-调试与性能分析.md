# TypeScript 调试与性能分析

## 详细描述

TypeScript 的调试和性能分析依托于 Node.js 的调试基础设施和 Chrome DevTools 生态，加上 TS 特有的类型性能分析。

**Node.js 调试器**基于 V8 Inspector 协议。`node --inspect index.js` 启动调试模式，在 `chrome://inspect` 中连接 DevTools。`node --inspect-brk index.js` 在第一行自动暂停（适合调试启动阶段的代码）。VS Code 内建的 Node.js 调试器提供与 Chrome DevTools 相同的能力——断点、监视、调用栈、日志点。

**Chrome DevTools** 是最全面的 JavaScript/TypeScript 调试环境。Sources 面板支持断点、条件断点、XHR 断点、事件监听器断点、DOM 断点。Performance 面板录制运行时性能，生成火焰图和时间线。Memory 面板用于检测内存泄漏——堆快照（Heap Snapshot）、分配时间线（Allocation Timeline）、分配采样（Allocation Sampling）。

**clinic** 是 Node.js 专用的性能诊断工具套件：
- **clinic doctor**：综合健康检查，对 CPU、内存、事件循环延迟、GC 进行分析
- **clinic flame**：生成 CPU 火焰图
- **clinic bubbleprof**：分析异步操作的延迟蝴蝶图

**0x** 是一个简单但强大的火焰图生成工具：`npx 0x my-script.js` 获取分析结果后在浏览器中以交互式火焰图形式查看。

**类型性能分析**（TS 特有）：TS 编译器本身可能成为性能瓶颈。`tsc --generateTrace traces` 生成编译过程跟踪，用 Chrome DevTools 的 Performance 面板加载分析 TypeScript 编译器性能，查看哪些文件/类型运算占据了编译时间。`--noEmit` 配合 `--incremental` 可加速类型检查。

**console 高级方法**：`console.time("label")`/`console.timeEnd("label")` 测量执行时间；`console.memory`（Chrome）查看内存统计；`console.trace()` 打印调用栈而不中断执行；结构化日志用 `console.table()`。

## 代码实例

```typescript
// ========== 1. 性能测量辅助函数 ==========

// 高精度计时器（Node.js）
const { performance } = require("perf_hooks");

function benchmark(name: string, fn: () => void, iterations: number = 1000): void {
    // 预热（让 JIT 优化）
    for (let i = 0; i < 100; i++) fn();
    
    const start = performance.now();
    for (let i = 0; i < iterations; i++) fn();
    const end = performance.now();
    
    const avgMs = (end - start) / iterations;
    console.log(`[${name}] avg: ${avgMs.toFixed(4)}ms (${iterations} iterations)`);
}

// 使用示例
const arr = Array.from({ length: 10000 }, (_, i) => i);

benchmark("for loop", () => {
    let sum = 0;
    for (let i = 0; i < arr.length; i++) sum += arr[i];
});

benchmark("for-of", () => {
    let sum = 0;
    for (const n of arr) sum += n;
});

benchmark("reduce", () => {
    arr.reduce((a, b) => a + b, 0);
});

// ========== 2. 内存泄漏检测模式 ==========

// 场景：EventEmitter 遗漏清理导致内存泄漏
import { EventEmitter } from "events";

class LeakyComponent {
    private emitter = new EventEmitter();
    
    constructor() {
        // ❌ 潜在泄漏：listener 引用了 this，GC 无法回收
        this.emitter.on("data", this.handleData);
    }
    
    handleData = (data: unknown): void => {
        console.log(data);
    };
    
    // ✅ 正确：提供清理方法
    destroy(): void {
        this.emitter.removeAllListeners();
        this.emitter.off("data", this.handleData);
    }
}
```

```bash
# ========== Node.js 调试命令 ==========

# 启动调试（等待 DevTools 连接）
node --inspect-brk src/index.ts

# 使用 tsx 调试 TypeScript
node --inspect-brk --import tsx src/index.ts

# VS Code launch.json:
# {
#   "type": "node",
#   "request": "launch",
#   "name": "Debug TS",
#   "runtimeExecutable": "npx",
#   "runtimeArgs": ["tsx", "src/index.ts"],
#   "console": "integratedTerminal"
# }

# ========== 火焰图工具 ==========
# 0x：一键生成火焰图
npx 0x src/index.js
# 输出到浏览器 -> 交互式火焰图

# clinic doctor：综合诊断
npx clinic doctor -- node src/index.js

# ========== TS 编译性能分析 ==========
# 生成编译 trace
tsc --generateTrace traces --incremental
# 在 chrome://tracing 中加载 trace 文件分析

# 查看哪些文件编译最慢
tsc --listFiles --noEmit 2>&1 | head -20
```

## 推荐书籍

1. **《Node.js Design Patterns》（第3版）** — Mario Casciaro / Luciano Mammino
   第 6 章（Performance）和第 4 章（Memory Management）有深入的 Node.js 性能分析和优化。

2. **Chrome DevTools 官方文档**（developer.chrome.com/docs/devtools）
   Performance、Memory 面板的完整教程和最佳实践。

3. **《Systems Performance》（第2版）** — Brendan Gregg
   虽然偏系统层面，但方法论（USE 方法、工作负载分析）适用于任何性能调优场景。
