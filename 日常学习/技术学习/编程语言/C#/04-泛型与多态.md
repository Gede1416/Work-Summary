# C# 泛型与多态

## 详细描述

C# 的泛型系统在运行时保留类型信息（具象化泛型，Reified Generics），这与 Java 的类型擦除形成鲜明对比。配合运行时多态（虚方法），构成了 C# 代码重用和抽象的基石。

**泛型类/方法/接口**：泛型参数用尖括号 `<T>` 声明，可以出现在类、结构体、接口、方法、委托中。C# 的泛型由 CLR 原生支持——值类型泛型参数会为每种类型生成独立的特化代码（性能优化），引用类型泛型参数共享同一份 JIT 编译代码。

**泛型约束**限制了泛型参数的类型范围：`where T : class`（引用类型）、`where T : struct`（值类型）、`where T : new()`（有无参构造函数）、`where T : BaseClass`（要求继承自特定类）、`where T : ISomeInterface`（要求实现接口）、`where T : notnull`（非空，C# 8）。C# 11 引入了 `where T : INumber<T>` 等数学泛型接口。

**协变与逆变**（Variance）通过 `out`（协变）和 `in`（逆变）修饰泛型接口和委托的类型参数。协变允许 `IEnumerable<Dog>` 赋值给 `IEnumerable<Animal>`（类型参数只出现在输出位置）；逆变允许 `Action<Animal>` 赋值给 `Action<Dog>`（类型参数只出现在输入位置）。理解协变/逆变的关键是"输入"和"输出"的区分。

**运行时多态**通过 `virtual` / `override` 机制实现。基类用 `virtual` 标记可被重写的方法，派生类用 `override` 提供具体实现。`new` 关键字隐藏而非重写基类成员。`sealed override` 阻止进一步重写。C# 的虚方法分派通过方法表（Method Table）在运行时实现。

**泛型 + 接口 + 多态**的组合是 .NET 中最强大的抽象模式。例如 `IRepository<T>` 模式：定义泛型仓储接口，通过协变允许 `IRepository<Customer, Guid>` 的使用，结合依赖注入实现松耦合。

## 代码实例

```csharp
// ========== 1. 泛型类与方法 ==========

// 泛型类
public class Repository<T> where T : class, IIdentifiable, new()
{
    private readonly List<T> _items = new();
    
    public void Add(T item) => _items.Add(item);
    
    public T? GetById(Guid id) =>
        _items.FirstOrDefault(x => x.Id == id);
    
    public IReadOnlyList<T> GetAll() => _items.AsReadOnly();
    
    // 泛型方法 + 额外约束
    public TResult? SelectFirst<TResult>(Func<T, TResult> selector)
        where TResult : class
    {
        return _items.Count > 0 ? selector(_items[0]) : null;
    }
}

// ========== 2. 泛型约束详解 ==========

public interface IEntity { Guid Id { get; } }

// 多种约束组合
public class GenericProcessor<T> where T : class, IEntity, IComparable<T>, new()
{
    public T CreateDefault()
    {
        var item = new T();  // 需要 new() 约束
        return item;
    }
    
    public int Compare(T a, T b) => a.CompareTo(b);  // 需要 IComparable<T>
}

// C# 11 数学泛型
// public static T Add<T>(T a, T b) where T : INumber<T> => a + b;

// ========== 3. 协变（out）与逆变（in） ==========

// 协变 out：输出位置（只读）
public interface IReadOnlyRepository<out T>
{
    T? GetById(Guid id);
    IEnumerable<T> GetAll();
    // 注意：不能有 void Add(T item) 因为 T 出现在输入位置
}

// 逆变 in：输入位置（只写）
public interface IWriteOnlyRepository<in T>
{
    void Add(T item);
    void Remove(T item);
    // 注意：不能有 T GetById(Guid id) 因为 T 出现在输出位置
}

// 不变（默认，可读可写）
public interface IRepository<T> : IReadOnlyRepository<T>, IWriteOnlyRepository<T>
{
    void Update(T item);
}

public class Animal { public string Name { get; set; } = ""; }
public class Dog : Animal { public string Breed { get; set; } = ""; }

public static class VarianceDemo
{
    public static void Demonstrate()
    {
        IEnumerable<Dog> dogs = new List<Dog> { new Dog { Name = "Rex" } };
        
        // 协变：Dog 的集合可以赋值给 Animal 的集合
        IEnumerable<Animal> animals = dogs; // OK：IEnumerable<T> 是协变的
        
        List<Dog> dogList = new List<Dog>();
        // List<Animal> animalList = dogList; // 错误：List<T> 不是协变的
        
        // 逆变：Animal 的处理函数可以接受 Dog
        Action<Animal> animalAction = a => Console.WriteLine(a.Name);
        Action<Dog> dogAction = animalAction; // OK：Action<T> 是逆变的
    }
}

// ========== 4. 运行时多态 ==========

public abstract class Shape
{
    public abstract double Area();  // 必须被重写
    
    public virtual string Description() => $"Shape with area {Area():F2}";  // 可选重写
    
    // 非虚方法（不能被重写）
    public static Shape CreateCircle(double r) => new Circle(r);
}

public class Circle : Shape
{
    private readonly double _radius;
    
    public Circle(double radius) => _radius = radius;
    
    public override double Area() => Math.PI * _radius * _radius;
    
    public override string Description() =>
        $"Circle(r={_radius}) with area {Area():F2}";
    
    public new static Circle CreateCircle(double r) => new Circle(r);  // 隐藏基类方法
}

public class ColoredCircle : Circle
{
    public string Color { get; }
    
    public ColoredCircle(double radius, string color) : base(radius)
    {
        Color = color;
    }
    
    // sealed override：不再允许子类继续重写
    public sealed override string Description() =>
        $"{Color} {base.Description()}";
}

// ========== 5. 泛型委托与函数式多态 ==========

// 泛型委托
public delegate TResult Transformer<T, TResult>(T input) where T : notnull;

public static class FunctionalPolymorphism
{
    // 高阶函数：接受泛型委托
    public static IEnumerable<TResult> Transform<T, TResult>(
        this IEnumerable<T> source,
        Transformer<T, TResult> transformer)
        where T : notnull
    {
        foreach (var item in source)
            yield return transformer(item);
    }
    
    // 约束泛型高阶函数
    public static T FindMax<T>(IEnumerable<T> source) where T : IComparable<T>
    {
        using var enumerator = source.GetEnumerator();
        if (!enumerator.MoveNext())
            throw new InvalidOperationException("Empty sequence");
        
        T max = enumerator.Current;
        while (enumerator.MoveNext())
        {
            if (enumerator.Current.CompareTo(max) > 0)
                max = enumerator.Current;
        }
        return max;
    }
    
    public static void Demonstrate()
    {
        var numbers = new[] { 3, 1, 4, 1, 5, 9 };
        var strings = new[] { "apple", "banana", "cherry" };
        
        Console.WriteLine(FindMax(numbers));  // 9
        Console.WriteLine(FindMax(strings));  // cherry
    }
}
```

## 推荐书籍

1. **《C# 12 in a Nutshell》** — Joseph Albahari
   第 4 章（Generics）和第 2 章（Inheritance）提供了完整的泛型、协变逆变、运行时多态的详尽参考。

2. **《C# in Depth》（第4版）** — Jon Skeet
   深入讲解泛型、协变逆变在 C# 中引入的历史、设计决策和实际使用陷阱。

3. **《Functional Programming in C#》（第2版）** — Enrico Buonanno
   展示泛型约束和委托如何构建类型安全的函数式抽象，包括泛型函子和单子。

4. **《CLR via C#》（第4版）** — Jeffrey Richter
   第 12 章（Generics）从 CLR 层面讲解泛型的内部实现——值类型特化、引用类型共享代码，以及协变逆变的 IL 实现。
