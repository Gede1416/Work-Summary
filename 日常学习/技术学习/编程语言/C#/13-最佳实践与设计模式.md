# C# 最佳实践与设计模式

## 详细描述

C# 的最佳实践围绕 **SOLID 原则**、**依赖注入**、**异步习惯用法**和**性能意识**展开。

**SOLID 原则**在 C# 中通过接口和 DI 容器自然实现：`IUserRepository` 接口用于隔离数据访问，`IEmailService` 接口用于隔离外部服务。单一职责原则（SRP）指导我们拆分类；依赖倒置原则（DIP）要求依赖抽象而非具体实现。

**依赖注入**（DI）是 .NET 生态的基石。ASP.NET Core 内建 DI 容器，通过构造函数注入服务。`services.AddScoped<IOrderService, OrderService>()` 注册服务生命周期（Singleton、Scoped、Transient）。Microsoft.Extensions.DependencyInjection 是整个 .NET 生态的 DI 抽象层。

**异步习惯用法**：始终使用 `async/await` 而非 `.Wait()` 或 `.Result`（避免死锁）；库代码使用 `ConfigureAwait(false)`；`async void` 仅用于事件处理器；`CancellationToken` 参数传播到所有异步调用。

**可空引用类型**（NRT，C# 8+）是类型安全的重大改进。启用 `<Nullable>enable</Nullable>` 后，引用类型默认不可为 null，需要在变量后加 `?` 明确可为 null。极大减少了 NullReferenceException。

**性能最佳实践**：使用 `Span<T>`/`Memory<T>` 避免分配；`ArrayPool<T>` 共享缓冲区；`stackalloc` 栈分配；`StringBuilder` 大量拼接；避免装箱（boxing）；结构体保持小而不可变。

## 代码实例

```csharp
// ========== 依赖注入模式 ==========
public interface IOrderRepository
{
    Task<Order?> GetByIdAsync(int id);
    Task SaveAsync(Order order);
}

public class OrderService : IOrderService
{
    private readonly IOrderRepository _repo;
    private readonly ILogger<OrderService> _logger;
    
    public OrderService(IOrderRepository repo, ILogger<OrderService> logger)
    {
        _repo = repo;
        _logger = logger;
    }
    
    public async Task<Result<Order>> ProcessOrderAsync(int orderId, CancellationToken ct)
    {
        var order = await _repo.GetByIdAsync(orderId);
        if (order is null) return Result<Order>.Failure("Order not found");
        
        order.Process();
        await _repo.SaveAsync(order);
        _logger.LogInformation("Order {Id} processed", order.Id);
        return Result<Order>.Success(order);
    }
}

// ========== Builder 模式 (Fluent API) ==========
public class HttpClientBuilder
{
    private string _baseUrl = "";
    private int _timeoutSeconds = 30;
    private int _retries = 0;
    
    public HttpClientBuilder WithBaseUrl(string url) { _baseUrl = url; return this; }
    public HttpClientBuilder WithTimeout(int seconds) { _timeoutSeconds = seconds; return this; }
    public HttpClientBuilder WithRetries(int count) { _retries = count; return this; }
    
    public HttpClient Build() => new() { BaseAddress = new(_baseUrl), Timeout = TimeSpan.FromSeconds(_timeoutSeconds) };
}

// ========== 策略模式 ==========
public interface IPricingStrategy { decimal CalculatePrice(Order order); }
public class RegularPricing : IPricingStrategy {
    public decimal CalculatePrice(Order order) => order.Items.Sum(i => i.Price);
}
public class DiscountPricing : IPricingStrategy {
    public decimal CalculatePrice(Order order) => order.Items.Sum(i => i.Price) * 0.9m;
}
```

## 推荐书籍

1. **《Adaptive Code via C#》** — Gary McLean Hall
   SOLID 原则和 DI 在 C# 中的实践圣经，包含大量重构示例和设计模式演化。
2. **《Dependency Injection in .NET》** — Mark Seemann
   DI 理论的权威著作，从 DI 模式到容器实现到具体场景的 DI 设计。
3. **《C# 12 in a Nutshell》** — Joseph Albahari
   包含最佳实践相关的章节：异步模式、可空引用类型、性能与 Span。
