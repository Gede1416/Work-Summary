# C# 错误处理

## 详细描述

C# 的错误处理核心是**异常机制**，辅以现代的**结果模式**。理解何时抛异常、何时返回错误，是编写可靠 .NET 应用的基础。

**try-catch-finally** 是异常处理的基本结构。`try` 块包含可能出错的代码，`catch` 块处理特定类型的异常（从最具体到最通用），`finally` 块无论是否抛异常都会执行——用于释放资源。C# 6.0+ 支持异常筛选器（`catch when`），可以在 catch 之前添加条件判断。

**异常类层次**：所有异常最终继承自 `System.Exception`。`SystemException` 是系统异常的基类，`ApplicationException` 曾是自定义异常的基类（现已不推荐）。常见异常：`ArgumentException`、`InvalidOperationException`、`NullReferenceException`、`NotSupportedException`。自定义异常应继承 `Exception` 并实现序列化构造函数。

**Exception 筛选器**（C# 6.0）：`catch (HttpRequestException ex) when (ex.StatusCode == 503)` — 可以在不影响原始堆栈跟踪的情况下决定是否捕获异常。与在 catch 块中 rethrow 不同，筛选器返回 false 时原始堆栈被保留。

**结果模式**是函数式编程对异常机制的补充。通过返回封装了成功/失败的 `Result<T>` 或 `OneOf<T, Error>` 类型，将错误作为返回值的一部分，强制调用者显式处理。这对于明确可控的失败场景（如验证、业务规则）特别合适。

**ArgumentException.ThrowIfNull**（.NET 6）和 **ArgumentOutOfRangeException.ThrowIf**（.NET 7）等静态辅助方法让空值检查和参数验证更简洁。

**全局异常处理**：`AppDomain.CurrentDomain.UnhandledException`（所有未捕获异常的最后屏障）、`TaskScheduler.UnobservedTaskException`（被遗忘的异步任务异常）。在 ASP.NET Core 中通过中间件（Middleware）统一处理 HTTP 管道中的异常。

## 代码实例

```csharp
// ========== 1. 自定义异常 ==========

public class DomainException : Exception
{
    public string ErrorCode { get; }
    
    public DomainException(string message, string errorCode)
        : base(message)
    {
        ErrorCode = errorCode;
    }
    
    public DomainException(string message, string errorCode, Exception inner)
        : base(message, inner)
    {
        ErrorCode = errorCode;
    }
}

public class InsufficientFundsException : DomainException
{
    public decimal Balance { get; }
    public decimal Requested { get; }
    
    public InsufficientFundsException(decimal balance, decimal requested)
        : base($"余额不足: 需要 {requested:C}, 可用 {balance:C}", "INSUFFICIENT_FUNDS")
    {
        Balance = balance;
        Requested = requested;
    }
}

// ========== 2. try-catch-finally 与筛选器 ==========

public class OrderService
{
    private readonly HttpClient _httpClient;
    private readonly ILogger<OrderService> _logger;
    
    public OrderService(HttpClient httpClient, ILogger<OrderService> logger)
    {
        _httpClient = httpClient;
        _logger = logger;
    }
    
    public async Task<Order?> FetchOrderAsync(int orderId)
    {
        try
        {
            var response = await _httpClient.GetAsync($"/orders/{orderId}");
            response.EnsureSuccessStatusCode();
            return await response.Content.ReadFromJsonAsync<Order>();
        }
        catch (HttpRequestException ex) when (ex.StatusCode == System.Net.HttpStatusCode.NotFound)
        {
            _logger.LogWarning(ex, "Order {OrderId} not found", orderId);
            return null;
        }
        catch (HttpRequestException ex) when (ex.StatusCode >= System.Net.HttpStatusCode.InternalServerError)
        {
            _logger.LogError(ex, "Server error fetching order {OrderId}", orderId);
            throw;  // 重新抛出，保留堆栈
        }
        catch (TaskCanceledException)
        {
            _logger.LogInformation("Fetch for order {OrderId} was cancelled", orderId);
            return null;
        }
        finally
        {
            _logger.LogDebug("Fetch order {OrderId} operation completed", orderId);
        }
    }
}

// ========== 3. 结果模式 ==========

public readonly record struct Result<T>
{
    public T? Value { get; }
    public string? Error { get; }
    public bool IsSuccess { get; }
    
    private Result(T value)
    {
        Value = value;
        IsSuccess = true;
        Error = null;
    }
    
    private Result(string error)
    {
        Value = default;
        IsSuccess = false;
        Error = error;
    }
    
    public static Result<T> Success(T value) => new(value);
    public static Result<T> Failure(string error) => new(error);
    
    public TResult Match<TResult>(Func<T, TResult> success, Func<string, TResult> failure)
        => IsSuccess ? success(Value!) : failure(Error!);
    
    public Result<TResult> Map<TResult>(Func<T, TResult> mapper)
        => IsSuccess
            ? Result<TResult>.Success(mapper(Value!))
            : Result<TResult>.Failure(Error!);
}

public static class BankService
{
    public static Result<decimal> Withdraw(decimal balance, decimal amount)
    {
        if (amount <= 0)
            return Result<decimal>.Failure("提款金额必须大于零");
        
        if (amount > balance)
            return Result<decimal>.Failure($"余额不足: {balance:C}");
        
        return Result<decimal>.Success(balance - amount);
    }
    
    public static void Demonstrate()
    {
        var result = Withdraw(100, 50)
            .Map(newBalance => newBalance * 0.9m);  // 转换成功值
        
        var message = result.Match(
            success: balance => $"新余额: {balance:C}",
            failure: error => $"操作失败: {error}"
        );
        
        Console.WriteLine(message);
    }
}

// ========== 4. ASP.NET Core 全局异常中间件 ==========

public class GlobalExceptionMiddleware
{
    private readonly RequestDelegate _next;
    private readonly ILogger<GlobalExceptionMiddleware> _logger;
    
    public GlobalExceptionMiddleware(RequestDelegate next, ILogger<GlobalExceptionMiddleware> logger)
    {
        _next = next;
        _logger = logger;
    }
    
    public async Task InvokeAsync(HttpContext context)
    {
        try
        {
            await _next(context);
        }
        catch (DomainException ex)
        {
            _logger.LogWarning(ex, "Domain exception: {ErrorCode}", ex.ErrorCode);
            context.Response.StatusCode = 400;
            await context.Response.WriteAsJsonAsync(new
            {
                error = ex.ErrorCode,
                message = ex.Message,
            });
        }
        catch (Exception ex)
        {
            _logger.LogError(ex, "Unhandled exception");
            context.Response.StatusCode = 500;
            await context.Response.WriteAsJsonAsync(new
            {
                error = "INTERNAL_ERROR",
                message = "An unexpected error occurred.",
            });
        }
    }
}
```

## 推荐书籍

1. **《C# 12 in a Nutshell》** — Joseph Albahari
   第 11 章（Exception Handling）提供了异常筛选器、自定义异常、throw 语义的完整参考。

2. **《CLR via C#》（第4版）** — Jeffrey Richter
   第 20 章从 CLR 层面深入 SEH（结构化异常处理）和 C# 异常机制的底层实现。

3. **《Functional Programming in C#》（第2版）** — Enrico Buonanno
   第 3 章详细介绍结果模式、Either 模式在 C# 中的实现，以及何时用异常、何时用结果的决策指南。

4. **《Dependency Injection in .NET》** — Mark Seemann
   包含 ASP.NET Core 中错误处理和异常日志的最佳实践，特别是全局异常处理与 DI 的集成。
