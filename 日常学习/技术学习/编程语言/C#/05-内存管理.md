# C# 内存管理

## 详细描述

C# 的内存管理由 CLR 的垃圾回收器（GC）自动处理，但这不意味着开发者可以完全忽略内存问题。理解 GC 的工作原理和托管堆的行为，对于编写高性能 .NET 应用至关重要。

**分代垃圾回收**（Generational GC）是 .NET GC 的核心机制。托管堆分为三代：
- **Gen 0**（第 0 代）：新分配的对象，回收最频繁，速度最快
- **Gen 1**（第 1 代）：从 Gen 0 存活下来的对象，作为 Gen 0 和 Gen 2 之间的缓冲区
- **Gen 2**（第 2 代）：长期存活的对象，如静态数据、缓存等，回收成本最高

对象在前几代中被快速回收（大多数对象是短暂存活的），只有少量长期对象最终到达 Gen 2。大对象（≥85KB）直接进入大对象堆（LOH），不参与分代回收。

**IDisposable 与 using**：不是所有资源都是托管内存——文件句柄、网络连接、数据库连接、原生内存等非托管资源需要在确定的时间点释放。`IDisposable` 接口和 `Dispose` 模式提供了确定性的资源清理。`using` 语句（C# 8.0 简化为 using 声明）保证在作用域结束时调用 `Dispose`，即使发生异常。这是 .NET 版的 RAII。

**Span\<T\> 与 Memory\<T\>** 是 .NET 近期最重要的性能创新。它们提供了对连续内存的类型安全、免分配视图。`Span<T>` 是 ref struct，只能在栈上存在，提供对数组段、栈分配内存、非托管内存的零开销访问。`Memory<T>` 可以存储在堆上（用于异步场景）。通过它们，字符串操作、JSON 解析、网络处理都可以做到免分配或低分配。

**stackalloc**：在栈上分配一块内存（需要 unsafe 或在 C# 7.2+ 中使用 `Span<T>` 安全访问）。栈分配没有 GC 开销，适合临时小缓冲区。

**强引用与弱引用**：`WeakReference<T>` 允许引用一个对象而不阻止它被 GC 回收。适用于缓存场景——如果对象被其他地方引用则可以获取，否则接受 null 并重新创建。

**终结器**（Finalizer，`~ClassName()`）是最后的安全网，当 GC 回收一个没有调用 `Dispose` 的对象时运行。但不应该依赖终结器来清理资源——无法控制执行时机，且带终结器的对象需要两次 GC 才能完全回收。

## 代码实例

// ========== 1. IDisposable 模式 ==========
public class ResourceCache : IDisposable
{
    private readonly Dictionary<string, byte[]> _cache = new();
    private readonly FileStream? _logStream;
    private bool _disposed;

    public ResourceCache(string logPath)
    {
        _logStream = new FileStream(logPath, FileMode.Append, FileAccess.Write);
    }

    public void Store(string key, byte[] data)
    {
        ObjectDisposedException.ThrowIf(_disposed, this);
        _cache[key] = data;
        Log($"Stored {key}: {data.Length} bytes");
    }

    private void Log(string message)
    {
        var bytes = System.Text.Encoding.UTF8.GetBytes(
            $"{DateTime.UtcNow:O} {message}\n");
        _logStream?.Write(bytes, 0, bytes.Length);
    }

    // Dispose 模式
    public void Dispose()
    {
        if (_disposed) return;
        
        _logStream?.Dispose();
        _cache.Clear();
        _disposed = true;
        GC.SuppressFinalize(this);  // 不需要终结器了
    }
}

// using 声明（C# 8.0+ 简洁语法）
public static void UsingExample()
{
    using var cache = new ResourceCache("operations.log");
    cache.Store("user-data", new byte[] { 1, 2, 3 });
    // 作用域结束时自动 Dispose
}

// ========== 2. Span<T> 零分配操作 ==========
public static class SpanExamples
{
    // 解析 CSV 行，零分配
    public static IEnumerable<(int Start, int Length)> ParseCsv(ReadOnlySpan<char> line)
    {
        var results = new List<(int, int)>();
        int start = 0;
        
        for (int i = 0; i < line.Length; i++)
        {
            if (line[i] == ',')
            {
                results.Add((start, i - start));
                start = i + 1;
            }
        }
        results.Add((start, line.Length - start));
        return results;
    }

    // 栈分配 + Span
    public static string GenerateId(int length)
    {
        const string chars = "ABCDEFGHIJKLMNOPQRSTUVWXYZ0123456789";
        Span<char> buffer = stackalloc char[length];  // 栈分配
        Random.Shared.GetItems<char>(chars, buffer);
        return new string(buffer);
    }

    // 字符串处理免分配
    public static ReadOnlySpan<char> TrimToLength(ReadOnlySpan<char> input, int max)
    {
        return input.Length <= max ? input : input[..max];
    }
}

// ========== 3. 弱引用缓存 ==========
public class WeakCache<TKey, TValue> where TValue : class where TKey : notnull
{
    private readonly Dictionary<TKey, WeakReference<TValue>> _cache = new();
    private readonly Func<TKey, TValue> _factory;

    public WeakCache(Func<TKey, TValue> factory) => _factory = factory;

    public TValue GetOrCreate(TKey key)
    {
        // 检查缓存
        if (_cache.TryGetValue(key, out var weakRef) &&
            weakRef.TryGetTarget(out var existing))
        {
            return existing;
        }

        // 创建新值
        var value = _factory(key);
        _cache[key] = new WeakReference<TValue>(value);
        return value;
    }

    public void Cleanup()
    {
        foreach (var key in _cache.Keys.ToList())
        {
            if (!_cache[key].TryGetTarget(out _))
                _cache.Remove(key);
        }
    }
}

// ========== 4. ArrayPool<T> 共享缓冲区 ==========
public static class PooledProcessing
{
    public static byte[] ProcessLargeFile(string path)
    {
        // 从池中租用缓冲区（不需要分配）
        byte[] buffer = ArrayPool<byte>.Shared.Rent(4096);
        using var ms = new MemoryStream();
        
        try
        {
            using var fs = File.OpenRead(path);
            int bytesRead;
            while ((bytesRead = fs.Read(buffer, 0, buffer.Length)) > 0)
            {
                ms.Write(buffer, 0, bytesRead);
            }
            return ms.ToArray();
        }
        finally
        {
            // 归还缓冲区（必须！）
            ArrayPool<byte>.Shared.Return(buffer);
        }
    }
}

// ========== 5. GC 控制与监控 ==========
public static class GcMonitor
{
    public static void Demonstrate()
    {
        // 获取 GC 信息
        Console.WriteLine($"Gen 0 collections: {GC.CollectionCount(0)}");
        Console.WriteLine($"Gen 1 collections: {GC.CollectionCount(1)}");
        Console.WriteLine($"Gen 2 collections: {GC.CollectionCount(2)}");
        Console.WriteLine($"Total memory: {GC.GetTotalMemory(false) / 1024} KB");
        
        // 手动触发 GC（通常不需要，但基准测试中有用）
        GC.Collect();
        GC.WaitForPendingFinalizers();
        GC.Collect();  // 第二次收集终结器处理过的对象
        
        // 低延迟模式（服务器/实时场景）
        // GCLatencyMode oldMode = GCSettings.LatencyMode;
        // GCSettings.LatencyMode = GCLatencyMode.LowLatency;
        // ... 执行延迟敏感代码 ...
        // GCSettings.LatencyMode = oldMode;
    }

## 推荐书籍

1. **《CLR via C#》（第4版）** — Jeffrey Richter
   第 21-22 章是 .NET GC 和内存管理的权威参考——从分代 GC 的内部机制到 Dispose 模式，涵盖所有细节。

2. **《Pro .NET Memory Management》** — Konrad Kokosa
   专门讲 .NET 内存管理的完整著作，从 GC 内部、LOH、POH 到 Span/Memory、性能分析工具。

3. **《Writing High-Performance .NET Code》（第2版）** — Ben Watson
   从实战角度讲解 Span、ArrayPool、减少分配等性能优化技术，包含 BenchmarkDotNet 基准测试。

4. **《C# 12 in a Nutshell》** — Joseph Albahari
   第 12 章（Disposal and Garbage Collection）和第 24 章（Span\<T\> and Memory\<T\>）提供全面的 API 参考。
