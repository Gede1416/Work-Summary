# C# 异步与并发编程

## 详细描述

C# 的异步编程模型基于 `Task` 和 `async/await`，配合丰富的并发原语，构建了从 UI 响应到服务器吞吐的完整异步生态。

**async/await** 是 C# 5.0 引入的革命性异步语法。`async` 标记的方法被编译器转换为状态机，`await` 在等待异步操作完成时"挂起"方法并将控制权归还调用者。关键理解：await 不会阻塞线程——被挂起的方法在异步操作完成后由捕获的 SynchronizationContext（UI 线程）或线程池线程恢复执行。`ConfigureAwait(false)` 是库代码的标准实践，避免不必要的上下文捕获。

**Task 与 ValueTask**：`Task<T>` 表示可能尚未完成的异步操作，是 .NET 异步编程的核心类型。`ValueTask<T>`（C# 7.0）是 Task 的零分配替代品——当操作经常同步完成时（如缓存命中），ValueTask 可以避免堆分配。但使用规则更严格：不能多次 await，不能并发等待，不能阻塞。

**取消令牌**（CancellationToken）提供了协作式取消机制。异步方法接受 `CancellationToken` 参数，定期检查 `IsCancellationRequested` 或调用 `ThrowIfCancellationRequested()`。`CancellationTokenSource` 生成 token 并在超时或手动取消时发出信号。

**IAsyncEnumerable\<T\>**（C# 8.0）是异步世界中的 `IEnumerable<T>`——支持用 `await foreach` 消费异步流，生产者用 `yield return` 逐个产生元素。这是处理分页 API、流式数据处理、实时事件流的标准方式。

**lock 与 SemaphoreSlim**：`lock` 是互斥锁的糖衣语法，同一个线程可以递归进入（可重入）。`SemaphoreSlim` 是更灵活的同步原语——支持异步等待（`WaitAsync()`），可以限制并发数量（信号量模式），适合控制 HTTP 请求并发数等场景。

**Channel\<T\>**（System.Threading.Channels）实现了生产者-消费者模式的高性能异步管道。支持有界/无界通道、单/多生产者/消费者，内部使用 `ValueTask` 实现零分配等待。是构建数据处理管道的利器。

**PLINQ**（Parallel LINQ）通过 `.AsParallel()` 让 LINQ 查询自动并行化。`Parallel.ForEach` 和 `Parallel.For` 提供数据并行。适合 CPU 密集型计算的并行化，内部使用任务并行库（TPL）自动管理线程分配和工作窃取。

## 代码实例

```csharp
// ========== 1. async/await 基础 ==========
public class WeatherService
{
    private readonly HttpClient _httpClient;

    public WeatherService(HttpClient httpClient)
    {
        _httpClient = httpClient;
    }

    public async Task<WeatherData?> GetWeatherAsync(string city)
    {
        try
        {
            var response = await _httpClient.GetAsync($"/weather/{city}");
            response.EnsureSuccessStatusCode();
            return await response.Content.ReadFromJsonAsync<WeatherData>();
        }
        catch (HttpRequestException ex)
        {
            Console.WriteLine($"Network error: {ex.Message}");
            return null;
        }
    }

    // 并发请求多个城市
    public async Task<IReadOnlyDictionary<string, WeatherData?>> GetWeatherForCitiesAsync(
        params string[] cities)
    {
        var tasks = cities.ToDictionary(
            city => city,
            city => GetWeatherAsync(city));  // 不 await，直接获取 Task

        await Task.WhenAll(tasks.Values);  // 并发等待所有

        return tasks.ToDictionary(
            kvp => kvp.Key,
            kvp => kvp.Value.Result);
    }
}

// ========== 2. ValueTask 与取消 ==========
public class DataCache
{
    private readonly ConcurrentDictionary<string, CachedItem> _cache = new();

    public async ValueTask<string?> GetAsync(string key, CancellationToken ct = default)
    {
        // 缓存命中：同步返回，零分配
        if (_cache.TryGetValue(key, out var item) && !item.IsExpired)
            return new ValueTask<string?>(item.Value);

        // 缓存未命中：异步获取
        var value = await FetchFromDatabaseAsync(key, ct);
        _cache[key] = new CachedItem(value, DateTime.UtcNow.AddMinutes(5));
        return value;
    }

    private async Task<string> FetchFromDatabaseAsync(string key, CancellationToken ct)
    {
        await Task.Delay(100, ct);  // 模拟数据库操作
        ct.ThrowIfCancellationRequested();
        return $"data-for-{key}";
    }

    private record CachedItem(string? Value, DateTime ExpiresAt)
    {
        public bool IsExpired => DateTime.UtcNow > ExpiresAt;
    }
}

// ========== 3. IAsyncEnumerable 流式处理 ==========
public class LogStreamer
{
    public async IAsyncEnumerable<string> ReadLogsAsync(
        string path,
        [EnumeratorCancellation] CancellationToken ct = default)
    {
        using var reader = new StreamReader(path);
        while (!reader.EndOfStream)
        {
            ct.ThrowIfCancellationRequested();
            var line = await reader.ReadLineAsync(ct);
            if (line != null)
            {
                yield return line;
            }
        }
    }

    public async Task ProcessLogsAsync(string path)
    {
        await foreach (var line in ReadLogsAsync(path).Where(l => l.Contains("ERROR")))
        {
            Console.WriteLine($"Error found: {line}");
        }
    }
}

// ========== 4. SemaphoreSlim 并发控制 ==========
public class RateLimitedHttpClient
{
    private readonly HttpClient _httpClient;
    private readonly SemaphoreSlim _semaphore;

    public RateLimitedHttpClient(HttpClient httpClient, int maxConcurrency = 5)
    {
        _httpClient = httpClient;
        _semaphore = new SemaphoreSlim(maxConcurrency);
    }

    public async Task<string> GetStringAsync(string url, CancellationToken ct = default)
    {
        await _semaphore.WaitAsync(ct);
        try
        {
            return await _httpClient.GetStringAsync(url, ct);
        }
        finally
        {
            _semaphore.Release();
        }
    }
}

// ========== 5. Channel<T> 生产者-消费者 ==========
public class DataPipeline
{
    private readonly Channel<int> _channel = Channel.CreateBounded<int>(
        new BoundedChannelOptions(100)
        {
            FullMode = BoundedChannelFullMode.Wait,
            SingleWriter = true,
            SingleReader = false,
        });

    public async Task ProduceAsync(CancellationToken ct)
    {
        for (int i = 0; i < 1000; i++)
        {
            await _channel.Writer.WriteAsync(i, ct);
            Console.WriteLine($"Produced: {i}");
        }
        _channel.Writer.Complete();
    }

    public async Task ConsumeAsync(int consumerId, CancellationToken ct)
    {
        await foreach (var item in _channel.Reader.ReadAllAsync(ct))
        {
            await Task.Delay(50, ct);  // 模拟处理
            Console.WriteLine($"Consumer {consumerId} processed: {item}");
        }
    }

    public async Task RunPipelineAsync()
    {
        var cts = new CancellationTokenSource();
        var consumers = Enumerable.Range(0, 3)
            .Select(i => ConsumeAsync(i, cts.Token));
        
        await Task.WhenAll(consumers.Prepend(ProduceAsync(cts.Token)));
    }
}

// ========== 6. PLINQ 数据并行 ==========
public static class ParallelDemo
{
    public static void Demonstrate()
    {
        var numbers = Enumerable.Range(1, 1000000);
        
        // PLINQ：自动并行化查询
        var primeCount = numbers
            .AsParallel()
            .WithDegreeOfParallelism(Environment.ProcessorCount)
            .Where(IsPrime)
            .Count();
        
        Console.WriteLine($"Prime count: {primeCount}");
        
        // Parallel.ForEach
        var results = new ConcurrentBag<int>();
        Parallel.ForEach(numbers, n =>
        {
            if (IsPrime(n))
                results.Add(n);
        });
    }
    
    private static bool IsPrime(int n)
    {
        if (n < 2) return false;
        for (int i = 2; i <= Math.Sqrt(n); i++)
            if (n % i == 0) return false;
        return true;
    }
}
```

## 推荐书籍

1. **《C# 12 in a Nutshell》** — Joseph Albahari
   第 14 章（Concurrency and Asynchrony）和第 23 章（Parallel Programming）是 C# 异步和并发的百科全书式参考。

2. **《Concurrency in C# Cookbook》（第2版）** — Stephen Cleary
   以 Cookbook 形式组织，覆盖 async/await、Task、ValueTask、Channel、异步流等所有异步场景的实战方案。

3. **《Pro .NET Memory Management》** — Konrad Kokosa
   其中有专门的章节讨论 ValueTask 的内存语义和 Channel 的内部实现机制。

4. **《Async in C# 5.0》** — Alex Davies
   虽然讲的是 C# 5.0，但对 async/await 的状态机转换和 SynchronizationContext 的深入解释至今不可替代。
