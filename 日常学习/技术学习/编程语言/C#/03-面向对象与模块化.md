# C# 面向对象与模块化

## 详细描述

C# 的面向对象系统设计精良，融合了传统 OOP 的精髓和现代语言的便利特性。理解 C# 的类型层次和模块化机制是构建大型 .NET 应用的基础。

**类与对象**：C# 的类支持单一继承（一个类只能有一个直接基类），但可以实现多个接口。所有类型最终继承自 `System.Object`，获得 `ToString()`、`Equals()`、`GetHashCode()` 等基本方法。成员访问级别有 `public`、`private`、`protected`、`internal`、`protected internal`、`private protected`（C# 7.2）六个级别。

**抽象类与接口**：抽象类（`abstract`）可以包含实现代码、字段和构造函数，适合有共享状态的继承体系。接口（`interface`）只定义契约，C# 8.0+ 接口可以包含默认实现——这一特性使得在不破坏现有实现的前提下为接口添加方法成为可能。

**record**（C# 9.0）彻底改变了不可变数据类型的定义方式。`record` 或 `record class` 是基于值的相等比较（比较所有属性值而非引用），`record struct`（C# 10）是值类型的等价物。`with` 表达式基于现有记录创建修改了部分属性的新副本，在不可变编程模型中价值巨大。编译器自动为 record 生成 `Equals`、`GetHashCode`、`ToString` 和 `Deconstruct` 方法。

**sealed** 关键字阻止类被继承或方法被继续重写，有助于 JIT 编译器进行去虚化优化（devirtualization），提升性能。

**扩展方法**（Extension Methods）是 C# 最优雅的特性之一——可以为已存在的类型（包括你无法修改源代码的类型）添加"方法"，而无需创建新的派生类型。本质上是对静态方法的语法糖，但使用起来像是类型的实例方法。LINQ 的整个方法链式 API 就是通过 `IEnumerable<T>` 的扩展方法实现的。

**程序集**（Assembly）是 .NET 中的部署、版本控制和代码重用的基本单元，可以是 `.dll`（类库）或 `.exe`（可执行文件）。**项目引用**在编译时将多个项目关联起来。

**命名空间**组织类型，避免命名冲突。C# 10 引入了文件作用域命名空间（`namespace MyApp;`），去掉了传统块声明的额外缩进层级。

## 代码实例

```csharp
// ========== 1. 类、抽象类与接口 ==========

// 接口（C# 8.0+ 可以有默认实现）
public interface IIdentifiable
{
    Guid Id { get; }
    
    // 默认实现
    string DisplayId() => Id.ToString("D")[..8];
}

public interface IAuditable
{
    DateTime CreatedAt { get; }
    DateTime UpdatedAt { get; }
    
    // 默认实现：检查是否为最近创建
    bool IsRecentlyCreated() =>
        (DateTime.UtcNow - CreatedAt).TotalHours < 24;
}

// 抽象类：含共享状态
public abstract class Entity
{
    public Guid Id { get; protected init; } = Guid.NewGuid();
    public DateTime CreatedAt { get; init; } = DateTime.UtcNow;
    public DateTime UpdatedAt { get; protected set; } = DateTime.UtcNow;
    
    public abstract string Describe();
    
    // 钩子方法：子类可重写
    protected virtual void OnUpdate() { }
    
    public void MarkUpdated()
    {
        UpdatedAt = DateTime.UtcNow;
        OnUpdate();
    }
}

// 具体类：继承 Entity + 实现两个接口
public class Customer : Entity, IIdentifiable, IAuditable
{
    public string Name { get; set; }
    public string Email { get; set; }
    
    public Customer(string name, string email)
    {
        Name = name;
        Email = email;
    }
    
    public override string Describe() =>
        $"Customer: {Name} ({Email})";
    
    protected override void OnUpdate()
    {
        Console.WriteLine($"Customer {Name} updated");
    }
}

// ========== 2. Record 与不可变数据 ==========

// 不可变记录
public record Address(string Street, string City, string Country, string PostCode);

// 可变记录需要显式声明
public record MutableRecord
{
    public string Value { get; set; }
}

// 记录结构体（值类型版本，C# 10）
public readonly record struct Point(double X, double Y, double Z);

// Record 使用示例
public static class RecordDemo
{
    public static void Demonstrate()
    {
        var addr1 = new Address("123 Main St", "London", "UK", "SW1A 1AA");
        var addr2 = new Address("123 Main St", "London", "UK", "SW1A 1AA");
        
        Console.WriteLine(addr1 == addr2);  // true（值相等，非引用相等）
        
        // with 表达式：创建修改后的副本
        var addr3 = addr1 with { PostCode = "EC1A 1BB" };
        Console.WriteLine(addr3);
        
        // 解构（编译器自动生成）
        var (street, city, country, postCode) = addr1;
        
        // 非破坏性变异（Non-destructive Mutation）
        var updated = addr1 with { Street = "456 High St" };
    }
}

// ========== 3. 扩展方法 ==========
public static class StringExtensions
{
    // 为 string 添加扩展方法
    public static string Truncate(this string value, int maxLength)
    {
        ArgumentNullException.ThrowIfNull(value);
        return value.Length <= maxLength
            ? value
            : value[..maxLength] + "...";
    }
    
    public static bool IsValidEmail(this string value)
    {
        return value.Contains('@') && value.Contains('.');
    }
    
    // 为 IAuditable 添加扩展方法
    public static string AgeDescription(this IAuditable auditable)
    {
        var age = DateTime.UtcNow - auditable.CreatedAt;
        return age.TotalDays switch
        {
            < 1 => "今天创建",
            < 7 => $"{age.Days}天前",
            < 365 => $"{age.Days / 30}个月前",
            _ => $"{age.Days / 365}年前",
        };
    }
}

// ========== 4. 程序集与模块化 ==========

// 不同的访问修饰符
public class VisibilityDemo
{
    public int PublicField;          // 完全公开
    private int PrivateField;        // 类内可见
    protected int ProtectedField;    // 类内 + 派生类
    internal int InternalField;      // 同一程序集内可见
    protected internal int Both;     // 派生类或同一程序集
    private protected int Narrow;    // 派生类且同一程序集（C# 7.2）
}

// ========== 5. 单例与依赖注入模式 ==========
public class ConfigManager
{
    private static readonly Lazy<ConfigManager> _instance = new(() => new ConfigManager());
    public static ConfigManager Instance => _instance.Value;
    
    public string ConnectionString { get; private set; } = "";
    
    private ConfigManager() { }
    
    public void Load(string path) { /* 加载配置 */ }
}

// ========== 6. IDisposable 模式 ==========
public class ResourceWrapper : IDisposable
{
    private bool _disposed;
    private readonly FileStream _stream;
    
    public ResourceWrapper(string path)
    {
        _stream = File.OpenRead(path);
    }
    
    public void Process() { /* 使用 _stream */ }
    
    public void Dispose()
    {
        if (!_disposed)
        {
            _stream.Dispose();
            _disposed = true;
        }
        GC.SuppressFinalize(this);
    }
}
```

## 推荐书籍

1. **《C# 12 in a Nutshell》** — Joseph Albahari
   第 3-4 章（类型基础与面向对象）和 第 17 章（程序集）提供了完整的 C# OOP 参考。

2. **《Adaptive Code via C#》** — Gary McLean Hall
   深入讲解 SOLID 原则在 C# 中的具体应用、依赖注入模式、以及通过接口和抽象类设计可测试代码。

3. **《Dependency Injection in .NET》** — Mark Seemann
   系统讲解接口、抽象类在实际项目中的依赖注入设计，是理解 .NET 模块化的关键参考。

4. **《C# in Depth》（第4版）** — Jon Skeet
   记录（record）、init 访问器、默认接口方法等新特性有详细演进分析和内部实现讲解。
