# Lua 基础语法与类型系统

## 详细描述

Lua 的类型系统极为精简，总共只有 **8 种基础类型**：`nil`、`boolean`、`number`、`string`、`function`、`userdata`、`thread` 和 `table`。这种极简主义是 Lua 高效和可嵌入性的关键。

**nil** 是 Lua 中表示"无"的唯一类型。未初始化的变量默认为 `nil`，删除表中字段的方式也是将其赋值为 `nil`。这是 Lua 中最常见的类型之一，几乎所有条件判断中 `nil` 都被视为 `false`。

**boolean** 只有 `true` 和 `false`。需要注意：Lua 中 **只有 `false` 和 `nil` 被视为假值**，包括 `0`（零）和空字符串 `""` 都是真值，这与 C/C++/Python 截然不同。

**number** 在 Lua 5.3 之前默认是双精度浮点数（IEEE 754 double）。Lua 5.3 引入了整数子类型（64 位有符号整数），支持位运算和 `//`（整数除法）。对于 LuaJIT，number 默认也是 double，但可以通过编译选项定制。

**string** 是不可变的字节序列。Lua 字符串可以用单引号、双引号、或双中括号（`[[ ]]`，用于多行字符串）表示。字符串有内部化（interning）机制，相同内容的字符串只存储一份，因此字符串比较是 O(1) 的引用比较。自动数字与字符串之间的转换是常见陷阱。

**function** 是 Lua 的头等公民——函数可以赋值给变量、作为参数传递、作为返回值。配合闭包特性，Lua 的函数式编程表达力很强。

**table** 是 Lua 唯一的数据结构，它既是数组（整数键）也是哈希表（任意键）。数组索引从 1 开始（而不是 0），这是 Lua 的一个重要特点。table 的灵活性使其可以模拟模块、类、命名空间等概念。

**userdata** 是 C 语言暴露给 Lua 的自定义数据类型，分为 full userdata（Lua 管理内存）和 light userdata（C 管理内存，只是一个指针）。

**多重赋值**是 Lua 语法的独特特性：`a, b = 1, 2`。多余的值被丢弃，不足的变量被赋为 `nil`。这个特性常用于交换变量（`a, b = b, a`）和接收函数的多返回值。

**local 变量**通过 `local` 关键字声明，访问速度远快于全局变量（全局变量实际存储在 `_ENV` 表中）。最佳实践要求默认使用 `local`。

## 代码实例

```lua
-- ========== 1. 基本类型 ==========

-- nil
local nothing = nil
print(nil == nil)   --> true
print(type(nil))    --> "nil"

-- boolean
local flag = true
local also_false = false

-- 注意：只有 false 和 nil 是假值！
if 0 then print("0 是真值") end           --> 输出
if "" then print("空字符串是真值") end     --> 输出
if nil then print("不会输出") end          --> 不输出
if false then print("不会输出") end        --> 不输出

-- number（Lua 5.3+ 支持整数子类型）
local float_num = 3.14
local int_num = 42
print(math.type(3.14))   --> "float"
print(math.type(42))     --> "integer"

-- 整数运算
local a, b = 10, 3
print(a // b)   --> 3（整数除法）
print(a & b)    --> 2（位与）
print(a << 1)   --> 20（左移）

-- string
local s1 = "hello"
local s2 = 'world'
local s3 = [[
多行字符串
不需要转义 "引号" 和 '单引号'
]]
print(#s1)            --> 5（长度运算符）
print(s1 .. " " .. s2) --> "hello world"（.. 是字符串拼接）

-- 字符串与数字的自动转换（需谨慎）
print("10" + 1)    --> 11（自动转换）
print("hello" + 1) --> 错误！（非数字字符串不能转换）
print(tostring(42) .. " 是答案") --> "42 是答案"
print(tonumber("99") + 1)        --> 100

-- ========== 2. 多重赋值 ==========

-- 基本多重赋值
local x, y, z = 1, 2, 3

-- 交换变量，无需临时变量
x, y = y, x
print(x, y) --> 2, 1

-- 多返回值接收
local function getMinMax(t)
    local min, max = t[1], t[1]
    for _, v in ipairs(t) do
        if v < min then min = v end
        if v > max then max = v end
    end
    return min, max
end

local minVal, maxVal = getMinMax({5, 2, 9, 1, 7})
print(minVal, maxVal) --> 1, 9

-- 不足的变量被赋为 nil
local a2, b2, c2 = 1, 2
print(a2, b2, c2) --> 1, 2, nil

-- ========== 3. Table ==========

-- table 是数组与哈希表的统一
local t = {}

-- 数组用法（索引从 1 开始！）
local fruits = {"apple", "banana", "cherry"}
print(fruits[1])     --> "apple"（不是 [0]！）
print(#fruits)       --> 3（长度运算符）

-- 哈希表用法
local person = {
    name = "Alice",
    age = 30,
    ["favorite-language"] = "Lua",  -- 特殊键需要方括号
}
print(person.name)               --> "Alice"
print(person["age"])             --> 30
print(person["favorite-language"]) --> "Lua"

-- 迭代
for key, value in pairs(person) do
    print(key, "=", value)
end

for index, value in ipairs(fruits) do
    print(index, value)
end

-- ========== 4. local 与全局变量 ==========

-- 推荐：始终使用 local
local private_var = "我是局部变量"

-- 避免：未声明的变量默认全局
global_var = "我是全局变量（危险！）"  -- 实际存储在 _ENV.global_var

-- do-end 块创建局部作用域
local outer = "外部"
do
    local inner = "内部"
    print(inner)  --> "内部"
    print(outer)  --> "外部"（可以访问外部变量）
end
-- print(inner) -- 错误！inner 已超出作用域

-- 变长参数 ...
local function sum(...)
    local total = 0
    local args = {...}  -- 打包为 table
    for _, v in ipairs(args) do
        total = total + v
    end
    return total
end

print(sum(1, 2, 3, 4, 5)) --> 15

-- variadic 结合多重赋值
local function firstTwo(...)
    local a, b = ...  -- 取前两个参数
    return a, b
end

print(firstTwo(10, 20, 30)) --> 10, 20
```

## 推荐书籍

1. **《Programming in Lua》（第4版）** — Roberto Ierusalimschy
   Lua 作者亲自撰写的权威教材，从基础语法到高级特性，涵盖 Lua 5.3，是所有 Lua 学习者的必读经典。内容循序渐进，示例丰富。

2. **《Lua Quick Start Guide》** — Gabor Szauer
   面向初学者的实用入门指南，以项目驱动的方式讲解 Lua，适合快速上手并动手实践。

3. **《Lua Programming Gems》** — Luiz Henrique de Figueiredo 等编
   Lua 社区贡献的精华文章集，涵盖性能优化、C API、游戏脚本等进阶主题，适合读完后对 Lua 有深入理解需求的读者。

4. **《Lua 5.4 Reference Manual》**
   官方参考手册是 Lua 最精确的文档，不建议作为入门教材，但学习过程中遇到疑问时应以此为最终权威。
