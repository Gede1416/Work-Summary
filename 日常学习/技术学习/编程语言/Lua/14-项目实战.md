# Lua 项目实战

## 详细描述

Lua 的实战能力体现在多个领域：游戏脚本、嵌入式扩展、高性能网关配置、协程调度框架。

**游戏脚本（AI/UI）**：实现 NPC 行为树（Behavior Tree）系统，每个节点是 Lua 函数并通过协程实现异步行为（等待、巡逻、追击）。行为树节点可组合：Sequence（顺序执行）、Selector（选择执行）、Decorator（条件节点）。

**Nginx 动态配置**（OpenResty）：在 Nginx 的 access 阶段用 Lua 实现复杂的鉴权和限流逻辑。通过 `ngx.shared.DICT` 共享内存实现分布式计数器和缓存。

**C 程序 Lua 扩展**：通过 Lua C API 将 C 函数暴露给 Lua 脚本。注册函数、管理 userdata、处理错误。这是理解 Lua 可嵌入性本质的实践——Lua 不仅是脚本语言，更是 C 程序的扩展接口。

**协程调度框架**：实现一个基于协程的简易异步框架——定时器、网络 I/O 等待、Future/Promise 模型。虽然简单，但能深入理解非抢占式调度的本质。

## 代码实例

```lua
-- ========== 行为树系统 ==========
local BehaviorTree = {}

-- Sequence: 按顺序执行，一个失败整体失败
function BehaviorTree.Sequence(children)
    return function()
        for _, child in ipairs(children) do
            local status = child()
            if status ~= "success" then return status end
        end
        return "success"
    end
end

-- Selector: 按顺序尝试，一个成功整体成功
function BehaviorTree.Selector(children)
    return function()
        for _, child in ipairs(children) do
            local status = child()
            if status ~= "failure" then return status end
        end
        return "failure"
    end
end

-- 条件节点
function BehaviorTree.Condition(condition, action)
    return function()
        if condition() then return action() end
        return "failure"
    end
end

-- 示例：敌人 AI
local function isPlayerNearby()
    return math.random() > 0.3  -- 模拟检测
end

local function chasePlayer()
    print("Enemy: Chasing player!")
    return "success"
end

local function patrol()
    print("Enemy: Patrolling...")
    return "running" -- 持续进行中
end

local enemyAI = BehaviorTree.Selector({
    BehaviorTree.Condition(isPlayerNearby, chasePlayer),
    patrol,
})

for i = 1, 5 do
    print("Tick", i, "->", enemyAI())
end

-- ========== 简易协程调度器 ==========
local Scheduler = {}
Scheduler.__index = Scheduler

function Scheduler:new()
    return setmetatable({
        tasks = {},       -- { {co, wakeTime} }
        time = 0,
    }, self)
end

function Scheduler:spawn(fn)
    local co = coroutine.create(fn)
    table.insert(self.tasks, {co = co, wakeTime = self.time})
    coroutine.resume(co)
end

function Scheduler:sleep(seconds)
    local current = nil
    for i, task in ipairs(self.tasks) do
        if task.co == coroutine.running() then
            current = i
            break
        end
    end
    if current then
        self.tasks[current].wakeTime = self.time + seconds
    end
    coroutine.yield()
end

function Scheduler:run()
    while #self.tasks > 0 do
        self.time = self.time + 1
        
        for i = #self.tasks, 1, -1 do
            local task = self.tasks[i]
            if task.wakeTime <= self.time then
                if coroutine.status(task.co) == "dead" then
                    table.remove(self.tasks, i)
                else
                    coroutine.resume(task.co)
                end
            end
        end
    end
end

-- 使用调度器
local sched = Scheduler:new()

sched:spawn(function()
    print("[Task A] Start")
    sched:sleep(2)
    print("[Task A] After 2 ticks")
    sched:sleep(1)
    print("[Task A] Done")
end)

sched:spawn(function()
    for i = 1, 4 do
        print(string.format("[Task B] Tick %d", i))
        sched:sleep(1)
    end
end)

sched:run()
```

## 推荐书籍

1. **《Programming in Lua》（第4版）** — Roberto Ierusalimschy
   第 24 章（C API）和第 25 章（Extending Your Application）是关于 Lua 嵌入和扩展的权威资料。
2. **《Game AI Pro》** 系列
   虽然不是 Lua 专门书，但其中行为树和状态机的实现思路很适合用 Lua 落地。
3. **OpenResty 官方教程**（openresty.org）
   Nginx + Lua 的开源实战文档，包含完整的 API 网关实现案例。
