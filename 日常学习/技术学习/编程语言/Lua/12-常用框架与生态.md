# Lua 常用框架与生态

## 详细描述

Lua 的生态以**嵌入脚本**和**游戏开发**为核心，辅以高性能网络和配置管理场景。

**游戏脚本**是 Lua 最核心的应用领域：
- **LÖVE（Love2D）**：2D 游戏引擎，API 覆盖图形、音频、物理、输入。用 Lua 编写，简单优雅。
- **World of Warcraft**：暴雪使用 Lua 作为界面脚本语言（WoW AddOn），数以万计的用户插件活跃至今。
- **Cocos2d-x** / **Unity**：游戏引擎中使用 Lua 作为脚本层（通常通过 tolua++ 或 xLua 绑定）。
- **LuaJIT** 是游戏领域的首选 Lua 实现（高性能、FFI 方便），王者荣耀等手机游戏大量使用。

**Neovim** 将 Lua 作为头等配置语言，替代了 VimScript。Neovim 的整个插件生态都基于 Lua，通过内置的 LuaJIT 运行时执行。`vim.api` 提供完整的编辑器 API。

**Redis** 内嵌 Lua 解释器，通过 `EVAL` 和 `EVALSHA` 命令执行 Lua 脚本来保证原子性操作。

**Nginx / OpenResty**：OpenResty 将 LuaJIT 嵌入 Nginx，提供 `ngx.*` API，能够用 Lua 在 Nginx 层面处理 HTTP 请求、访问缓存、调用后端——性能极高，非阻塞 I/O。

**LuaJIT** 不仅是高性能的 Lua 实现，其 FFI（Foreign Function Interface）允许直接调用 C 函数而无需编写 C 绑定代码，极大简化了与 C 库的交互。

## 代码实例

```lua
-- ========== LÖVE 游戏示例 ==========
--[[
function love.load()
    x, y = 400, 300
    image = love.graphics.newImage("player.png")
end

function love.update(dt)
    if love.keyboard.isDown("left") then x = x - 200 * dt end
    if love.keyboard.isDown("right") then x = x + 200 * dt end
end

function love.draw()
    love.graphics.draw(image, x, y)
end
--]]

-- ========== Neovim 插件示例 ==========
--[[
-- ~/.config/nvim/lua/myplugin/init.lua
local M = {}

function M.setup(opts)
    opts = opts or {}
    
    -- 创建快捷键
    vim.keymap.set("n", "<leader>h", function()
        print("Hello from Lua plugin!")
    end, { desc = "My plugin greeting" })
    
    -- 创建命令
    vim.api.nvim_create_user_command("MyCommand", function()
        vim.notify("Command executed!")
    end, {})
end

return M
--]]

-- ========== Redis Lua 脚本示例 ==========
--[[
-- 原子性地转账
local from = KEYS[1]
local to = KEYS[2]
local amount = tonumber(ARGV[1])

local fromBalance = tonumber(redis.call("GET", from) or 0)
if fromBalance < amount then
    return {err = "Insufficient balance"}
end

redis.call("DECRBY", from, amount)
redis.call("INCRBY", to, amount)
return {ok = "Transfer completed"}
-- EVAL "script" 2 account:A account:B 100
--]]

-- ========== OpenResty 示例 ==========
--[[
-- nginx.conf embedded Lua
server {
    listen 80;
    location /api {
        content_by_lua_block {
            -- 检查缓存
            local cache = ngx.shared.my_cache
            local key = ngx.var.uri
            local cached = cache:get(key)
            if cached then
                ngx.say(cached)
                return
            end
            
            -- 调用后端服务
            local http = require("resty.http")
            local httpc = http.new()
            local res, err = httpc:request_uri("http://backend/" .. key)
            
            if res then
                cache:set(key, res.body, 60)
                ngx.say(res.body)
            else
                ngx.say("Error: ", err)
            end
        }
    }
}
--]]
```

## 推荐书籍

1. **《LÖVE for Lua Game Programming》** — Darmie Akinlaja
   系统讲解用 LÖVE 引擎开发 2D 游戏的完整流程。
2. **《OpenResty Best Practices》**（在线开源）
   OpenResty 官方维护的最佳实践指南，覆盖 ngx.* API、性能优化、安全。
3. **《Neovim: Lua Guide》**（neovim.io 官方文档）
   完整的 Neovim Lua API 手册和插件开发教程。
