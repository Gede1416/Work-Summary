# Lua 最佳实践与设计模式

## 详细描述

Lua 的最佳实践围绕其动态特性展开：善用 `local`、闭包封装、原型式多态、减少 GC 压力是写好 Lua 代码的核心。

**用 local 提升性能**是 Lua 优化第一原则。局部变量的访问速度远快于全局变量（全局变量需要哈希表查找 `_ENV`）。在循环中频繁访问的全局函数（如 `math.sin`、`table.insert`）应缓存到局部变量。

**表即对象**：Lua 中一切皆 table——对象是 table，类是 table，模块是 table。不要对抗这个模式，而是利用它。`self` 是函数调用的第一个隐式参数，通过 `table:method()` 语法糖传递。

**闭包封装私有状态**是 Lua 实现封装的惯用方式。在 `do-end` 块中定义 private 变量，暴露公开函数通过闭包访问这些变量。这种模式比元表模拟的 private 更可靠，因为外部完全无法访问私有变量。

**基于原型的多态**：利用 `__index` 实现方法查找链，利用鸭子类型实现多态——不检查类型，只检查方法是否存在。

**避免全局表污染**：每个模块返回一个 table 而非修改全局表。使用 `_ENV` 或 `setfenv`（Lua 5.1）控制沙箱环境。

**函数式风格处理数据**：Lua 的函数式特性（闭包、高阶函数、迭代器）鼓励链式数据处理。`table` 库提供 `sort`、`concat` 等基础操作，自定义的 `map`/`filter`/`reduce` 可以轻松实现。

## 代码实例

```lua
-- ========== 1. local 性能优化 ==========
-- ❌ 慢：每次循环都进行全局查找
local function slowSum(t)
    local result = 0
    for i = 1, #t do
        result = result + t[i]
    end
    return result
end

-- ✅ 快：缓存常用函数
local function fastSum(t)
    local result = 0
    local len = #t  -- 缓存长度（虽然 #t 本身就很快）
    for i = 1, len do
        result = result + t[i]
    end
    return result
end

-- ========== 2. 闭包封装模块 ==========
local Counter = {}
do
    -- 私有变量，外部完全不可见
    local instances = {}
    
    function Counter:new(initial)
        local count = initial or 0
        local instance = {
            increment = function()
                count = count + 1
                return count
            end,
            decrement = function()
                count = count - 1
                return count
            end,
            getValue = function() return count end,
        }
        table.insert(instances, instance)
        return instance
    end
    
    function Counter.totalInstances()
        return #instances
    end
end

local c1 = Counter:new(0)
local c2 = Counter:new(10)
print(c1.increment())          --> 1
print(c1.increment())          --> 2
print(c2.getValue())           --> 10
print(Counter.totalInstances()) --> 2
-- print(c1.count) -- nil！真正私有，无法从外部访问

-- ========== 3. 观察者模式 ==========
local EventEmitter = {}
EventEmitter.__index = EventEmitter

function EventEmitter:new()
    return setmetatable({ listeners = {} }, self)
end

function EventEmitter:on(event, callback)
    if not self.listeners[event] then
        self.listeners[event] = {}
    end
    table.insert(self.listeners[event], callback)
end

function EventEmitter:emit(event, ...)
    local handlers = self.listeners[event]
    if handlers then
        for _, handler in ipairs(handlers) do
            handler(...)
        end
    end
end

-- ========== 4. 函数式数据管道 ==========
local function map(t, fn)
    local result = {}
    for i, v in ipairs(t) do
        result[i] = fn(v, i)
    end
    return result
end

local function filter(t, fn)
    local result = {}
    for _, v in ipairs(t) do
        if fn(v) then table.insert(result, v) end
    end
    return result
end

local function reduce(t, fn, initial)
    local acc = initial
    for _, v in ipairs(t) do
        acc = fn(acc, v)
    end
    return acc
end

-- 使用管道处理数据
local data = {5, 12, 3, 8, 15, 7}
local result = reduce(
    map(
        filter(data, function(n) return n > 5 end),
        function(n) return n * 2 end
    ),
    function(acc, n) return acc + n end,
    0
)
print(result)  --> (12*2)+(8*2)+(15*2)+(7*2) = 84
```

## 推荐书籍

1. **《Programming in Lua》（第4版）** — Roberto Ierusalimschy
   第 16 章（OOP）、第 17 章（Weak Tables）、第 21 章（Modules）包含 Lua 的最佳实践哲学。
2. **《Lua Programming Gems》** — L.H. de Figueiredo
   社区精华文章，展示了闭包、迭代器、协程在实际应用中的优雅用法。
3. **《Lua Design Patterns》**（在线）— 社区
   收集了观察者、策略、工厂、装饰器等常见模式在 Lua 中的惯用实现。
