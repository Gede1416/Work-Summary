# Lua 元编程与元表

## 详细描述

Lua 的元编程能力完全建立在**元表**（Metatable）和**元方法**（Metamethod）之上——这是 Lua 最强大也最具特色的机制。与其他语言不同，Lua 的所有元编程都是运行时的。

**元表机制**：每个 table（和 userdata）都可以关联一个元表。元表是一个普通的 table，其键是特殊的"事件"名称（以 `__` 开头），值是对应的处理函数（元方法）。当 Lua 在 table 上执行某操作失败时（或执行某些特殊操作时），会检查元表中的相应元方法。

**核心元方法**：
- `__index`：读取不存在的键时调用，可设为 table（原型链）或函数
- `__newindex`：向不存在的键赋值时调用，可用于实现只读表、代理表
- `__call`：将 table 作为函数调用时触发
- `__add`/`__sub`/`__mul`/`__div` 等：算术运算符重载
- `__eq`/`__lt`/`__le`：比较运算符重载
- `__tostring`：`tostring()` 调用时的行为
- `__len`：`#` 运算符的行为
- `__concat`：`..` 字符串拼接运算符
- `__gc`：对象被 GC 回收时的回调（终结器）
- `__mode`：控制弱引用行为
- `__name`：错误消息中使用的类型名称

**`__index` 实现继承和默认值**：如果 `__index` 是一个 table，Lua 自动在表中查找；如果是函数，则调用函数并传入 table 和 key。这是 Lua OOP 实现的核心。

**`__newindex` 实现只读表和代理**：拦截对 table 的写操作。如果元方法存在，赋值操作被捕获——可以用它来实现只读表（禁止赋值）、触发式更新（观察者模式）、或者限制允许的键。

**运算符重载**让自定义类型可以参与到标准 Lua 运算符中。向量、复数、矩阵等数学类型都可以通过运算符重载获得自然的使用语法。

**debug 库**提供更深层的元编程能力：访问调用栈（`debug.traceback`）、查看函数信息（`debug.getinfo`）、读写局部变量（`debug.getlocal`/`debug.setlocal`）、修改变量值（`debug.setupvalue`）。这些 API 在 Lua 调试器和 hot-reload 系统中使用。

**完全运行时**意味着 Lua 的元编程不需要编译步骤、不需要宏、不需要代码生成——所有操作都在运行时通过普通的 Lua 代码完成。这让 Lua 非常灵活，但也意味着错误只在运行时才能被发现。

## 代码实例

```lua
-- ========== 1. __index：默认值与原型链 ==========

-- 带默认值的表
local function createDefaultTable(default)
    return setmetatable({}, {
        __index = function(t, key)
            return default  -- 任何缺失的键都返回默认值
        end
    })
end

local t = createDefaultTable(0)
t.a = 10
print(t.a)  --> 10
print(t.b)  --> 0（默认值）
print(t.c)  --> 0

-- ========== 2. __newindex：只读表与跟踪 ==========

-- 只读表
local function readonly(table)
    local proxy = {}
    local mt = {
        __index = table,
        __newindex = function()
            error("Attempt to modify a read-only table", 2)
        end,
        __pairs = function() return pairs(table) end,
        __len = function() return #table end,
    }
    setmetatable(proxy, mt)
    return proxy
end

local config = readonly({
    host = "localhost",
    port = 8080,
})
print(config.host)   --> "localhost"
-- config.port = 9090  -- 错误！

-- 跟踪表：每次修改记录日志
local function tracked(table, name)
    name = name or "table"
    return setmetatable({}, {
        __index = table,
        __newindex = function(_, key, value)
            print(string.format("[%s] %s changed to %s", name, key, tostring(value)))
            table[key] = value
        end,
    })
end

local data = tracked({}, "settings")
data.theme = "dark"    --> [settings] theme changed to dark
data.fontSize = 14     --> [settings] fontSize changed to 14

-- ========== 3. __call：函数式对象 ==========

-- 可调用的累加器
local Accumulator = {}
Accumulator.__index = Accumulator

function Accumulator:new()
    local acc = { sum = 0 }
    setmetatable(acc, self)
    return acc
end

function Accumulator:__call(n)
    self.sum = self.sum + n
    return self.sum
end

local acc = Accumulator:new()
print(acc(10))  --> 10
print(acc(5))   --> 15
print(acc(3))   --> 18

-- ========== 4. 运算符重载 ==========

-- 2D 向量
local Vector2D = {}
Vector2D.__index = Vector2D

function Vector2D:new(x, y)
    local v = { x = x or 0, y = y or 0 }
    setmetatable(v, self)
    return v
end

function Vector2D:__add(other)
    return Vector2D:new(self.x + other.x, self.y + other.y)
end

function Vector2D:__sub(other)
    return Vector2D:new(self.x - other.x, self.y - other.y)
end

function Vector2D:__mul(scalar)
    return Vector2D:new(self.x * scalar, self.y * scalar)
end

function Vector2D:__eq(other)
    return self.x == other.x and self.y == other.y
end

function Vector2D:__tostring()
    return string.format("Vector2D(%.2f, %.2f)", self.x, self.y)
end

function Vector2D:magnitude()
    return math.sqrt(self.x * self.x + self.y * self.y)
end

local v1 = Vector2D:new(3, 4)
local v2 = Vector2D:new(1, 2)
local v3 = v1 + v2
local v4 = v1 * 2

print(v3)              --> Vector2D(4.00, 6.00)
print(v4)              --> Vector2D(6.00, 8.00)
print(v1 == v2)       --> false
print(v1 == Vector2D:new(3, 4))  --> true
print(v1:magnitude()) --> 5.0

-- ========== 5. debug 库元编程 ==========

-- 获取和设置局部变量
local function inspectLocals()
    local function inner(a, b)
        local x = 10
        local y = 20
        
        for i = 1, debug.getinfo(2, "u").nparams + debug.getinfo(2, "u").nups do
            local name, value = debug.getlocal(2, i)
            if name then
                print(string.format("局部变量 %s = %s", name, tostring(value)))
            end
        end
    end
    
    inner(1, 2)
end

-- 运行时修改函数的上值（upvalue）
local function createCounter(initial)
    local count = initial
    return function()
        count = count + 1
        return count
    end
end

local counter = createCounter(0)
print(counter()) --> 1
print(counter()) --> 2

-- 使用 debug 库重置 counter 的上值
local i = 1
while true do
    local name, value = debug.getupvalue(counter, i)
    if not name then break end
    if name == "count" then
        debug.setupvalue(counter, i, 100)  -- 重置为 100
        break
    end
    i = i + 1
end

print(counter()) --> 101

-- ========== 6. 沙箱环境 ==========

-- 为不受信任的代码创建受限环境
local function sandbox(code, allowedGlobals)
    -- 创建一个只包含允许函数的全局环境
    local env = {}
    for _, name in ipairs(allowedGlobals) do
        env[name] = _G[name]
    end
    
    -- 在沙箱中执行代码
    local chunk, err = load(code, "sandbox", "t", env)
    if not chunk then
        return nil, err
    end
    
    return pcall(chunk)
end

-- 执行受限制的代码
local ok, result = sandbox(
    [[
        local x = 10
        return x * 2
    ]],
    {"print", "math"}  -- 只允许 print 和 math
)
print("结果:", result)  --> 20

-- 危险代码被阻断
local ok2, err2 = sandbox(
    [[ os.execute("rm -rf /") ]],
    {"print"}
)
-- load 失败，因为 os 不在允许的全局列表中
```

## 推荐书籍

1. **《Programming in Lua》（第4版）** — Roberto Ierusalimschy
   第 13 章（Metatables and Metamethods）和第 17 章（Weak Tables and Finalizers）是关于元表和元方法的权威讲解。

2. **《Lua Programming Gems》** — L.H. de Figueiredo
   第 3 章和第 9 章展示了元表在实际项目中的高级应用，包括 DSL 构建和运算符重载的模式。

3. **《Lua 5.4 Reference Manual》**
   第 2.4 节（Metatables and Metamethods）精确定义了所有元方法的触发条件和返回值语义。
