# Lua 函数与作用域

## 详细描述

Lua 中**函数是头等公民**——函数可以存储在变量中、作为参数传递、作为返回值返回。这一特性配合闭包机制，赋予了 Lua 强大的函数式编程能力和灵活的代码组织方式。

**函数定义**有两种等价形式：`function name(params) body end` 和 `name = function(params) body end`。前者本质上是后者的语法糖——将匿名函数赋值给变量。理解这一点对理解 Lua 的函数传递和模块系统至关重要。

**闭包**是 Lua 最强大的特性之一。当一个函数访问其外部函数的局部变量时，就形成了闭包——这些被捕获的变量（称为 upvalue）的生命周期会被延长，即使外部函数已经返回。闭包在 Lua 中广泛用于实现迭代器、回调、状态封装、模块私有状态等模式。

**尾调用优化**（Tail Call Optimization, TCO）是 Lua 的一个独特优势。当函数的最后一个动作是调用另一个函数时，Lua 会复用当前栈帧而不是创建新的。这意味着只要满足"尾调用"条件，递归就不会导致栈溢出。这一特性使得用 Lua 写状态机非常高效，每个状态转换就是一个尾调用。

**可变参数**通过 `...` 表示，函数可以接收任意数量的参数。使用 `{...}` 将所有变长参数打包为 table，使用 `select("#", ...)` 获取参数个数，使用 `select(i, ...)` 获取从第 i 个开始的所有参数。Lua 5.1 中 `...` 有一些限制（如在嵌套函数中使用），5.2+ 已大幅改善。

**作用域规则**与多数语言不同：Lua **没有块级作用域**。变量要么是全局的（存储在 `_ENV` 表中），要么是通过 `local` 声明的局部变量，局部变量的作用域是所在的整个 block（如 then-end、do-end、函数体）。这意味着在 if 或 for 的 body 中声明的 `local` 变量在整个 block 中可见。

**do-end 块**专门用于创建局部作用域——在 do 和 end 之间声明的 local 变量在 end 之后完全不可见。这在需要临时变量时非常有用。

**严格与非严格模式**：Lua 本身没有自动的严格模式，但可以通过 Lua 的沙盒环境和自定义的 `_ENV` 来实现类严格的行为（如禁止全局变量创建）。Lua 5.4 引入了 `<const>` 声明（实验性），可以为局部变量标记常量属性。

**函数多返回值**是 Lua 的特色——一个函数可以返回多个值，调用者可以用多重赋值接收。如果函数调用不是表达式列表中的最后一个，多返回值会被截断为第一个值。

## 代码实例

```lua
-- ========== 1. 函数定义 ==========

-- 语法糖形式
function factorial(n)
    if n <= 1 then return 1 end
    return n * factorial(n - 1)
end

-- 等价形式：函数是值
local factorial2 = function(n)
    if n <= 1 then return 1 end
    return n * factorial2(n - 1)  -- 注意：需要引用变量名
end

-- 函数作为参数
local function apply(func, value)
    return func(value)
end

local double = function(x) return x * 2 end
print(apply(double, 21)) --> 42

-- 函数作为返回值
local function makeMultiplier(factor)
    return function(x)
        return x * factor  -- factor 被闭包捕获
    end
end

local triple = makeMultiplier(3)
print(triple(10)) --> 30

-- ========== 2. 闭包应用 ==========

-- 闭包实现计数器
local function createCounter(initial)
    local count = initial or 0
    return {
        increment = function() count = count + 1; return count end,
        decrement = function() count = count - 1; return count end,
        getValue = function() return count end,
        reset = function(v) count = v or 0; return count end,
    }
end

local c1 = createCounter(10)
local c2 = createCounter(0)

c1.increment(); c1.increment()
c2.decrement()
print(c1.getValue(), c2.getValue()) --> 12, -1
-- c1 和 c2 各有独立的闭包状态

-- 闭包实现迭代器（Lua 惯用模式）
local function rangeIterator(from, to, step)
    step = step or 1
    local current = from - step
    return function()
        current = current + step
        if current <= to then return current end
        return nil  -- 迭代结束信号
    end
end

for num in rangeIterator(1, 10, 2) do
    io.write(num .. " ") --> 1 3 5 7 9
end
print()

-- 闭包实现模块私有状态
local MyModule = {}
do
    -- 这些变量对于外部不可见
    local privateData = {}
    local secretKey = 12345
    
    function MyModule.store(key, value)
        privateData[key] = value
    end
    
    function MyModule.retrieve(key)
        return privateData[key]
    end
    
    function MyModule.validate(key)
        return key == secretKey
    end
end

-- ========== 3. 尾调用优化 ==========

-- 状态机：每个状态是一个尾调用
local function stateA(count)
    print("State A:", count)
    if count > 10 then
        return stateC(count)  -- 尾调用
    else
        return stateB(count)  -- 尾调用
    end
end

local function stateB(count)
    print("State B:", count)
    return stateA(count + 1)  -- 尾调用，不增长栈
end

local function stateC(count)
    print("State C: done at", count)
    return "finished"
end

print(stateA(1))  -- 即使循环数万次也不会栈溢出

-- 非尾调用示例（这些不会触发 TCO）：
-- return 1 + func()        -- 需要做加法
-- return func() or default -- 需要判断
-- return (func())          -- 括号阻止尾调用

-- ========== 4. 可变参数 ==========

local function info(first, second, ...)
    print("First:", first)
    print("Second:", second)

    local rest = {...}  -- 打包为 table
    print("Number of extra args:", #rest)

    for i, v in ipairs(rest) do
        print("  Extra", i, ":", v)
    end
end

info("a", "b", "c", "d", "e")

-- select 用法
local function pickFrom(n, ...)
    return select(n, ...)  -- 返回从第 n 个开始的所有参数
end
print(pickFrom(3, "a", "b", "c", "d")) --> c, d

-- ========== 5. 作用域与 do-end ==========

local x = 10  -- 外层 x

if true then
    local x = 20  -- 新局部变量，遮蔽外层 x
    print(x)      --> 20
end
print(x)  --> 10（外层 x 未受影响）

-- do-end 块创建独立作用域
do
    local temp = "temporary"
    print(temp)  --> "temporary"
end
-- print(temp) -- 错误！temp 已超出作用域

-- 习惯用法：用 do-end 限制局部变量作用域
local result
do
    local a = complexCalculation1()
    local b = complexCalculation2()
    result = a + b
    -- a 和 b 在块结束后被回收
end
print(result)

-- ========== 6. 多返回值 ==========

local function stats(t)
    local sum, count = 0, 0
    for _, v in ipairs(t) do
        sum = sum + v
        count = count + 1
    end
    return sum, count, sum / count  -- 返回三个值
end

local total, n, avg = stats({10, 20, 30, 40, 50})
print("Sum:", total, "Count:", n, "Average:", avg)

-- 注意：非最后一个表达式时被截断
print(stats({10, 20, 30, 40})) --> 100（只输出第一个返回值！）
-- 解决方案：用括号强制单个值
print((stats({10, 20, 30, 40}))) --> 100
```

## 推荐书籍

1. **《Programming in Lua》（第4版）** — Roberto Ierusalimschy
   第 6 章深入讲解函数、闭包和尾调用，是理解 Lua 函数式特性的最佳资料。作者用清晰的语言和实用示例展示了闭包的各种应用模式。

2. **《Lua Programming Gems》** — Luiz Henrique de Figueiredo 等编
   第 2-4 章展示了闭包、协程和迭代器在生产环境中的实际应用，包括游戏 AI 和 DSL 设计。

3. **《Lua 5.4 Reference Manual》**
   第 3.4 节（Expressions - Function Calls）对尾调用和多返回值的规则有精确定义。
