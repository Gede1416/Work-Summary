# C++ 面向对象与模块化

## 详细描述

C++ 的面向对象编程建立在编译期绑定（静态多态）和运行时绑定（动态多态）的双重机制之上。虽然现代 C++ 推崇组合优于继承，但理解类的继承体系对于编写合格的 C++ 代码仍是必须的。

**类的基础**：C++ 中 `class` 和 `struct` 的唯一区别是默认访问权限——`class` 默认为 `private`，`struct` 默认为 `public`。习惯上，`struct` 用于简单的数据聚合，`class` 用于包含不变量和行为的类型。成员初始化列表是在构造函数体执行前初始化成员的唯一方式（对于 const 成员和引用成员是必须的）。

**继承体系**支持三种访问方式：public（is-a 关系，最常用）、protected（实现细节但允许派生类使用）、private（实现细节，完全隐藏）。虚函数通过 `virtual` 关键字声明，通过虚函数表（vtable）在运行时实现动态分发。`override` 关键字（C++11）明确标注意图并让编译器检查签名匹配，`final` 禁止进一步重写或继承。

**多重继承**是 C++ 一个强大但容易误用的特性。一个类可以继承多个基类，形成菱形继承（diamond inheritance）时需要虚继承（`virtual` 继承）来解决"最终基类"的重复问题。多重继承的经典安全用法是实现多个纯虚接口（类似于 Java 的 interface）。

**RTTI**（运行时类型信息）通过 `typeid` 和 `dynamic_cast` 提供有限的运行时类型判断。`dynamic_cast` 在失败时对指针返回 `nullptr`，对引用抛出 `std::bad_cast`。由于 RTTI 有运行时开销，许多游戏和嵌入式项目禁用它。

**抽象类**包含至少一个纯虚函数（`virtual void f() = 0`），不能直接实例化。纯虚函数可以有实现，派生类通过 `Base::f()` 调用它。C++ 没有 interface 关键字，纯抽象类（所有成员函数都是纯虚函数）充当接口角色。

**头文件与源文件分离**是 C++ 历史最悠久的模块化方式。头文件（`.h`/`.hpp`）包含声明，源文件（`.cpp`）包含实现。这种分离带来了编译单元的独立性，也导致了头文件包含顺序依赖、重复包含等问题。include guard 和 `#pragma once` 解决重复包含，前向声明减少头文件依赖。

**C++20 Module**从根本上重构了编译模型。`export module` 声明模块接口，`import` 引入模块依赖，不再需要头文件。模块提供了更好的封装（模块内部符号对外不可见）、更快的编译速度（模块只编译一次，接口稳定时无需重新编译依赖方），以及消除宏污染。

## 代码实例

```cpp
#include <iostream>
#include <memory>
#include <vector>
#include <string>

// ========== 1. 类、继承与虚函数 ==========
class Animal {
public:
    explicit Animal(std::string name) : name_(std::move(name)) {}
    virtual ~Animal() = default;  // 虚析构函数（多态基类必须）
    
    virtual std::string speak() const = 0;  // 纯虚函数
    virtual void move() { std::cout << name_ << " moves\n"; }
    
    const std::string& name() const { return name_; }
protected:
    std::string name_;
};

class Dog : public Animal {
public:
    using Animal::Animal;  // 继承构造函数
    
    std::string speak() const override { return "Woof!"; }
    void move() override { std::cout << name() << " runs\n"; }
    void wagTail() { std::cout << name() << " wags tail\n"; }
};

class Cat final : public Animal {  // final: 禁止进一步继承
public:
    using Animal::Animal;
    std::string speak() const override { return "Meow!"; }
    void move() override { std::cout << name() << " slinks\n"; }
};

// ========== 2. 多重继承与虚继承 ==========

// 接口类（纯抽象类）
class IFlyable {
public:
    virtual void fly() = 0;
    virtual ~IFlyable() = default;
};

class ISwimmable {
public:
    virtual void swim() = 0;
    virtual ~ISwimmable() = default;
};

// 多重继承接口
class Duck : public Animal, public IFlyable, public ISwimmable {
public:
    using Animal::Animal;
    std::string speak() const override { return "Quack!"; }
    void fly() override { std::cout << name() << " flies\n"; }
    void swim() override { std::cout << name() << " swims\n"; }
};

// 虚继承解决菱形继承
struct Device { int id; };
struct Printer : virtual Device {};  // 虚继承
struct Scanner : virtual Device {};  // 虚继承
struct AllInOne : Printer, Scanner {};  // 只有一份 Device::id

// ========== 3. RTTI ==========
void rtti_example() {
    std::vector<std::unique_ptr<Animal>> zoo;
    zoo.push_back(std::make_unique<Dog>("Rex"));
    zoo.push_back(std::make_unique<Cat>("Whiskers"));
    zoo.push_back(std::make_unique<Duck>("Donald"));
    
    for (const auto& animal : zoo) {
        std::cout << animal->name() << " says " << animal->speak() << "\n";
        
        // dynamic_cast 检查
        if (auto* dog = dynamic_cast<Dog*>(animal.get())) {
            dog->wagTail();  // 安全：确认是 Dog
        }
        
        if (auto* flyable = dynamic_cast<IFlyable*>(animal.get())) {
            flyable->fly();
        }
        
        // typeid 比较
        if (typeid(*animal) == typeid(Cat)) {
            std::cout << "This is exactly a Cat\n";
        }
    }
}

// ========== 4. C++20 Module ==========
// 注意：模块语法在不同编译器中支持程度不同
// 以下展示概念，实际编译需启用 C++20 模块支持

/*

    std::string name_;

    explicit Dog(std::string name) : name_(std::move(name)) {}
    std::string speak() const { return "Woof!"; }

    std::cout << "Feeding " << dog.speak() << "\n";
}

    Dog rex("Rex");

// ========== 5. RAII 与资源管理 ==========
class FileHandle {
    FILE* file_;
public:
    explicit FileHandle(const char* path, const char* mode)
        : file_(fopen(path, mode)) {
        if (!file_) throw std::runtime_error("Failed to open file");
    }
    
    ~FileHandle() { if (file_) fclose(file_); }
    
    // 禁止拷贝，允许移动
    FileHandle(const FileHandle&) = delete;
    FileHandle& operator=(const FileHandle&) = delete;
    FileHandle(FileHandle&& other) noexcept : file_(other.file_) {
        other.file_ = nullptr;
    }
    FileHandle& operator=(FileHandle&& other) noexcept {
        if (this != &other) {
            if (file_) fclose(file_);
            file_ = other.file_;
            other.file_ = nullptr;
        }
        return *this;
    }
    
    FILE* get() const { return file_; }
};

int main() {
    rtti_example();
    
    Dog rex("Rex");
    Cat whiskers("Whiskers");
    Duck donald("Donald");
    
    std::cout << rex.speak() << "\n";
    std::cout << whiskers.speak() << "\n";
    donald.fly();
    donald.swim();
    
    return 0;
}
```

## 推荐书籍

1. **《C++ Primer》（第5版）** — Stanley Lippman
   第 7 章（类）、第 15 章（面向对象编程）、第 18 章（多重继承与虚继承），提供了系统而全面的 OOP 基础。

2. **《Effective C++》（第3版）** — Scott Meyers
   条款 32-40 专门讲解继承和面向对象设计：is-a 关系、继承 vs 组合、virtual 函数机制、非虚接口等。

3. **《A Tour of C++》（第3版）** — Bjarne Stroustrup
   第 4 章简明扼要地讲了 C++20 Module 的设计和使用，适合快速上手新编译模型。
