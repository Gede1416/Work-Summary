# TypeScript 错误处理

## 详细描述

TypeScript 的错误处理基于 JavaScript 的异常机制，但类型系统可以提供编译期的错误预防。在 `strict` 模式下，TS 能消除一大类与 `null`/`undefined` 相关的运行时错误。

**严格模式**（`strict: true`）是错误处理的第一道防线。它启用 `strictNullChecks`（使 `null`/`undefined` 独立于其他类型）、`noImplicitAny`、`strictFunctionTypes` 等子选项。在严格模式下，访问可能为 `null` 的值会在编译期报错，从源头消除了 NullPointerException 级别的错误。

**try-catch-finally** 与 JS 相同，但 TS 有一个重要限制：catch 捕获的异常是 `unknown` 类型（在 `useUnknownInCatchVariables` 选项下，TS 4.4+ 默认为 true）。这意味着你不能直接访问 `error.message`——必须先进行类型守卫。这是为了安全：JavaScript 中可以 throw 任何值，不仅限于 Error 实例。

**Promise 错误传播**：Promise 链中的错误会沿链向下传播到最近的 `.catch()` 或在 `async/await` 中被 try-catch 捕获。未处理的 Promise rejection 在 Node.js 中会触发 `unhandledRejection` 事件，可能导致进程崩溃。正确使用 `.catch()` 或 `try/catch` 包围 `await` 是必需的。

**类型级别禁止 any**：`any` 类型实际上是一种"错误处理的失败"——它关闭了编译器的所有检查，让运行时错误可以毫无察觉地溜进生产环境。ESLint 规则 `@typescript-eslint/no-explicit-any` 和 `@typescript-eslint/strict-boolean-expressions` 在类型级别上强制执行错误预防。

**结果类型模式**（Result Type）通过类型系统强制错误处理。定义一个 `Result<T, E>` 类型，成功的值封装在 `ok` 变体中，错误封装在 `err` 变体中。使用 discriminated union 进行模式匹配，编译器确保你处理了两种可能。

**防御性编程**：使用 `in` 运算符、`typeof` 守卫、`instanceof` 守卫和自定义类型谓词来在运行时安全地处理 unknown 类型的数据（如 API 响应、用户输入）。

## 代码实例

```typescript
// ========== 1. try-catch-finally + 类型守卫 ==========

async function fetchUserData(userId: string): Promise<{ name: string; email: string }> {
    try {
        const response = await fetch(`https://api.example.com/users/${userId}`);
        
        if (!response.ok) {
            throw new APIError(
                `Failed to fetch user: ${response.status}`,
                response.status,
            );
        }
        
        const data: unknown = await response.json();
        
        // 运行时验证 + 类型守卫
        if (!isUserData(data)) {
            throw new ValidationError("Invalid user data format", data);
        }
        
        return data;
    } catch (error) {
        // 使用 unknown（TS 4.4+）
        if (error instanceof APIError) {
            console.error(`API Error (${error.statusCode}): ${error.message}`);
            throw error;  // 重新抛出自定义错误
        }
        
        if (error instanceof ValidationError) {
            console.error("Validation Error:", error.message, error.rawData);
            throw error;
        }
        
        // 处理非 Error 的 throw 值
        const message = error instanceof Error ? error.message : String(error);
        console.error("Unknown error:", message);
        
        throw new ApplicationError(`Unexpected error fetching user ${userId}`, { cause: error });
    } finally {
        console.log(`User fetch attempt for ${userId} completed`);
    }
}

// 类型守卫：unknown → 具体类型
interface UserData {
    name: string;
    email: string;
}

function isUserData(data: unknown): data is UserData {
    return (
        typeof data === "object" &&
        data !== null &&
        "name" in data &&
        "email" in data &&
        typeof (data as Record<string, unknown>).name === "string" &&
        typeof (data as Record<string, unknown>).email === "string"
    );
}

// ========== 2. 自定义错误类 ==========

class ApplicationError extends Error {
    constructor(
        message: string,
        options?: { cause?: unknown },
    ) {
        super(message, options);
        this.name = this.constructor.name;
        Error.captureStackTrace?.(this, this.constructor);
    }
}

class APIError extends ApplicationError {
    constructor(
        message: string,
        public readonly statusCode: number,
    ) {
        super(message);
    }
}

class ValidationError extends ApplicationError {
    constructor(
        message: string,
        public readonly rawData: unknown,
    ) {
        super(message);
    }
}

// ========== 3. Result 类型模式 ==========

// 判别联合类型
type Result<T, E = Error> =
    | { ok: true; value: T }
    | { ok: false; error: E };

// 辅助函数
const Result = {
    ok<T>(value: T): Result<T, never> {
        return { ok: true, value };
    },
    err<E>(error: E): Result<never, E> {
        return { ok: false, error };
    },
};

// 使用 Result 的解析函数
function parseJSON<T>(text: string, validator: (data: unknown) => data is T): Result<T, SyntaxError | ValidationError> {
    try {
        const parsed: unknown = JSON.parse(text);
        if (!validator(parsed)) {
            return Result.err(new ValidationError("Invalid JSON structure", parsed));
        }
        return Result.ok(parsed);
    } catch (error) {
        if (error instanceof SyntaxError) {
            return Result.err(error);
        }
        throw error;  // 非预期错误仍然抛出
    }
}

// 使用 Result 类型
function demoResult(): void {
    const text = '{"name": "Alice", "age": "30"}';
    const result = parseJSON(text, isUserData);
    
    if (result.ok) {
        console.log("User:", result.value.name);  // 类型安全
    } else {
        console.error("Parse error:", result.error.message);
    }
}

// ========== 4. Promise 错误处理 ==========

// 安全的 async 包装器
async function safeAsync<T>(
    promise: Promise<T>,
): Promise<Result<T, Error>> {
    try {
        const value = await promise;
        return Result.ok(value);
    } catch (error) {
        return Result.err(
            error instanceof Error ? error : new Error(String(error)),
        );
    }
}

// Promise.all 带错误隔离
async function fetchAllSafe<T>(urls: string[]): Promise<Result<T, Error>[]> {
    return Promise.all(
        urls.map(url =>
            safeAsync(fetch(url).then(r => r.json() as Promise<T>))
        ),
    );
}

// ========== 5. 全局未处理 Rejection 处理 ==========
if (typeof process !== "undefined") {
    process.on("unhandledRejection", (reason: unknown, promise: Promise<unknown>) => {
        console.error("Unhandled Rejection at:", promise, "reason:", reason);
        // 在大多数应用中，应该记录并优雅退出
        // process.exit(1);
    });
    
    process.on("uncaughtException", (error: Error) => {
        console.error("Uncaught Exception:", error);
        // 未捕获的异常应该使进程退出
        process.exit(1);
    });
}

// ========== 6. 不要 any ==========
// ❌ any 隐藏错误
function badParse(input: string): any {
    return JSON.parse(input);  // 调用者获得 any，失去所有类型安全
}

// ✅ unknown 强制类型检查
function goodParse(input: string): unknown {
    return JSON.parse(input);
}
// 调用者必须进行类型守卫才能使用返回值
```

## 推荐书籍

1. **《Effective TypeScript》** — Dan Vanderkam
   条款 44-51 涵盖错误处理、unknown vs any、严格模式、类型守卫等 TS 特有的错误预防技术。

2. **《Programming TypeScript》** — Boris Cherny
   第 6 章详细讲解了 `unknown` 类型、类型守卫和 discriminated union。

3. **《Node.js Design Patterns》（第3版）** — Mario Casciaro
   第 1 章和第 9 章讨论了 Node.js 中的异步错误传播和 Promise 错误处理模式。
