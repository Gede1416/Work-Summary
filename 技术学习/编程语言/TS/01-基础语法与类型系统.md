# TypeScript 基础语法与类型系统

## 详细描述

TypeScript 的类型系统是其核心价值所在——在 JavaScript 的动态世界之上，构建了一套完整的静态类型层。理解这套类型系统是高效使用 TypeScript 的前提。

**基础类型**：`string`、`number`、`boolean`、`symbol`、`bigint` 是 TS 的原始类型，与 JS 运行时一一对应。`null` 和 `undefined` 既是值也是类型。`void` 表示函数无返回值，`never` 表示永远不会到达的类型（异常抛出或死循环）。

**顶层类型与底层类型**：`unknown` 是类型安全的"任意"类型——任何值都可以赋值给 `unknown`，但 `unknown` 不能赋值给其他类型（除非类型断言或类型守卫）。`any` 则完全关闭类型检查，应尽量避免。`never` 是所有类型的子类型，在类型运算中充当"空集"角色。

**联合类型与交叉类型**是 TS 类型系统的两大核心组合机制。联合类型（`A | B`）表示"要么是 A，要么是 B"，配合**字面量类型**（`"success" | "error"`）可以实现枚举式的精确约束。交叉类型（`A & B`）表示同时满足 A 和 B，常用于 mixin 模式。

**类型守卫**是在运行时缩小类型范围的关键技术。`typeof`、`instanceof`、`in` 运算符、自定义类型谓词函数（`arg is Type`），以及 discriminated union（通过共享的字面量属性区分变体），让类型在控制流中自动收窄。

**类型断言**（`as` 语法）告诉编译器"我知道这比你推断的更具体"，是不安全但有时必要的逃生口。`as const` 则将值收窄为其最精确的字面量类型。

**类型操作符**：
- `keyof T` 获取 T 所有属性名的字面量联合类型
- `typeof val` 在类型位置获取 `val` 的 TypeScript 类型
- `infer` 在条件类型中声明待推断的类型变量，是高级类型运算的关键
- `extends` 既是接口继承也是泛型约束和条件类型的关键字

**严格模式**（`strict: true`）是任何生产项目的基线配置，它启用了 `strictNullChecks`（null/undefined 独立于其他类型）、`noImplicitAny`、`strictFunctionTypes` 等子选项，是类型安全的基础保障。

## 代码实例

```typescript
// ========== 1. 基础类型与顶层/底层类型 ==========

// 基础类型
let name: string = "Alice";
let age: number = 30;
let isAdmin: boolean = true;
let id: symbol = Symbol("id");
let bigNumber: bigint = 9007199254740991n;

// void - 函数无返回值
function log(message: string): void {
    console.log(message);
}

// never - 永远不会到达
function throwError(message: string): never {
    throw new Error(message);
}

function infiniteLoop(): never {
    while (true) {}
}

// unknown vs any
let unknownValue: unknown = "hello";
// unknownValue.toUpperCase(); // ❌ 错误：unknown 不能直接使用
if (typeof unknownValue === "string") {
    unknownValue.toUpperCase(); // ✅ 类型守卫后安全使用
}

let anyValue: any = "hello";
anyValue.toUpperCase();  // ⚠️ 不报错但失去类型安全
anyValue.nonexistent();  // ⚠️ 运行时崩溃！

// ========== 2. 联合类型与交叉类型 ==========

// 联合类型
type Status = "pending" | "fulfilled" | "rejected";
type Result = 
    | { type: "success"; data: unknown }
    | { type: "error"; message: string };

// 交叉类型
interface Nameable { name: string; }
interface Ageable { age: number; }
type Person = Nameable & Ageable;  // { name: string; age: number }

const alice: Person = { name: "Alice", age: 30 };

// ========== 3. 类型守卫 ==========

// typeof 守卫
function processValue(value: string | number) {
    if (typeof value === "string") {
        return value.toUpperCase(); // 类型收窄为 string
    }
    return value.toFixed(2); // 类型收窄为 number
}

// 自定义类型谓词
function isPerson(obj: unknown): obj is Person {
    return (
        typeof obj === "object" &&
        obj !== null &&
        "name" in obj &&
        "age" in obj
    );
}

// discriminated union
type Shape =
    | { kind: "circle"; radius: number }
    | { kind: "rectangle"; width: number; height: number }
    | { kind: "triangle"; base: number; height: number };

function area(shape: Shape): number {
    switch (shape.kind) {
        case "circle":
            return Math.PI * shape.radius ** 2;
        case "rectangle":
            return shape.width * shape.height;
        case "triangle":
            return (shape.base * shape.height) / 2;
    }
}

// ========== 4. 类型操作符 ==========

// keyof
interface User {
    id: number;
    name: string;
    email: string;
}
type UserKey = keyof User; // "id" | "name" | "email"

// typeof 在类型位置
const config = {
    host: "localhost",
    port: 8080,
    debug: true,
};
type Config = typeof config;
// { host: string; port: number; debug: boolean }

// as const - 收窄为字面量类型
const colors = ["red", "green", "blue"] as const;
type Color = (typeof colors)[number]; // "red" | "green" | "blue"


/**
 * 帮助保证输入合法性
*/
// ========== 5. 字面量类型与模板字面量类型 ==========

// 字面量类型
type Direction = "north" | "south" | "east" | "west";
type DiceRoll = 1 | 2 | 3 | 4 | 5 | 6;

function move(direction: Direction, steps: number): void {
    console.log(`Moving ${steps} steps ${direction}`);
}

// 模板字面量类型（TS 4.1+）
type EventName<T extends string> = `on${Capitalize<T>}`;
type ClickEvent = EventName<"click">; // "onClick"

type Align = "start" | "end";
type Axis = "x" | "y";
type AlignClass = `align-${Align}-${Axis}`;
// "align-start-x" | "align-start-y" | "align-end-x" | "align-end-y"

// ========== 6. 枚举 ==========

// 数字枚举
enum HttpStatus {
    OK = 200,
    NotFound = 404,
    InternalServerError = 500,
}

// 字符串枚举（推荐：更易调试）
enum LogLevel {
    Debug = "DEBUG",
    Info = "INFO",
    Warn = "WARN",
    Error = "ERROR",
}

// const enum - 内联，零运行时开销
const enum Direction2 {
    Up,
    Down,
    Left,
    Right,
}
let dir = Direction2.Up;  // 编译后直接变为 0
```

## 推荐书籍

1. **《Programming TypeScript》** — Boris Cherny
   最适合系统性学习 TypeScript 的书籍，从基础类型到高级类型编程，覆盖类型运算、泛型、声明文件等，示例清晰实用。

2. **《Effective TypeScript》** — Dan Vanderkam
   以 62 个具体条款组织内容，涵盖类型系统、类型推导、any 的危害、类型声明、代码生成等实战话题，适合有一定 TS 基础后精读。

3. **《TypeScript 类型编程》**（TypeScript Cookbook）— Stefan Baumgartner
   聚焦 TS 类型系统的高级用法，包括条件类型、模板字面量类型、映射类型等，适合深入学习类型体操。

4. **《Learning TypeScript》** — Josh Goldberg
   面向初学者的友好入门书，覆盖 TS 全貌，从类型基础到 React/Node 项目中的应用，渐进式学习路径清晰。
