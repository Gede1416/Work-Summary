# TypeScript 泛型与高级类型

## 详细描述

TypeScript 的类型系统提供了业界最强大的泛型和类型运算能力。它是纯编译时的、零运行时开销的，允许开发者用类型表达极其复杂的约束和变换。

**泛型基础**：泛型参数用尖括号 `<T>` 声明。函数、类、接口、type 别名都支持泛型。与 C#/Java 不同，TS 泛型完全在编译期处理，编译产物中不包含任何类型相关的代码。

**泛型约束**通过 `extends` 限制类型参数的范围——`<T extends HasLength>` 表示 T 必须符合 HasLength 接口的形状。约束可以是 interface、type、甚至另一个泛型参数。与 C# 的 where 从句不同，TS 直接在类型参数后写 extends。

**条件类型**是 TS 类型系统的"if-else"：`T extends U ? X : Y`。如果 T 可以赋值给 U，类型为 X，否则为 Y。条件类型配合 `infer` 可以在类型层面"解构"类型：`type ReturnType<T> = T extends (...args: any[]) => infer R ? R : never`。条件类型的分配律特性（Distributive Conditional Types）——当 T 是联合类型时，条件类型自动分布到每个成员——是实现许多高级类型操作的基础。

**映射类型**可以基于已有类型创建新类型，类似类型层面的 `Array.prototype.map`：`{ [K in keyof T]: NewType }`。内置映射类型包括 `Partial<T>`（所有属性可选）、`Required<T>`、`Readonly<T>`、`Pick<T, K>`、`Record<K, V>`、`Omit<T, K>` 等。

**模板字符串字面量类型**（TS 4.1）允许在类型层面拼接字符串：`` type EventName<T extends string> = `on${Capitalize<T>}` ``。配合递归条件类型，可以实现类型层面的字符串解析、路由匹配、CSS 类名验证等。

**类型运算实现高级多态**：通过组合映射类型、条件类型、keyof、infer，TS 可以实现函数重载无法描述的复杂类型关系——比如"如果参数是 string 返回 number，如果参数是 number[] 返回 Map<string, number>"。这种类型级编程（类型体操）已成为 TS 高级开发者的标志性技能。

**变体**（Variance）：TS 的类型系统在默认情况下是"结构化"且"协变"的。函数参数位置是逆变的（在 strictFunctionTypes 下），这是类型安全的重要保障。

## 代码实例

```typescript
// ========== 1. 泛型基础 ==========

// 泛型函数
function first<T>(arr: T[]): T | undefined {
    return arr[0];
}

const n = first([1, 2, 3]);       // 类型: number | undefined
const s = first(["a", "b"]);      // 类型: string | undefined

// 泛型约束
interface HasLength {
    length: number;
}

function logLength<T extends HasLength>(item: T): T {
    console.log(item.length);
    return item;
}

logLength("hello");           // OK: string 有 length
logLength([1, 2, 3]);        // OK: array 有 length
// logLength(123);           // 错误: number 没有 length

// 多泛型参数 + 约束
function merge<T extends object, U extends object>(a: T, b: U): T & U {
    return { ...a, ...b };
}

const merged = merge(
    { name: "Alice" },
    { age: 30 }
);
// 类型: { name: string } & { age: number }

// ========== 2. 条件类型与 infer ==========

// 基本条件类型
type IsString<T> = T extends string ? "yes" : "no";
type A = IsString<string>;      // "yes"
type B = IsString<number>;      // "no"

// infer：推断类型变量
type ReturnType2<T> = T extends (...args: any[]) => infer R ? R : never;

type Fn = (x: number, y: string) => boolean;
type FnReturn = ReturnType2<Fn>;  // boolean

type ArrayElement<T> = T extends (infer E)[] ? E : never;
type NumElement = ArrayElement<number[]>;  // number

// 递归条件类型
type DeepReadonly<T> = {
    readonly [K in keyof T]: T[K] extends object
        ? DeepReadonly<T[K]>
        : T[K];
};

// 条件类型的分配律（Distributive）
type MyExclude<T, U> = T extends U ? never : T;
type Excluded = MyExclude<"a" | "b" | "c", "a" | "b">;  // "c"
// 内部流程：("a" extends "a"|"b" ? never : "a") | ("b" extends ...) | ("c" extends ...)

// ========== 3. 映射类型 ==========

interface Person {
    name: string;
    age: number;
    readonly email: string;
    phone?: string;
}

// 内置映射类型的实现原理
type MyPartial<T> = { [K in keyof T]?: T[K] };
type MyRequired<T> = { [K in keyof T]-?: T[K] };
type MyReadonly<T> = { readonly [K in keyof T]: T[K] };
type MyPick<T, K extends keyof T> = { [P in K]: T[P] };

// 自定义映射类型
type Nullable<T> = { [K in keyof T]: T[K] | null };
type NullablePerson = Nullable<Person>;

// 字段重命名
type Prefix<T, P extends string> = {
    [K in keyof T as `${P}${Capitalize<string & K>}`]: T[K];
};
type PrefixedPerson = Prefix<Person, "user">;
// { userName: string; userAge: number; ... }

// 条件映射
type OnlyFunctions<T> = {
    [K in keyof T as T[K] extends (...args: any[]) => any ? K : never]: T[K];
};

interface Mixed {
    name: string;
    greet(): void;
    age: number;
    calculate(x: number): number;
}

type FuncsOnly = OnlyFunctions<Mixed>;
// { greet: () => void; calculate: (x: number) => number }

// ========== 4. 模板字符串字面量类型 ==========

// 基础用法
type EventName<T extends string> = `on${Capitalize<T>}`;
type ClickEvent = EventName<"click">;  // "onClick"

// 路由参数类型
type ExtractRouteParams<S extends string> =
    S extends `${infer _Start}:${infer Param}/${infer Rest}`
        ? Param | ExtractRouteParams<`/${Rest}`>
        : S extends `${infer _Start}:${infer Param}`
            ? Param
            : never;

type Params = ExtractRouteParams<"/users/:userId/posts/:postId">;
// "userId" | "postId"

// CSS 实用类型
type CssUnit = "px" | "em" | "rem" | "%";
type CssValue = `${number}${CssUnit}`;

const width: CssValue = "100px";  // OK
// const bad: CssValue = "100abc"; // 错误！

// ========== 5. 高级类型体操 ==========

// 将联合类型转为元组（交叉类型分发）
type UnionToIntersection<U> =
    (U extends any ? (k: U) => void : never) extends (k: infer I) => void
        ? I
        : never;

type Intersected = UnionToIntersection<
    { a: string } | { b: number }
>;
// { a: string; b: number }

// 深度 Partial
type DeepPartial<T> = T extends object
    ? { [K in keyof T]?: DeepPartial<T[K]> }
    : T;

// 函数链式的管道类型
type Pipe<Fns extends [Function, ...Function[]]> = {
    [K in keyof Fns]: K extends "0"
        ? Fns[K] extends (arg: infer P) => any ? P : never
        : never;
}[0];

// （注：完整的 pipe 类型实现非常复杂，这里展示的是概念示例）
```

## 推荐书籍

1. **《Programming TypeScript》** — Boris Cherny
   第 4 章（Functions）、第 5 章（Generics）和第 7 章（Advanced Types）系统讲解了类型运算的全部内容。

2. **《Effective TypeScript》** — Dan Vanderkam
   条款 45-55 详细讨论了泛型、条件类型、映射类型的最佳实践和常见陷阱。

3. **《TypeScript Cookbook》** — Stefan Baumgartner
   包含了大量实际项目中用到的高级类型模式，如类型安全的 API 客户端、DSL 类型设计等。

4. **《Type Challenges》**（在线项目）— 社区维护
   一个通过解决类型体操题目来学习 TS 高级类型的开源项目，从简单到 Extremely Hard，是练习类型运算的绝佳资源。
