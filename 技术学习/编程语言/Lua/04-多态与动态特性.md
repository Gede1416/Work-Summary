# Lua 多态与动态特性

## 详细描述

Lua 没有静态类型系统，意味着它也没有传统意义上的泛型（Generics）或模板。但 Lua 通过其动态特性和元表机制，提供了另一种维度的灵活性和代码复用能力。

**鸭子类型**是 Lua 多态的基础——"如果它走起来像鸭子，叫起来像鸭子，那它就是鸭子"。Lua 不关心对象的"类型"，只关心对象是否实现了所需的方法。这种动态分派天然就是"运行时多态"的极致形式：任何两个对象只要实现了同名方法，就可以在多态上下文中互换使用。

**元表实现多态**：通过为不同 table 设置共享元表，可以使它们共享方法。改变元表就改变了对象的行为——这种动态性是静态类型语言难以比拟的。`__index` 实现了方法查找，`__call` 允许将 table 当作函数调用，`__add`/`__sub` 等实现运算符重载。

**接口式编程**（无 interface 关键字的接口）：Lua 程序员通过文档和约定定义"接口"——一组期望的方法签名。任何 table 只要实现了这些方法，就可以作为该"接口"的使用对象。这是 Go 语言的 interface 模式在动态语言中的极致简化。

**多重分派**（Multiple Dispatch）可以通过检查和手动分派实现，比单分派（仅根据 self 的类型分派）更灵活。常用于实现访问者模式、事件系统等。

**泛型的替代方案**：由于缺乏类型检查，Lua 的函数天然就是"泛型"的——任何函数都可以接受任何类型的参数。责任落在调用者身上：确保传入的参数支持所需操作。这是"动态语言的自由"也是"动态语言的风险"。

**LuaJIT 的类型特化**：虽然 Lua 源码没有类型，但 LuaJIT 的 JIT 编译器会在运行时刻对热点代码进行类型特化——如果某个函数总是收到 number 参数，JIT 会生成专门的 number 版本代码。这在一定程度上弥补了动态类型的性能损失。

**`__type` 元方法**（Lua 5.4 实验性）：允许自定义 `type()` 函数的返回值，使自定义"类型"（table 通过元表模拟的类）可以被 `type()` 识别。

## 代码实例

```lua
-- ========== 1. 鸭子类型多态 ==========

-- 没有继承关系，只要实现了 speak() 就可以
local dog = {
    name = "Rex",
    speak = function(self)
        return self.name .. " says: Woof!"
    end
}

local cat = {
    name = "Whiskers",
    speak = function(self)
        return self.name .. " says: Meow!"
    end
}

local robot = {
    id = "R2D2",
    speak = function(self)
        return self.id .. " says: Beep Boop!"
    end
}

-- 多态函数：不关心对象的"类型"
local function chorus(performers)
    for _, p in ipairs(performers) do
        print(p:speak())
    end
end

chorus({dog, cat, robot})
-- Rex says: Woof!
-- Whiskers says: Meow!
-- R2D2 says: Beep Boop!

-- ========== 2. 元表方法分派（运算符重载多态） ==========

-- 向量类型
local Vector = {}
Vector.__index = Vector

function Vector:new(x, y)
    local v = { x = x or 0, y = y or 0 }
    setmetatable(v, self)
    return v
end

function Vector.__add(a, b)
    return Vector:new(a.x + b.x, a.y + b.y)
end

function Vector.__sub(a, b)
    return Vector:new(a.x - b.x, a.y - b.y)
end

function Vector.__mul(a, b)
    -- 支持向量 * 标量
    if type(b) == "number" then
        return Vector:new(a.x * b, a.y * b)
    elseif type(a) == "number" then
        return Vector:new(b.x * a, b.y * a)
    end
    -- 支持点积
    return a.x * b.x + a.y * b.y
end

function Vector:__tostring()
    return string.format("Vector(%.1f, %.1f)", self.x, self.y)
end

function Vector:__eq(other)
    return self.x == other.x and self.y == other.y
end

local v1 = Vector:new(3, 4)
local v2 = Vector:new(1, 2)

print(v1 + v2)   --> Vector(4.0, 6.0)
print(v1 - v2)   --> Vector(2.0, 2.0)
print(v1 * 2)    --> Vector(6.0, 8.0)
print(v1 * v2)   --> 11（点积）
print(v1 == Vector:new(3, 4)) --> true

-- 复数类型也可以实现同样的元方法
local Complex = {}
Complex.__index = Complex

function Complex:new(r, i)
    local c = { r = r or 0, i = i or 0 }
    setmetatable(c, self)
    return c
end

function Complex.__add(a, b)
    return Complex:new(a.r + b.r, a.i + b.i)
end

function Complex:__tostring()
    return string.format("Complex(%s%si)", self.r, self.i >= 0 and "+" or "", self.i)
end

local c1 = Complex:new(3, 4)
local c2 = Complex:new(1, -2)
print(c1 + c2)  --> Complex(4+2i)

-- ========== 3. __call 元方法：把对象当函数调用 ==========

local Multiplier = {}
Multiplier.__index = Multiplier

function Multiplier:new(factor)
    local m = { factor = factor }
    setmetatable(m, self)
    return m
end

function Multiplier:__call(x)
    return x * self.factor
end

local double = Multiplier:new(2)
local triple = Multiplier:new(3)

print(double(10))   --> 20
print(triple(10))   --> 30
print(double("ha")) --> "haha"（也适用于字符串重复）

-- ========== 4. 接口式编程 ==========

-- 定义一个"文件接口"的约定
-- open(), read(), write(), close()

-- 内存文件（实现了接口）
local function createMemoryFile(initialContent)
    local content = initialContent or ""
    return {
        open = function() return true end,
        read = function(_, pattern)
            if pattern == "*a" then
                return content
            elseif pattern == "*l" then
                local line, rest = content:match("([^\n]*)\n?(.*)")
                content = rest
                return line
            end
            return nil
        end,
        write = function(_, data)
            content = content .. data
            return true
        end,
        close = function() return true end,
    }
end

-- 日志处理器：接受任何"类文件"对象
local function logProcessor(file)
    file:open()
    file:write("[INFO] Application started\n")
    file:write("[WARN] Low memory\n")
    file:write("[INFO] Application stopped\n")
    print("Content:", file:read("*a"))
    file:close()
end

local memFile = createMemoryFile()
logProcessor(memFile)  -- 多态：不关心具体是哪种文件

-- ========== 5. 方法检查（运行时接口验证） ==========

local function requires(obj, ...)
    for _, method in ipairs({...}) do
        if type(obj[method]) ~= "function" then
            error(string.format("Object missing required method: %s", method))
        end
    end
    return true
end

local function safelyLog(file)
    requires(file, "open", "read", "write", "close")
    file:open()
    file:write("Log entry\n")
    file:close()
end

-- ========== 6. 选择器模式（多态 + 策略） ==========

local SortStrategies = {
    asc = function(a, b) return a < b end,
    desc = function(a, b) return a > b end,
    byLength = function(a, b) return #a < #b end,
}

local function sortList(list, strategy)
    strategy = strategy or SortStrategies.asc
    table.sort(list, strategy)
end

local items = {"apple", "kiwi", "banana", "grape"}
sortList(items, SortStrategies.byLength)
-- {"kiwi", "apple", "grape", "banana"}  -- 按长度排序
```

## 推荐书籍

1. **《Programming in Lua》（第4版）** — Roberto Ierusalimschy
   第 13 章（Metatables and Metamethods）和第 16 章（OOP）详细讲解了通过元表实现的所有多态形式。

2. **《Lua Programming Gems》** — L.H. de Figueiredo 等
   第 1 章和第 5 章展示了如何在 Lua 中利用动态特性实现优雅的 API 设计和插件系统。

3. **《Lua Design Patterns》**（在线）— 社区
   收集了 Lua 中常见的多态模式实现，包括策略、观察者、装饰器等模式的 Lua 惯用写法。
