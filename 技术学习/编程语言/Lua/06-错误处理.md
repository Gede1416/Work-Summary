# Lua 错误处理

## 详细描述

Lua 没有 try-catch 式的异常机制，而是通过**保护调用**（pcall/xpcall）和错误对象来实现错误处理。这种设计源于 Lua 的极简主义——简单、可控、适合嵌入式场景。

**pcall（Protected Call）**是最核心的错误处理函数。`pcall(func, ...)` 以保护模式调用函数，返回两个值：`ok`（布尔值，表示成功/失败）和结果（成功时是函数返回值，失败时是错误对象）。关键：pcall 捕获错误后，调用栈不会被破坏，程序继续执行。这与 try-catch 类似。

**xpcall（Extended Protected Call）**允许指定错误处理函数，在错误发生时能获取调用栈信息。`xpcall(func, errorHandler)` 在 func 抛出错误时，先调用 errorHandler(errorObj) 处理错误（通常会记录调用栈），然后 xpcall 返回 false 和错误对象。

**error()** 抛出错误。可选的第二参数（level）控制调用栈报告的层级：level=0 和 level=1 报告 error 调用处，level=2 报告调用 error 的函数处。这个细节对错误定位很重要。

**assert()** 是 `error` 的便捷包装：`assert(val, message)` 如果 val 为 false 或 nil 则抛出错误并附带 message，否则返回 val。常用于前条件检查。

**无异常机制**意味着 Lua 中的错误处理完全依赖返回值模式——这既是优点（控制流清晰，没有"异常黑洞"）也是缺点（如果忘记检查返回值，错误会被静默忽略）。

**错误对象**可以是任何 Lua 值，但字符串最常见。复杂的错误对象（如包含错误码的 table）在大型应用中更实用。

**debug 库**在错误处理中也很重要：`debug.traceback` 生成调用栈字符串，`debug.getinfo` 获取函数信息。它们是构建良好错误消息的关键。

**应用模式**：Lua 程序通常在顶层使用 pcall 包裹主逻辑，实现类似"全局异常处理"的效果。在游戏脚本中，单个脚本文件的执行用 pcall 保护以确保一个脚本崩了不影响其他脚本。

## 代码实例

```lua
-- ========== 1. pcall 基础 ==========

-- 基本用法
local ok, result = pcall(function()
    return 10 / 2
end)
print(ok, result)  --> true, 5.0

-- 错误场景
local ok2, err = pcall(function()
    error("Something went wrong!")
end)
print(ok2, err)  --> false, test.lua:3: Something went wrong!

-- 带参数的 pcall
local function divide(a, b)
    if b == 0 then
        error("Division by zero", 2)
    end
    return a / b
end

local ok3, result3 = pcall(divide, 10, 0)
if not ok3 then
    print("Error:", result3)  --> Error: test.lua:...: Division by zero
end

-- ========== 2. xpcall + 错误处理函数 ==========

-- 错误处理函数：添加更丰富的上下文
local function errorHandler(err)
    local trace = debug.traceback("Error occurred:", 2)
    return string.format("[%s] %s\n%s", os.date("%Y-%m-%d %H:%M:%S"), tostring(err), trace)
end

local function riskyOperation(input)
    if type(input) ~= "number" then
        error("Expected number, got " .. type(input))
    end
    return math.sqrt(input)
end

local ok, result = xpcall(riskyOperation, errorHandler, -1)
if not ok then
    print(result)
    -- 输出类似：
    -- [2026-05-14 10:30:00] Expected number, got string
    -- stack traceback:
    --     test.lua:10: in function 'riskyOperation'
    --     ...
end

-- ========== 3. assert 用法 ==========

-- 简单断言
local function safeSqrt(x)
    assert(type(x) == "number", "Argument must be a number")
    assert(x >= 0, "Cannot compute square root of negative number")
    return math.sqrt(x)
end

-- assert 返回值
local function getConfig(key)
    local config = { host = "localhost", port = 8080 }
    return assert(config[key], "Missing config key: " .. key)
end

print(getConfig("host"))  --> "localhost"
-- print(getConfig("timeout")) -- 错误：Missing config key: timeout

-- ========== 4. 应用：顶层保护 ==========

-- 模块加载器（防止单模块错误拖垮整个系统）
local ModuleLoader = {}

function ModuleLoader.loadModules(moduleList)
    local loaded = {}
    local failed = {}
    
    for _, moduleName in ipairs(moduleList) do
        local ok, mod = pcall(require, moduleName)
        if ok then
            loaded[moduleName] = mod
            print("Loaded:", moduleName)
        else
            failed[moduleName] = mod  -- mod 是错误对象
            print("Failed:", moduleName, mod)
        end
    end
    
    return loaded, failed
end

-- ========== 5. 错误对象模式（带结构化数据） ==========

-- 复杂错误对象
local function createError(code, message, details)
    return {
        code = code,
        message = message,
        details = details or {},
        timestamp = os.time(),
        is_error = true,
        __tostring = function(self)
            return string.format("[%s] %s", self.code, self.message)
        end,
    }
end

local ERRORS = {
    INVALID_ARG = "ERR_INVALID_ARG",
    NOT_FOUND = "ERR_NOT_FOUND",
    TIMEOUT = "ERR_TIMEOUT",
}

local function findUser(userId)
    if type(userId) ~= "number" then
        error(createError(ERRORS.INVALID_ARG, "userId must be a number", {
            provided = type(userId),
        }))
    end
    
    -- 模拟数据库查询
    local users = { [1] = "Alice", [2] = "Bob" }
    local user = users[userId]
    if not user then
        error(createError(ERRORS.NOT_FOUND, "User not found", {
            userId = userId,
        }))
    end
    
    return user
end

-- 处理结构化错误
local ok, result = pcall(findUser, "abc")
if not ok then
    local err = result
    if type(err) == "table" and err.is_error then
        print("Error code:", err.code)
        print("Message:", err.message)
        if err.code == ERRORS.INVALID_ARG then
            print("Provided type:", err.details.provided)
        end
    else
        print("Unexpected error:", err)
    end
end

-- ========== 6. 安全文件读取模式 ==========

local function safeReadFile(path)
    local file, openErr = io.open(path, "r")
    if not file then
        return nil, string.format("Cannot open %s: %s", path, openErr)
    end
    
    -- 用 pcall 保护文件读取
    local ok, content = pcall(function()
        return file:read("*a")
    end)
    
    file:close()
    
    if not ok then
        return nil, string.format("Failed to read %s: %s", path, content)
    end
    
    return content
end

local content, err = safeReadFile("nonexistent.txt")
if not content then
    print("Warning:", err)  -- 温和地处理错误
    content = ""  -- 使用默认值
end
```

## 推荐书籍

1. **《Programming in Lua》（第4版）** — Roberto Ierusalimschy
   第 8 章（Compilation, Execution, and Errors）和第 23 章（The Debug Library）是 Lua 错误处理的权威参考。

2. **《Lua Programming Gems》** — L.H. de Figueiredo
   第 7 章展示了利用 pcall/xpcall 构建的健壮的错误报告系统。

3. **《Lua 5.4 Reference Manual》**
   第 2.4 节和第 6.10 节精确定义了 error、pcall、xpcall、assert 的语义和边界行为。
