# Lua 面向对象与模块化

## 详细描述

Lua 没有内置的 class 关键字，但通过 **table 和元表（metatable）** 的组合，可以优雅地模拟出面向对象的各种特性。理解 Lua 的"类"实现机制是掌握这门语言的关键。

**原型式继承**是 Lua OOP 的核心。通过设置元表的 `__index` 字段，可以让一个 table 在找不到某个键时自动去另一个 table（原型）中查找。这种设计非常轻量，一个对象就是一个 table，它的"类"就是它的元表——这种模式比经典的 class-instance 更加灵活。

**模拟类的标准模式**：定义一个 table 作为类模板（持有类方法），创建一个工厂函数或构造函数来生成实例（每个实例是独立的 table，其元表的 `__index` 指向类模板）。实例的字段直接存储在自身 table 中，方法通过 `__index` 从类模板中查找。

**self 关键字**（冒号语法糖）：`obj:method(params)` 等价于 `obj.method(obj, params)`。使用冒号定义方法时，编译器会自动添加 `self` 作为第一个隐式参数。这是 Lua OOP 风格的基石——`self` 就是实例本身。

**继承**通过设置类模板的 `__index` 元表指向父类实现。子类创建实例时，实例的 `__index` 查找链是：实例 → 子类 → 父类。这样构成了完整的原型链。

**多重继承**可以通过在 `__index` 中实现自定义查找逻辑（遍历多个父类）来实现。虽然这种技术不常用，但展示了 Lua 元表机制的灵活性。

**模块系统**非常简单直接：一个 Lua 文件（模块）就是一个返回 table 的函数（或直接返回 table）。调用 `require("module")` 时，Lua 查找路径中的文件，加载并执行它，将其返回值缓存到 `package.loaded` 表中。相同模块的后续 `require` 直接返回缓存的 table，不会重复执行。

**`package.loaded` 缓存机制**带来了单例行为——模块只加载一次的保障。如果需要热重载，只需 `package.loaded[name] = nil` 清空缓存后再次 `require`。

**面向对象 vs 函数式**：Lua 的社区实践在不同生态中差异很大。游戏脚本（如 WoW）中 OOP 风格更常见，而 Nginx/OpenResty 场景中函数式/数据驱动风格更流行。没有"正确"的风格，关键是理解团队和项目的惯例。

## 代码实例

```lua
-- ========== 1. 基础类实现 ==========

-- 类模板
local Animal = {}
Animal.__index = Animal  -- 关键：让实例能查找到方法

-- 构造函数
function Animal:new(name)
    local instance = {
        name = name,
        age = 0,
    }
    setmetatable(instance, self)  -- self 指向 Animal
    return instance
end

-- 方法（用冒号定义，自动接收 self）
function Animal:speak()
    return self.name .. " makes a sound"
end

function Animal:getOlder(years)
    self.age = self.age + (years or 1)
end

-- 使用
local a = Animal:new("Generic")
print(a:speak())       --> "Generic makes a sound"
a:getOlder(5)
print(a.age)           --> 5

-- ========== 2. 继承 ==========

-- 子类
local Dog = {}
setmetatable(Dog, { __index = Animal })  -- Dog 继承 Animal
Dog.__index = Dog  -- Dog 实例查找方法时先查 Dog

function Dog:new(name, breed)
    -- 调用父类构造函数
    local instance = Animal.new(self, name)  -- self 指向 Dog
    instance.breed = breed or "mutt"
    setmetatable(instance, self)
    return instance
end

-- 重写方法
function Dog:speak()
    return self.name .. " says: Woof!"
end

-- 新增方法
function Dog:fetch()
    return self.name .. " fetches the ball"
end

-- 调用父类方法
function Dog:speakLoudly()
    return Animal.speak(self) .. " LOUDLY!"  -- 显式调用父类
end

local dog = Dog:new("Rex", "German Shepherd")
print(dog:speak())          --> "Rex says: Woof!"
print(dog:fetch())          --> "Rex fetches the ball"
print(dog:speakLoudly())    --> "Rex makes a sound LOUDLY!"

-- 类型检查
print(dog.__index == Dog)           --> true
print(getmetatable(dog).__index == Dog) --> true

-- ========== 3. 多重继承 ==========

local Flyable = {}
Flyable.__index = Flyable

function Flyable:fly()
    return self.name .. " is flying"
end

local Swimmable = {}
Swimmable.__index = Swimmable

function Swimmable:swim()
    return self.name .. " is swimming"
end

-- 多重继承的 __index 函数
local function createMultiIndex(...)
    local parents = {...}
    return function(table, key)
        for _, parent in ipairs(parents) do
            local value = parent[key]
            if value ~= nil then return value end
        end
        return nil
    end
end

local Duck = {}
Duck.__index = Duck

function Duck:new(name)
    local instance = { name = name }
    setmetatable(instance, { __index = createMultiIndex(Duck, Flyable, Swimmable) })
    return instance
end

function Duck:speak()
    return self.name .. " says: Quack!"
end

local donald = Duck:new("Donald")
print(donald:speak())  --> "Donald says: Quack!"
print(donald:fly())    --> "Donald is flying"
print(donald:swim())   --> "Donald is swimming"

-- ========== 5. 高级 OOP 模式 ==========

-- 私有状态（通过闭包）
local function createPerson(name, age)
    -- 私有变量
    local _name = name
    local _age = age
    
    local self = {}
    
    function self.getName()
        return _name
    end
    
    function self.getAge()
        return _age
    end
    
    function self.birthday()
        _age = _age + 1
        return _age
    end
    
    return self
end

local p = createPerson("Alice", 30)
print(p.getName())     --> "Alice"
print(p.getAge())      --> 30
p.birthday()
print(p.getAge())      --> 31
-- print(p._age)       -- nil（真正的私有）

-- ========== 6. 热重载模块 ==========
local function hotReload(moduleName)
    package.loaded[moduleName] = nil  -- 清缓存
    return require(moduleName)         -- 重新加载
end
```

## 推荐书籍

1. **《Programming in Lua》（第4版）** — Roberto Ierusalimschy
   第 16 章（Object-Oriented Programming）和第 21 章（Modules and Packages）是学习 Lua OOP 和模块化的权威教材，从最基础的类实现到多重继承都有详尽讲解。

2. **《Lua Design Patterns》**（在线资源）— 社区维护
   收集了 Lua 社区的各种 OOP 实现模式，包括 class.lua、30log 等轻量级类库的设计和使用。

3. **《Lua Quick Start Guide》** — Gabor Szauer
   第 5-6 章提供了模块组织和 OOP 的实战演练，特别适合有主流语言背景需要快速在 Lua 中建立 OOP 思维的读者。
