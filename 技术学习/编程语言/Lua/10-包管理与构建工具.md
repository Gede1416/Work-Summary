# Lua 包管理与构建工具

## 详细描述

Lua 的包管理生态相对简单，核心工具是 **LuaRocks**，辅以待嵌入式场景的手动管理方式。

**LuaRocks** 是 Lua 的官方包管理器，类似于 Python 的 pip 或 Node.js 的 npm。它托管在 luarocks.org，提供数千个 Lua 模块（包括纯 Lua 和 C 扩展）。`luarocks install <module>` 下载、编译（如果需要 C 扩展）并安装模块到系统或用户目录。

**rockspec** 文件是包的元数据描述，包含名称、版本、依赖、源文件列表、构建指令等。通过 `luarocks pack` 和 `luarocks upload` 发布到 luarocks.org。

**手动文件拷贝**仍是 Lua 生态中常见的做法——直接将 `.lua` 文件放到项目的 `lib/` 或 `lua/` 目录下，通过 `LUA_PATH` 环境变量或 `package.path` 调整模块搜索路径。这在嵌入场景（游戏引擎、Nginx）中尤其常见。

**嵌入时的加载器控制**：当 Lua 嵌入到 C/C++ 程序中时，宿主程序完全控制模块的加载路径和方式。可以通过修改 `package.path`（Lua 模块搜索路径）和 `package.cpath`（C 扩展搜索路径），或者直接用 C API 实现自定义的 searcher 函数。

**版本管理**：Lua 生态没有内建的版本锁定机制（没有类似 package-lock.json 的东西）。最佳实践是使用 LuaRocks 的 `rockspec` 或外部工具（如 Makefile、CMake）来固定版本。

**Lua 版本差异**注意：LuaRocks 的包可能只支持特定 Lua 版本。LuaJIT vs 标准 Lua 5.1/5.4 之间的兼容性差异是最常见的坑。

## 代码实例

```lua
-- ========== 1. LuaRocks rockspec 文件 ==========
-- mymodule-1.0-1.rockspec

--[[
package = "mymodule"
version = "1.0-1"
source = {
    url = "https://github.com/user/mymodule/archive/v1.0.tar.gz",
}
description = {
    summary = "A sample Lua module",
    detailed = [[
        This module provides useful utilities for Lua applications.
    ]],
    license = "MIT",
    homepage = "https://github.com/user/mymodule",
}
dependencies = {
    "lua >= 5.1, <= 5.4",
    "luasocket ~> 3.0",
}
build = {
    type = "builtin",
    modules = {
        ["mymodule"] = "src/mymodule.lua",
        ["mymodule.utils"] = "src/utils.lua",
        ["mymodule.advanced"] = "src/advanced.lua",
    },
    install = {
        bin = { ["scripts/tool.lua"] = "mytool" },
    },
}
--]]

-- ========== 2. 包路径管理 ==========

-- 查看当前搜索路径
print("Lua package.path:")
for path in package.path:gmatch("[^;]+") do
    print("  " .. path)
end

-- 添加自定义搜索路径
-- 方式 1：代码中修改
package.path = package.path .. ";./lib/?.lua;./lib/?/init.lua"

-- 方式 2：环境变量（推荐）
-- export LUA_PATH="./lib/?.lua;./lib/?/init.lua;;"

-- ========== 3. 模块加载器自定义 ==========

-- 自定义 searcher：从缓存目录加载
local function cacheSearcher(moduleName)
    local cacheDir = "./cache/"
    local pattern = cacheDir .. string.gsub(moduleName, "%.", "/")
    
    local paths = {
        pattern .. ".lua",
        pattern .. "/init.lua",
    }
    
    for _, path in ipairs(paths) do
        local file = io.open(path, "r")
        if file then
            local content = file:read("*a")
            file:close()
            local chunk, err = load(content, path)
            if chunk then
                return chunk, path
            end
        end
    end
    
    return "\n\tno cache file found"
end

-- 插入自定义 searcher（在默认 searcher 之前）
table.insert(package.searchers or package.loaders, 2, cacheSearcher)

-- ========== 4. 依赖管理实践 ==========

-- 简单的依赖检查器
local function checkDependencies(requiredModules)
    local missing = {}
    for _, moduleName in ipairs(requiredModules) do
        local ok, result = pcall(require, moduleName)
        if not ok then
            table.insert(missing, moduleName)
        end
    end
    
    if #missing > 0 then
        error("Missing modules: " .. table.concat(missing, ", ") ..
              "\nInstall with: luarocks install <module>")
    end
end

-- 在应用入口检查依赖
checkDependencies({"luasocket", "lpeg", "dkjson"})

-- ========== 5. 项目结构最佳实践 ==========
--[[
my-lua-project/
├── src/
│   ├── main.lua          -- 入口文件
│   └── myproject/
│       ├── init.lua       -- 模块初始化
│       ├── core.lua       -- 核心逻辑
│       └── utils.lua      -- 工具函数
├── lib/                   -- 第三方依赖（手动管理）
│   └── vendor_module/
├── rockspecs/             -- LuaRocks 打包配置
│   └── myproject-1.0-1.rockspec
├── test/
│   └── test_core.lua
├── scripts/
│   └── build.lua
├── Makefile
├── .luacheckrc           -- luacheck 配置（linter）
└── README.md
--]]

-- ========== 6. 简单的构建脚本 ==========
-- scripts/build.lua

-- 复制文件到构建目录
local function buildProject()
    local lfs = require("lfs")  -- LuaFileSystem

    local function ensureDir(path)
        lfs.mkdir(path)
    end

    local function copyDir(src, dest)
        ensureDir(dest)
        for file in lfs.dir(src) do
            if file ~= "." and file ~= ".." then
                local srcPath = src .. "/" .. file
                local destPath = dest .. "/" .. file
                local attr = lfs.attributes(srcPath)
                if attr.mode == "directory" then
                    copyDir(srcPath, destPath)
                else
                    local srcFile = io.open(srcPath, "rb")
                    local content = srcFile:read("*a")
                    srcFile:close()
                    local destFile = io.open(destPath, "wb")
                    destFile:write(content)
                    destFile:close()
                end
            end
        end
    end

    print("Building project...")
    copyDir("src", "build")
    copyDir("lib", "build/lib")
    print("Build complete.")
end

buildProject()
```

## 推荐书籍

1. **《Programming in Lua》（第4版）** — Roberto Ierusalimschy
   第 21 章（Modules and Packages）和第 24 章（The External World）涵盖了 require 机制和包管理。

2. **LuaRocks 官方文档**（luarocks.org）
   学习创建 rockspec、管理依赖、发布包和私有 rocks 服务器的搭建。

3. **《Lua 5.4 Reference Manual》**
   第 6.3 节（Modules）精确定义了 `require` 和 `package` 库的行为。
