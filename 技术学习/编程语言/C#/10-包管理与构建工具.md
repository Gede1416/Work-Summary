# C# 包管理与构建工具

## 详细描述

.NET 的构建和包管理生态系统高度统一，围绕 `dotnet CLI`、`MSBuild` 和 `NuGet` 构建。这是 .NET 平台的一大优势——几乎所有项目使用相同的工具链。

**NuGet** 是 .NET 的官方包管理器。托管在 nuget.org 上的包通过 NuGet 客户端安装到项目中。包引用既可以写在 `.csproj` 文件中（`<PackageReference>`），也可以通过 dotnet CLI 管理（`dotnet add package Newtonsoft.Json`）。NuGet 支持语义版本（SemVer）、依赖传递、包还原（restore）、缓存（global packages folder）。

**dotnet CLI** 是统一的命令行界面。核心命令：
- `dotnet new`：从模板创建项目（console、webapi、mvc、classlib、xunit 等）
- `dotnet build`：编译项目
- `dotnet run`：编译并运行
- `dotnet test`：运行测试
- `dotnet publish`：发布可部署的输出
- `dotnet restore`：恢复 NuGet 包
- `dotnet add package/remove package`：管理包引用
- `dotnet tool`：管理全局或本地工具

**MSBuild** 是 .NET 的构建引擎，`.csproj` 文件就是 MSBuild 项目文件。SDK 风格项目（.NET Core+）比传统 .NET Framework 项目简洁得多——一个只有几行的 `.csproj` 即可定义完整的构建配置。

**项目文件**（`.csproj`）的结构：
- `<TargetFramework>` 指定目标框架（`net9.0`、`net8.0-windows`）
- `<PackageReference>` 声明 NuGet 依赖
- `<ProjectReference>` 引用同一解决方案中的其他项目
- `<Nullable>` 启用可空引用类型
- `<ImplicitUsings>` 自动导入常用命名空间

**解决方案**（`.sln`）包含多个项目的集合。`dotnet sln add/remove` 管理解决方案中的项目。

**中央包管理**（Central Package Management, .NET 8+）：在 `Directory.Packages.props` 文件中统一定义包版本，各项目只写包名不写版本号。

## 代码实例

```xml
<!-- ========== 项目文件示例 (.csproj) ========== -->
<Project Sdk="Microsoft.NET.Sdk.Web">

  <PropertyGroup>
    <TargetFramework>net9.0</TargetFramework>
    <Nullable>enable</Nullable>
    <ImplicitUsings>enable</ImplicitUsings>
    <RootNamespace>MyApp</RootNamespace>
    <AssemblyName>MyApp</AssemblyName>
    <Version>1.2.3</Version>
  </PropertyGroup>

  <!-- NuGet 包引用 -->
  <ItemGroup>
    <PackageReference Include="Microsoft.AspNetCore.OpenApi" Version="9.0.*" />
    <PackageReference Include="Microsoft.EntityFrameworkCore.SqlServer" Version="9.*" />
    <PackageReference Include="Serilog.AspNetCore" Version="8.*" />
  </ItemGroup>

  <!-- 项目引用 -->
  <ItemGroup>
    <ProjectReference Include="..\MyApp.Core\MyApp.Core.csproj" />
    <ProjectReference Include="..\MyApp.Infrastructure\MyApp.Infrastructure.csproj" />
  </ItemGroup>

  <!-- 文件排除 -->
  <ItemGroup>
    <Compile Remove="Generated\**" />
    <Content Remove="Generated\**" />
  </ItemGroup>

</Project>
```

```xml
<!-- ========== 中央包版本管理 (Directory.Packages.props) ========== -->
<Project>
  <PropertyGroup>
    <ManagePackageVersionsCentrally>true</ManagePackageVersionsCentrally>
  </PropertyGroup>
  
  <ItemGroup>
    <PackageVersion Include="Microsoft.EntityFrameworkCore" Version="9.0.0" />
    <PackageVersion Include="Serilog" Version="4.0.0" />
    <PackageVersion Include="FluentValidation" Version="11.9.0" />
  </ItemGroup>
</Project>
```

```bash
# ========== dotnet CLI 常用命令 ==========

# 创建新项目
dotnet new webapi -n MyApi
dotnet new classlib -n MyLib
dotnet new xunit -n MyTests

# 管理 NuGet 包
dotnet add package Microsoft.EntityFrameworkCore
dotnet add package --version 3.1.0 Newtonsoft.Json
dotnet remove package Newtonsoft.Json
dotnet list package --outdated

# 构建
dotnet build
dotnet build --configuration Release
dotnet build --no-restore

# 测试
dotnet test
dotnet test --filter "FullyQualifiedName~UnitTests"
dotnet test --collect:"XPlat Code Coverage"

# 发布
dotnet publish -c Release -o ./publish
dotnet publish -r win-x64 --self-contained true

# 工具管理
dotnet tool install --global dotnet-ef
dotnet tool update --global dotnet-ef
dotnet ef migrations add InitialCreate
dotnet ef database update
```

## 推荐书籍

1. **《C# 12 in a Nutshell》** — Joseph Albahari
   第 1 章详细介绍了 dotnet CLI、项目结构和构建流程。

2. **《MSBuild Trickery》**（在线资源）— 社区维护
   详解 MSBuild 的高级特性：自定义 Target、Property、ItemGroup 的表达式。

3. **NuGet 官方文档**（在线，learn.microsoft.com/nuget）
   覆盖包创建、版本管理、nuget.config、私有源等。
