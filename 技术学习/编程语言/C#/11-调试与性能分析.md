# C# 调试与性能分析

## 详细描述

.NET 生态的调试和性能分析工具与 Visual Studio 深度集成，同时也有强大的命令行工具支持 CI/CD 和服务器环境。

**Visual Studio 调试器**是最强大的 .NET 调试工具。断点（条件断点、数据断点、跟踪点）、单步执行（逐语句 F11 / 逐过程 F10）、调用堆栈、局部变量/自动窗口、即时窗口（Immediate Window）实时执行 C# 表达式、并行堆栈窗口可视化多线程执行、线程窗口管理线程状态。

**dotnet-trace**（.NET Core 3+）是命令行性能采样工具，基于 EventPipe（跨平台高性能事件管道）。`dotnet-trace collect --process-id <pid>` 收集运行时事件（GC、JIT、线程池、异常等），输出为 `.nettrace` 文件，可在 Visual Studio 或 PerfView 中查看。

**dotnet-dump** 用于分析和调试核心转储（core dump）。`dotnet-dump collect -p <pid>` 创建转储，`dotnet-dump analyze <dump>` 进入交互式分析——查看线程、堆状态、内存使用等。

**PerfView**（Windows）是 .NET 性能分析的专业工具，可以分析 ETW 事件，生成 GC 分配火焰图、JIT 编译时间、锁竞争等报告。

**BenchmarkDotNet** 是 .NET 基准测试的标准库。通过 `[Benchmark]` 标记测试方法，自动处理 JIT 预热、GC 清理、多次迭代统计、内存分配测量、硬件计数器。是验证性能优化假设的正确方式。

## 代码实例

```csharp
// ========== BenchmarkDotNet 基准测试 ==========
using BenchmarkDotNet.Attributes;
using BenchmarkDotNet.Running;

[MemoryDiagnoser]  // 测量内存分配
[SimpleJob(RuntimeMoniker.Net90)]
public class StringBenchmarks
{
    private string _text = "Hello, World! This is a test string for benchmarking.";
    
    [Benchmark(Baseline = true)]
    public string ReplaceWithString()
    {
        return _text.Replace(" ", "-");
    }
    
    [Benchmark]
    public string ReplaceWithSpan()
    {
        return string.Create(_text.Length, _text, (span, state) =>
        {
            for (int i = 0; i < state.Length; i++)
            {
                span[i] = state[i] == ' ' ? '-' : state[i];
            }
        });
    }
}

// ========== 运行方式 ==========
// BenchmarkRunner.Run<StringBenchmarks>();
// 或命令行：dotnet run -c Release -- --filter *StringBenchmarks*
```

```bash
# ========== dotnet-trace 命令行使用 ==========
# 收集运行时事件（60 秒）
dotnet-trace collect --process-id <pid> --duration 00:00:60

# 收集特定事件（GC、异常）
dotnet-trace collect -p <pid> --providers Microsoft-Windows-DotNETRuntime:4:5:4

# 转换为 speedscope 格式（火焰图）
dotnet-trace convert --format speedscope trace.nettrace

# ========== dotnet-dump 使用 ==========
# 创建核心转储
dotnet-dump collect -p <pid>

# 分析转储
dotnet-dump analyze dump.dmp
# > clrstack          -- 查看托管调用栈
# > dumpheap -stat    -- 按类型统计堆上的对象
# > gcroot -all <addr> -- 查看对象的 GC 根引用链

# ========== dotnet-counters 实时监控 ==========
dotnet-counters monitor -p <pid> --refresh-interval 1 \
    System.Runtime[dotnet.gc.pause_time] \
    System.Runtime[alloc-rate] \
    System.Runtime[threadpool-thread-count]
```

## 推荐书籍

1. **《Pro .NET Memory Management》** — Konrad Kokosa
   第 14 章专门讲性能分析工具（dotnet-trace、PerfView、BenchmarkDotNet）的实战使用。

2. **《Writing High-Performance .NET Code》（第2版）** — Ben Watson
   全书围绕 BenchmarkDotNet 和 PerfView 展开性能优化案例。

3. **BenchmarkDotNet 官方文档**（benchmarkdotnet.org）
   覆盖基准测试的所有特性：参数化、硬件计数器、内存诊断、反汇编诊断。
