# 英雄渡劫业务流程

## 1. 概述

**渡劫**是英雄等级突破系统，挂载在建筑 **E_116（渡劫台）** 上。英雄积累经验达到条件后，前往渡劫台完成渡劫，进阶到下一个境界（等级+1）。

## 2. 核心文件

| 文件 | 职责 |
|------|------|
| `HeroBreakPageController.cs` | 渡劫主页面 — 管理英雄列表、分类排序 |
| `HeroBreakPageItemController.cs` | 渡劫列表项 — 展示单个英雄渡劫状态、点击交互 |
| `HeroSystem_LvUp.cs` | 渡劫核心逻辑 — 发起渡劫请求、排队、处理完成回调 |
| `HeroSystem_Net.cs` | 网络层 — `RequestHeroLevelUp` 发送渡劫协议 |
| `HeroController.cs` | 英雄行为树 — `ForceHeroLevelUp` 强制前往渡劫台 |
| `HeroLevelUpResultDialogController.cs` | 渡劫结果弹窗 — 展示前后属性对比 |
| `EventBuilding.cs` | 事件定义 — `OnHeroLevelUpStart` / `OnForceHeroLevelUpStart` / `OnHeroLevelUpDone` |
| `CfgExt_HeroLevelupCollection.cs` | 配置扩展 — `GetCfgByLevel(level)` 查询等级对应的渡劫配置 |

## 3. 渡劫状态枚举（`HeroLevelUpType`）

| 状态 | 含义 | 对应 Prefab |
|------|------|-------------|
| `Ready` | 可渡劫（满足条件，可点击） | ItemPrefab_ready |
| `Unlocked` | 可渡劫但不可点击（等级已达建筑上限） | ItemPrefab_unlock |
| `Ing` | 渡劫中（正在读条） | ItemPrefab_ing |
| `InQueue` | 排队等待渡劫中 | ItemPrefab_inQueue |

列表展示顺序：**Ready > Unlocked > Ing > InQueue**

## 4. 英雄进入渡劫列表的条件

**Step 1 — 经验门槛检查（`ResetHeroDatas`）**

遍历所有活跃英雄，根据稀有度（Rarity 3/4/5）检查当前等级对应的 `hero_levelup` 配置：

```
Rarity = 3 → Exp >= levelCfg.Rarity3HeroExp
Rarity = 4 → Exp >= levelCfg.Rarity4HeroExp
Rarity = 5 → Exp >= levelCfg.Rarity5HeroExp
```

**Step 2 — 状态分类（`AddHeroData`）**

满足经验条件的英雄，按 `E_HeroAction` 和建筑等级上限进一步分类：

```
HeroAction == HeroActionHeroLevelUpQueue  → InQueue（已在排队）
HeroAction == HeroActionHeroLevelUpIng    → Ing（正在渡劫）
HeroLevel < P1306 + 建筑属性加成           → Ready（可渡劫）
其他                                      → Unlocked（建筑等级限制）
```

- `P1306` 是基础等级上限配置
- 建筑属性加成 = `GetAttributePercent(建筑属性类型) / 100 * 建筑属性值`

## 5. 完整业务流程

```
[1] 玩家点击 渡劫台(E_116) → 建筑面板加载 "渡劫" Tab
        ↓
[2] HeroBreakPageController.Create()
    ├─ ResetHeroDatas() → 遍历活跃英雄、分类到4个列表
    ├─ noHeroText 显隐控制（无可展示英雄时显示）
    └─ scroll.InitListView() → 按展示顺序渲染列表
        ↓
[3] OnGetItemByIndex() 按索引从4个列表中顺序取英雄
    → 根据 HeroLevelUpType 加载对应 ItemPrefab
    → itemScript.SetItemData(index, heroData, type, startTime, endTime)
        ↓
[4] 玩家点击 Ready 状态的 Item
    ├─ Ing/InQueue 状态 → Toast "正在渡劫中" (text_id 900010)
    ├─ Unlocked 状态 → Toast (text_id 900009)
    └─ Ready 状态:
        ├─ 发送 OnForceHeroLevelUpStart 事件（从列表移除）
        └─ 调用 heroController.ForceHeroLevelUp()
            ↓
[5] HeroController 行为树（HeroController.cs:1044-1059）:
    BtSequence:
      ├─ IsForceHeroLevelUp → 检查 isForceHeroLevelUp 标志
      ├─ MovingToTargetBuildingPos(E_116) → 移动到渡劫台
      │   └─ 同时设置 E_HeroAction = HeroActionHeroLevelUpGo
      └─ IsLevelUpping → 到达后触发升级
            ↓
[6] IsLevelUpping() → HeroSystem.HeroLvUp(heroData)
    ├─ 已有人渡劫中? (GetHeroCountOfSpecificAction(Ing) + RequestingLevelUpHeroCount > 0)
    │   └─ SetHeroAction(HeroActionHeroLevelUpQueue) → 排队等待
    └─ 无人渡劫?
        └─ RequestHeroLevelUp(heroUuid) → 发送协议
            ↓
[7] 服务端回包 SCHeroLevelUpRsp
    ├─ heroData.protoData.ActionStartTime = rsp.StartTime
    ├─ heroData.protoData.ActionEndTime = rsp.EndTime
    ├─ SetHeroAction(HeroActionHeroLevelUpIng)
    ├─ G.userData.buildingContainer.LevelUpHeroUUid = heroUuid
    └─ 发送 OnHeroLevelUpStart 事件 → HeroBreakPage.RefreshView()
        ↓
[8] HeroBreakPageItemController.Update()
    └─ 每帧 RefreshCountDown() → 更新进度条和倒计时显示
        ↓
[9] 服务端推送 MscSyncHeroLevelUpDone → OnCSyncHeroLevelUpDone()
    ├─ SetHeroAction(HeroActionHeroLevelUpDone)
    ├─ LevelUpHeroUUid = 0
    ├─ heroData.protoData.Level = msgBody.NewLevel（等级+1）
    ├─ heroData.protoData.Combat = msgBody.Combat（更新战力）
    ├─ 弹出 HeroLevelUpResultDialogController
    │   └─ 展示前后境界、HP/ATK/DEF 对比
    ├─ 若 heroController.isForceHeroLevelUp → 重置为 false
    └─ 发送 OnHeroLevelUpDone 事件 → HeroBreakPage.RefreshView()
        ↓
[10] 下一排队英雄自动开始渡劫（重复步骤6-9）
```

## 6. HeroBreakPageController 事件监听

| 事件 | 触发时机 | 处理方式 |
|------|----------|----------|
| `OnHeroLevelUpStart` | 服务端确认渡劫开始 | `RefreshView()` 全量刷新列表 |
| `OnForceHeroLevelUpStart` | 玩家点击可渡劫英雄 | 从 Ready 列表移除该英雄，刷新 |
| `OnHeroLevelUpDone` | 服务端推送渡劫完成 | `RefreshView()` 全量刷新列表 |
| `OnBuildingLevelUpSuccess` | 渡劫台建筑升级 | `RefreshView()` 全量刷新（可能解锁新上限） |

## 7. 关键机制

### 7.1 渡劫互斥

同一时间只有一个英雄能渡劫。`RequestingLevelUpHeroCount` 防止并发请求，`LevelUpHeroUUid` 记录当前渡劫中的英雄。

### 7.2 排队机制

当已有英雄在渡劫中时，新到达渡劫台的英雄会被设置为 `HeroActionHeroLevelUpQueue`，等前一个完成后自动由英雄行为树触发。

### 7.3 建筑等级上限

渡劫台等级决定英雄可渡劫的最高等级：
```
limit = P1306 (基础值) + AttrPercent(建筑属性) / 100 * 建筑属性值
```
建筑升级时触发 `OnBuildingLevelUpSuccess`，渡劫列表刷新后之前 `Unlocked` 状态的英雄可能变为 `Ready`。

### 7.4 强制前往

`HeroController.ForceHeroLevelUp()` 会先调用 `CancelAllForceAction()` 取消其他强制行为（装备交易、强制贸易等），然后设置 `isForceHeroLevelUp = true`。行为树中的 `IsForceHeroLevelUp` 节点检测到此标志后驱动英雄前往渡劫台。

### 7.5 读条计时

渡劫进行中（`HeroLevelUpType.Ing`），每帧通过 `Update()` 计算：
- `leftTime = EndTime - GetServerMilliSecond()`
- 进度 = `leftTime / totalTime`
- 显示格式：`mm:ss`

## 8. 渡劫结果弹窗（`HeroLevelUpResultDialogController`）

渡劫完成后弹出，展示：
- 英雄头像前后对比（战力变化）
- 境界名称前后对比（从 `hero_levelup` 配置读取 `Name + Name2`）
- 属性对比：血量(HP)、攻击(ATK)、防御(DEF)

## 9. 配置依赖

| 配置 | 说明 |
|------|------|
| `hero_levelup` 表 | 每个等级的经验门槛（按稀有度区分）、境界名称 |
| `hero` 表 | 英雄稀有度（Rarity） |
| `P1306` | 渡劫台基础等级上限 |
| `BuildingType.E_116` | 渡劫台建筑类型枚举 |
| `text_info` 900009/900010 | Toast 提示文本 |
