# 建筑系统 — 建筑信息获取完整流程

## 概述

建筑系统的数据分为两条线：
- **配置数据**：Excel 表导出 → `Cfg.Ins.building`，纯静态配置
- **运行时数据**：服务器协议下发 → `ClientBuildingContainer.BuildingDataDict`，反映玩家建筑的实际状态

两者通过 `ClientBuildingData.GetCfg()` 桥接：运行数据中的 `BuildingId` → 查到对应的配置行。

---

## 第一层：配置表 (`CfgGen.building`)

### 表来源

Excel `building` 表 → 自动生成 `CfgGen.building` 类 → 运行时通过 `Cfg.Ins.building` 单例访问。

### 配置表提供的查询 API

```csharp
// BuildingCollection (CfgExt_BuildingCollection.cs)
Cfg.Ins.building.get(buildingId)              // 按 ID 获取单条配置
Cfg.Ins.building.GetCfgListByType(type)       // 按 BuildingType 获取该建筑所有等级配置（已按 Level 升序排列）
Cfg.Ins.building.GetCfgByType(type)           // 按 BuildingType 获取 Level 最低的那条
Cfg.Ins.building.for_each((id, cfg) => ...)   // 遍历全部配置
```

### 扩展字段 (`CfgExt_Building.cs`)

配置表初始化时（`InitStep2`），遍历 `map_entity` 表，找到 `Type == 1`（建筑类实体）的条目，将其 `MapEntityId` 回填到对应 `building` 配置的 `MapEntityId` 字段。

```csharp
// CfgExt_Building.cs
public sealed partial class building
{
    public uint MapEntityId; // 补充 map_entity 表里的关联 id
}
```

### 关键配置字段

| 字段 | 说明 |
|------|------|
| `ID` / `Level` / `Type` | 建筑ID、等级、建筑类型枚举 |
| `Name` / `SmallPic` | 名称、小图标 |
| `NextLevel` | 下一级建筑ID（0 = 满级） |
| `CostIdList` / `CostCountList` | 升级消耗（物品ID列表 + 数量列表） |
| `AttributeList` / `AttributeValueList` | 属性ID列表 + 属性值列表 |
| `StageAttributeId` | 阶段属性ID |
| `RequestTypeList` / `PreRequestParamList` | 解锁前置条件（type=1前置建筑, type=2推图进度） |
| `MapEntityId` | 关联的地图实体ID |

## 第二层：运行时数据存储

### 数据结构

```csharp
// ClientBuildingData.cs
G.userData.buildingContainer                     // ClientBuildingContainer
    .BuildingDataDict                             // Dictionary<BuildingType, ClientBuildingData>
        [BuildingType] → ClientBuildingData
            .ProtoData                            // NetProto.BuildingData (服务器下发)
                .BuildingId                       // 当前建筑ID
                .BuildingType                     // 建筑类型
            .GetCfg()                             // → CfgGen.building (懒加载 + 缓存)
```

### `ClientBuildingData` 完整代码

```csharp
public class ClientBuildingData
{
    public NetProto.BuildingData ProtoData;

    CfgGen.building _cfg;
    public CfgGen.building GetCfg()
    {
        if (_cfg == null || _cfg.ID != ProtoData.BuildingId)
        {
            _cfg = Cfg.Ins.building.get(ProtoData.BuildingId);
        }
        return _cfg;
    }

    public void UpdateByProto(NetProto.BuildingData protoData)
    {
        this.ProtoData = protoData;
    }
}
```

**注意**：`GetCfg()` 有缓存失效逻辑 —— 当 `ProtoData.BuildingId` 变化（建筑升级后），自动重新查配置表。

### 数据来源：两个协议

| 协议 | 触发时机 | 行为 |
|------|----------|------|
| `MscInitBuildingInfo` | 进游戏初始化 / 断线重连 | **清空旧数据**，全量重建整个 `BuildingDataDict` |
| `MscSyncBuildingData` | 建筑升级等变更 | 增量更新指定 type 的 `ClientBuildingData`（调用 `UpdateByProto`） |

```csharp
// BuildingSystem_Net_Base.cs
// 初始化
private void OnInitBuildingInfo(CSMsg msg)
{
    G.userData.buildingContainer = new ClientBuildingContainer();
    var buildingInfo = msg.Body.MscInitBuildingInfo.UserBuildingData;
    for (int i = 0; i < buildingInfo.BuildingList.Count; i++)
    {
        AddBuildingDataByProto(buildingInfo.BuildingList[i]);
    }
}

// 增量同步
private void OnUpdateBuildingInfo(CSMsg msg)
{
    var building_list = msg.Body.MscSyncBuildingData.BuildingList;
    for (int i = 0; i < building_list.Count; i++)
    {
        var protoData = building_list[i];
        CfgEnum.BuildingType buildingType = (CfgEnum.BuildingType)protoData.BuildingType;
        G.userData.buildingContainer.BuildingDataDict[buildingType].UpdateByProto(protoData);
    }
}

// 写入字典
public void AddBuildingDataByProto(NetProto.BuildingData protoData)
{
    ClientBuildingData clientBuildingData = new() { ProtoData = protoData };
    G.userData.buildingContainer.BuildingDataDict[
        (CfgEnum.BuildingType)protoData.BuildingType] = clientBuildingData;
}
```

### 3.1 按类型查当前配置/数据（最常用）

```csharp
// BuildingSystem_Util.cs
public ClientBuildingData GetBuildingDataByType(CfgEnum.BuildingType buildingType)
{
    if (G.userData.buildingContainer.BuildingDataDict.TryGetValue(buildingType, out var data))
        return data;
    return null;
}

public CfgGen.building GetBuildingConfigByType(CfgEnum.BuildingType buildingType)
{
    return GetBuildingDataByType(buildingType).GetCfg();
}

public int GetBuildingLevel(CfgEnum.BuildingType buildingType)
{
    return this.GetBuildingConfigByType(buildingType).Level;
}
```

**这条路径是最核心的查询链路**：`BuildingType → BuildingDataDict → ProtoData.BuildingId → Cfg.Ins.building`

### 3.2 按 ID 查配置（纯配置表查询）

```csharp
public CfgGen.building GetBuildingConfigById(uint buildingId)
{
    return Cfg.Ins.building.get(buildingId);
}

public CfgGen.building GetNextBuildingConfigByType(CfgEnum.BuildingType buildingType)
{
    var cfg = GetBuildingConfigByType(buildingType);
    if (cfg.NextLevel == 0) return null;
    return Cfg.Ins.building.get((uint)cfg.NextLevel);
}

public CfgGen.building GetBuildingConfigByTypeAndLevel(CfgEnum.BuildingType buildingType, int level)
{
    return Cfg.Ins.building.GetCfgListByType((int)buildingType)
        .Find(cfg => cfg.Level == level);
}
```

### 3.3 状态判断

```csharp
public bool IsBuildingUnlocked(uint buildingId)
{
    // 当前建筑等级 >= 目标配置等级
    var targetCfg = Cfg.Ins.building.get(buildingId);
    var currentCfg = GetBuildingDataByType((CfgEnum.BuildingType)targetCfg.Type)?.GetCfg();
    return currentCfg.Level >= targetCfg.Level;
}

public bool IsMaxLevel(CfgEnum.BuildingType buildingType)
{
    return GetNextBuildingConfigByType(buildingType) == null;
}

public bool IsBuildingConstructedOnMainMap(CfgEnum.BuildingType buildingType)
{
    // Level > 0 视为已在主城生成实体
    var data = GetBuildingDataByType(buildingType);
    var cfg = data?.GetCfg();
    return cfg != null && cfg.Level > 0;
}
```

### 3.4 解锁/条件校验

```csharp
// 校验 request_type_list / pre_request_param_list 全部条件是否满足
// type=1: 前置建筑等级; type=2: 推图进度
// 返回 -1 = 全部满足，否则返回首个不满足条件的下标
public int IsBuildingCfgRequirementsMet(CfgGen.building cfg, int mapProgress, uint curBuildingId = 0)

// 链式解锁：从 Level=1 开始逐级校验，遇到断档或条件不满足即停止
// 返回能解锁到的最大等级
public int GetUnlockedMaxLevelByChain(int buildingType, int mapProgress)

// 可进入地图摆放流程的建筑 ID 列表（1级、条件满足、尚未生成实体）
public List<uint> GetConstructableBuildingIdsForMap()
```

## 第四层：UI 层

### 4.1 入口：从哪里打开建筑面板

两条路径：

#### A. 场景点击

`MainSceneController.OnClickBuilding(type, pos)` — 大地图上点击建筑模型时触发。

```csharp
// MainSceneController.cs
public void OnClickBuilding(CfgEnum.BuildingType buildingType, Vector3 pos)
{
    switch (buildingType)
    {
        case CfgEnum.BuildingType.E_101:
            BuildingBasePanelController.Create(buildingType, pos, 0).Forget();
            break;
        case CfgEnum.BuildingType.E_102: // ...
        // ... 其他建筑类型
    }
}
```

#### B. 代码跳转

`GotoView.GotoBuilding(type, pageIdx)` — 从其他 UI 跳转（如建筑升级前置条件快捷跳转）。

```csharp
// GotoView_Building.cs
public static async UniTask GotoBuilding(CfgEnum.BuildingType buildingType, int pageIndex0n)
{
    if (BuildingSystem.Ins.GetBuildingLevel(buildingType) <= 0)
    {
        // 未建造 → 打开建造界面
        await BuildUI.CreateWithTargetType(...);
        return;
    }
    // 已建造 → 获取坐标 → 移摄像机 → 打开建筑面板
    var logicPos = BuildingSystem.Ins.GetBuildingLogicPos(buildingType);
    var worldPos = MapSystem.LogicToWorldPos(...);
    // 摄像机移动到目标位置
    await BuildingBasePanelController.Create(buildingType, worldPos, pageIndex0n);
}
```

### 4.2 BuildingBasePanelController（建筑面板基类）

**生命周期**：

```csharp
public static async UniTask<BuildingBasePanelController> Create(
    CfgEnum.BuildingType buildingType, Vector3 worldPosition, int pageIdx0n)
{
    // 1. 移动摄像机到建筑位置
    // 2. 实例化 UI prefab
    // 3. InitBuilding(buildingType) → 存下 _buildingType
    // 4. CreateFuncPages() → 根据 buildingType 加载子页面
    // 5. ShowInitPage(pageIdx0n) → 显示默认页签
    // 6. RefreshView() → 刷新建筑名称等
}
```

**子页面配置**（`BuildingBasePanelController_Util.cs`）：

每种建筑类型注册对应的功能页签，例如：

```csharp
case CfgEnum.BuildingType.E_103:  // 铁匠铺
```

### 4.3 子页面控制器模式

所有子页面遵循统一的工厂模式：

```
PageController 持有 baseBuildingPanel 引用
    ↓
通过 Create(context, baseBuildingPanel) 注入
    ↓
通过 baseBuildingPanel.GetBuildingConfig() 获取配置
```

```csharp
// 以 WhiteJadeBedPageController 为例：
public partial class WhiteJadeBedPageController : IBaseController
{
    public BuildingBasePanelController baseBuildingPanel;

    public static async UniTask<Transform> Create(
        AsyncContext context, BuildingBasePanelController baseBuildingPanel)
    {
        var subPanel = await LogicLoader.AutoCreateInstanceAsync_ByPath(
            context, assetPath, baseBuildingPanel.GetPageParent());
        var controller = subPanel.GetComponent<WhiteJadeBedPageController>();
        controller.baseBuildingPanel = baseBuildingPanel;  // ← 注入引用
        return subPanel;
    }

    // 业务方法中获取配置：
    private long GetLowTime()
    {
        var buildingCfg = baseBuildingPanel.GetBuildingConfig();
        var baseVal = Cfg.Ins.property.P1085;
        var increaseVal = AttrSystem.Ins.GetAttributeUseValue(
            (uint)buildingCfg.AttributeList[0], buildingCfg.AttributeValueList[0]);
        return baseVal + increaseVal;
    }
}
```

### 4.4 两种获取配置的方式对比

| 方式 | 使用场景 | 示例 |
|------|----------|------|
| `baseBuildingPanel.GetBuildingConfig()` | 子页面与父面板绑定，类型由面板上下文决定 | `WhiteJadeBedPageController`、`HeroBreakPageController` 等 |
| `BuildingSystem.Ins.GetBuildingConfigByType(BuildingType.E_XXX)` | 子页面类型与建筑类型严格绑定，无需上下文 | `DispatchPageUIController`（硬编码 E_106）、`OrderPageUIController`（硬编码 E_104） |
