# 建筑银元操作文档

## 概述

银元是游戏中的基础货币，代码中定义为 `E_ItemID.Gold = 300500`（itemId: 300500），UI 显示为"银元"。

银元在建筑系统中的操作分为三类：
- **消耗**：建筑升级、装备锻造、订单交易
- **产出**：生产建筑制造商品并售卖给英雄
- **定价计算**：恢复商品的售价计算逻辑

## 二、银元消耗

### 2.1 建筑升级消耗

**涉及建筑**：全部建筑类型（102/103/107/108/109/110/111）

#### 配置表字段

配置表：`building`（Protobuf 生成类 `CfgGen.building`）

| 字段 | 类型 | 说明 |
|------|------|------|
| `CostIdList` | `RepeatedField<int>` | 升级消耗物品ID列表 |
| `CostCountList` | `RepeatedField<int>` | 升级消耗物品数量列表（与 ID 一一对应） |

> 当 `CostIdList` 中包含 `300500`（银元 itemId）时，升级需要消耗银元。

#### 调用链路

```

  → JianzhuUILevelupController.OnClickLevelUp()
    [Hot\Logic\UI\Building\LevelUp\JianzhuUILevelupController.cs:214]

  1. 获取下一级建筑配置
     → BuildingSystem.Ins.GetNextBuildingConfigByType(buildingType)
       [Hot\Logic\System\BuildingSystem\BuildingSystem_Util.cs:60]

  2. 检查材料是否充足
     → ItemSystem.Ins.IsItemEnough(idList[i], countList[i])
       (遍历 CostIdList / CostCountList，逐一检查)
       [JianzhuUILevelupController.cs:232-242]

  3. 材料不足 → 弹出 ResLackController 提示

  4. 材料充足 → 发送升级请求
     → BuildingSystem.Ins.BuildingUpgrade(currentBuildingCfg.ID, callback)
       [JianzhuUILevelupController.cs:252]

  5. 底层发送网络请求
```

## 三、银元产出

### 3.1 生产建筑制造商品并售卖给英雄

**涉及建筑**：102（参王汤）、109（孟茶摊）、110（纸扎庐）、111（鬼火锅）

#### 整体流程

```
制造流程（客户端模拟 + 服务器确认）:
  HuifuShengchanController.OnBtnMake()
    [Hot\Logic\UI\Building\Recover\HuifuShengchanController.cs:294]

  1. 获取原材料ID列表
     → RecoverSystem.Ins.GetOriginItemIdList(buildingType)
  2. 遍历背包中的原材料，生成制造任务
  3. 发送制造请求 CSMakeItemReq
  4. 等待制造完成（逐项延迟 Cfg.Ins.property.P1082 毫秒）
  5. 商品产出 → 英雄使用恢复建筑 → 服务器结算银元
```

#### 商品售价计算

```csharp
// RecoverSystem_Util.cs:178-183
public long GetProductPriceFinal(CfgEnum.BuildingType buildingType, CfgGen.item item)
{
    long originPrice = item.Param1List[0];                 // 基础价格（来自 item 配置表）
    long v3 = 10000 + GetPriceAddTenMyriad(buildingType);  // 10000 + 售价加成(万分比)
    return originPrice * v3 / 10000;                        // 最终售价
}
```

**售价公式**：`最终售价 = 基础价格 × (10000 + 售价加成万分比) / 10000`

#### 售价加成属性

各建筑的售价加成万分比来自属性系统：

| 建筑 | 属性ID | 说明 |
|------|--------|------|
| 102 参王汤 | 230001 | 血量恢复商品售价加成 |
| 109 孟茶摊 | 230011 | 心情恢复商品售价加成 |
| 110 纸扎庐 | 230015 | 精力恢复商品售价加成 |
| 111 鬼火锅 | 230019 | 体力恢复商品售价加成 |

#### 商品基础价格来源

`item.Param1List[0]` —— item 配置表的 `Param1List` 第一项，配置表中每个恢复商品的基础售价。

#### 商品ID配置

各建筑的原材料和商品 ID 清单来自 property 配置表：

| 建筑 | 商品列表 | 原材料列表 | 效果衰减 | 售价加成 |
|------|----------|------------|----------|----------|
| 102 | P1051 | P1052 | P1053 | 230001 |
| 109 | P1066 | P1067 | P1068 | 230011 |
| 110 | P1072 | P1073 | P1074 | 230015 |
| 111 | P1077 | P1078 | P1079 | 230019 |

> 详见 `RecoverSystem_Util.cs` 中各 `GetXxx` 方法的 switch 映射。

#### 银元产出链路汇总

```
生产建筑制造商品
  → 商品售价 = item.Param1List[0] × (10000 + Attr加成) / 10000
  → 英雄使用恢复建筑消耗商品
  → 服务器结算银元给玩家（服务器逻辑，客户端无法直接看到扣款代码）
```
