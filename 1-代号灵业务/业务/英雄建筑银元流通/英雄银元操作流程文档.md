# 英雄银元操作流程文档

> 银元 itemId: `300500`（`E_ItemID.Gold`），本文档梳理英雄使用恢复建筑和英雄购买装备两个完整流程，含客户端调用链路、协议交互及数据流。

---

## 一、英雄使用恢复建筑流程

### 1.1 流程总览

```
                   制造阶段                    恢复阶段                    结算阶段
              ╭──────↓──────╮          ╭───────↓───────╮          ╭───────↓───────╮
  玩家操作:   点击"制造"按钮  →   [获得商品]  → 英雄AI自动走到建筑  →  [服务端结算]
              │              │          │               │          │               │
  协议交互:   MakeItemReq    →         │  RecoverHeroXxxReq    →  SyncRecoverDone
              │              │          │               │          │    +
  数据变化:   原材料→商品      →         │  hero银元→玩家银元       │  SyncItemData
              ╰──────────────╯          ╰───────────────╯          ╰───────────────╯
```

### 1.2 阶段一：制造商品

玩家在生产建筑（102参王汤 / 109孟茶摊 / 110纸扎庐 / 111鬼火锅）点击"制造"按钮。

**调用链路**：

```
HuifuShengchanController.OnBtnMake()
  [HuifuShengchanController.cs:294]

1. 获取原材料ID列表
   → GetOriginItemIdList(buildingType)
   各建筑原材料列表来自 property 表:
   - 102 参王汤: P1052
   - 109 孟茶摊: P1066(=原材料P1067...实际是 P1067)
   - 110 纸扎庐: P1073
   - 111 鬼火锅: P1078

2. 遍历背包中原材料，生成制造任务 (MakingProductItem)
   for each unlockedOriginItemId:
       count = ItemSystem.GetItemCount(id)
       productItemId = item.Param1List[1]  // 原材料配置的 Param1List[1] = 产物ID
       makingProductItemInfoList.Add(originItemId, productItemId, count)

3. 原材料不足 → 弹出 ResLackController 引导获取材料

4. 发送制造请求 → 服务端扣除原材料 + 产出商品
   Network.Ins.helper.MakeItemReqAsync(CSMakeItemReq {
       ItemIdList = [productItemId1, productItemId2, ...]
   })
   协议: McsMakeItemReq / MscMakeItemRsp

5. 成功后客户端逐项模拟制造动画
   每项延迟 Cfg.Ins.property.P1082 毫秒
   makingProductItemInfoList[i].isFinished = true
```

**涉及配置表**：

| 配置表 | 字段 | 说明 |
|--------|------|------|
| `property` | P1052/P1067/P1073/P1078 | 各建筑的原材料ID列表 |
| `property` | P1082 | 每项制造耗时(ms) |
| `item` | `Param1List[1]` | 原材料对应产物ID |
| `item` | `Param1List[0]` | 产物基础售价 |

### 1.3 阶段二：英雄发起恢复

英雄 AI 通过行为树走到恢复建筑，触发 `IsRecovering()` 状态。

**调用链路**：

```
HeroController.IsRecovering()
  [HeroController.cs:873]

1. 获取最优可用商品
   itemId = RecoverSystem.GetRecoveryItemID(buildingType)
     [RecoverSystem_API.cs:186]
   → 获取商品列表 GetProductItemIdList(buildingType)
   → 从高到低遍历，返回第一个玩家库存 ≥1 的商品ID
   → 全部不足 → 返回0（英雄不会发起恢复）

2. 获取商品售价
   price = RecoverSystem.GetProductPrice(buildingType, itemId)
     [RecoverSystem_Util.cs:72]
   公式: item.Param1List[0] × (10000 + Attr加成万分比) / 10000

3. 检查英雄银元是否足够
   if heroData.BagList.Any(item => item.ItemId == 300500 && item.Count >= price)
   不足 → isBankrupt = true，跳过本次恢复

4. 发起恢复请求
   result = RecoverSystem.StartRecover(buildingType, heroUuid, callback)
     [RecoverSystem_API.cs:44]

     ├─ 队列检查: 当前恢复人数 + 正在请求人数 ≥ 建筑容量上限
     │    达上限 → 设置 HeroActionRecoveryXxxQueue（排队中）
     │    → return 1
     │
     ├─ 发送网络请求:
     │    RecoverHeroHpReqAsync(CSRecoverHeroHpReq { HeroUuid })
     │    (或 Mood/Energy/Strength 对应协议)
     │
     └─ 成功回调:
          heroData.ActionStartTime = rsp.StartTime
          heroData.ActionEndTime = rsp.EndTime
          heroData.Action = HeroActionRecoveryXxxIng (恢复中)
          加入恢复队列: recoverContainer.AddHeroInRecover(buildingType, heroUuid)
          发送事件: OnHeroRecoveryXxxStart
```

**四个恢复建筑对应的协议**：

| 建筑 | Req/Rsp 协议 | 调用方法 | 属性变化 |
|------|-------------|----------|----------|
| 102 参王汤 | McsRecoverHeroHpReq/Rsp | `RecoverHeroHpReqAsync` | HP↑ |
| 109 孟茶摊 | McsRecoverHeroMoodReq/Rsp | `RecoverHeroMoodReqAsync` | Mood↑ |
| 110 纸扎庐 | McsRecoverHeroEnergyReq/Rsp | `RecoverHeroEnergyReqAsync` | Energy↑ |
| 111 鬼火锅 | McsRecoverHeroStrengthReq/Rsp | `RecoverHeroStrengthReqAsync` | Strength↑ |

**请求体**（以HP为例）：

| 字段 | 类型 | 说明 |
|------|------|------|
| `HeroUuid` | ulong | 要恢复的英雄UUID |

**响应体**：

| 字段 | 类型 | 说明 |
|------|------|------|
| `StartTime` | long | 恢复开始时间戳 |
| `EndTime` | long | 恢复结束时间戳 |

### 1.4 阶段三：服务端结算

服务端在收到 `CSRecoverHeroHpReq` 后执行：

1. 扣除玩家背包中对应的恢复商品（1个）
2. 扣除英雄背包中 `price` 数量的银元
3. 将 `price` 银元增加给玩家
4. 推送 `MscSyncItemData`：更新玩家银元数量、物品变化
5. 推送 `MscSyncHeroRecoverHpDone`：恢复完成，HP已更新

**客户端接收银元更新**：

```
ItemSystem.OnSyncItemData(CSMsg)
  [ItemSystem_Net.cs:48]

遍历 msg.Body.MscSyncItemData.ItemList:
  - 找到 itemId=300500 的条目
  - 更新 itemMap 中的 count
  - 触发 OnItemCountChanged 事件 → UI 刷新银元显示
```

**客户端接收恢复完成**：

```
RecoverSystem.OnSyncRecoverHeroHpDone(CSMsg)
  [RecoverSystem_Net.cs:41]

1. 从恢复队列移除: recoverContainer.recoverHpHeroUuidList.Remove(heroUuid)
2. 更新英雄HP: hero.protoData.Attr.BaseAttr.CrtHp = msgBody.Hp
3. 清除动作状态: Action = HeroActionRecoveryHpDone
4. 播放飘字特效: FloatingRecoverInfo (显示HP变化量)
5. 发送事件: OnHeroRecoveryHpDone(heroUuid)
```

### 1.6 售价加成属性

各建筑售价加成来自属性系统，受建筑等级和养成影响：

| 建筑 | 加成属性ID | 说明 |
|------|-----------|------|
| 102 参王汤 | 230001 | 血量恢复商品售价加成万分比 |
| 109 孟茶摊 | 230011 | 心情恢复商品售价加成万分比 |
| 110 纸扎庐 | 230015 | 精力恢复商品售价加成万分比 |
| 111 鬼火锅 | 230019 | 体力恢复商品售价加成万分比 |

**售价公式**：
```
售价 = item.Param1List[0] × (10000 + AttrSystem.GetAttributeMyriad(attrId)) / 10000
```
- `item.Param1List[0]`：商品基础价格（来自 item 配置表）
- `AttrSystem.GetAttributeMyriad(attrId)`：售价加成万分比（来自属性系统）

### 1.7 队列与容量机制

```
StartRecover() 前置检查:

  当前恢复中人数 = recoverContainer.count + reqRecoverCount (正在请求中)
  if 当前人数 >= 建筑容量上限:
      → 设置 HeroActionRecoveryXxxQueue（排队，不发送请求）
      → 英雄会等待后重试
  else:
      → reqRecoverCount++
      → 发送网络请求
      → 回调时 reqRecoverCount--
```

各建筑容量上限来自 property 配置表：
| 建筑 | 容量属性 |
|------|---------|
| 102 | P1057 |
| 109 | P1071 |
| 110 | P1076 |
| 111 | P1081 |

## 二、英雄购买装备流程

### 2.1 流程总览

```
              ╭── 浏览装备 ──╮     ╭── 购买确认 ──╮     ╭── 结算 ──╮
  玩家操作:   打开交易界面    →     点击装备→交易     →     [完成]
              │              │     │              │     │         │
  数据检查:   遍历装备列表    →     hero银元 ≥ 价格?  →     
              │              │     │              │     │         │
  协议交互:   (本地数据)      →     HeroBuyEquipmentReq  →  SyncItemData
              │              │     │              │     │         │
  银元变化:   -              →     扣hero银元        →    装备→hero
              ╰──────────────╯     ╰──────────────╯     ╰─────────╯
```

### 2.2 阶段一：打开交易界面

玩家在装备锻造建筑（103/107/108）切换到交易Tab，打开英雄装备交易界面。

**涉及文件**：
`HeroEquipmentTradeDialogController.cs` — 交易主界面
`HeroEquipmentTradeDialogItemController.cs` — 每个装备的交易项

```
HeroEquipmentTradeDialogController.Init()
  [HeroEquipmentTradeDialogController.cs:40]

1. 刷新装备列表 RefreshEquipmentList()
   → 从 G.userData.equipmentContainer 读取玩家背包装备
   → 筛选可交易的装备

2. 初始化 ScrollView
   scrollEquipment.InitListView(equipmentList.Count, callback)

3. 显示英雄银元余额
   heroGold = heroData.BagList.Where(x => x.ItemId == 300500).FirstOrDefault()?.Count
```

### 2.3 阶段二：检查银元并购买

玩家点击某个装备 → 点击"交易"按钮。

**调用链路**：

```
HeroEquipmentTradeDialogItemController.OnBtnTradeClicked()
  [HeroEquipmentTradeDialogItemController.cs:145]

1. 获取装备配置和售价
   equipmentCfg = data.GetCfg()
   price = equipmentCfg.EquipmentPriceList[data.protoData.Quality]
   // 注意: 售价按品质(0~5)索引，不同品质价格不同

2. 获取英雄银元数据
   heroGoldData = heroData.BagList
       .Where(x => x.ItemId == (uint)E_ItemID.Gold)
       .FirstOrDefault()

3. 银元不足检查
   if heroGoldData == null || heroGoldData.Count < price:
       ToastMgr.ShowToast("银元不足")
       return  // ← 到此为止，不发请求

4. 发送购买请求
   EquipmentSystem.RequestHeroBuyEquipment(heroUuid, equipmentUuid, callback)
     [EquipmentSystem_Net.cs:138]
   → Network.Ins.helper.HeroBuyEquipmentReqAsync(
       CSHeroBuyEquipmentReq {
           HeroUuid = heroUuid,
           EquipmentUuid = equipmentUuid
       })

   协议: McsHeroBuyEquipmentReq / MscHeroBuyEquipmentRsp

5. 成功回调
   rsp != null → parentController.isTrade = true → UI刷新
```

**请求体** `CSHeroBuyEquipmentReq`：

| 字段 | 类型 | 说明 |
|------|------|------|
| `HeroUuid` | ulong | 购买装备的英雄UUID |
| `EquipmentUuid` | ulong | 要购买的装备UUID |

**价格来源**：`hero_equipment` 配置表的 `EquipmentPriceList`，按装备品质（0~5）索引：

```
EquipmentPriceList[0] → 品质0价格
EquipmentPriceList[1] → 品质1价格
...
EquipmentPriceList[5] → 品质5价格
```

### 2.4 阶段三：服务端处理

服务端收到 `CSHeroBuyEquipmentReq` 后：

1. 验证英雄银元余额 ≥ 装备售价
2. 扣除英雄背包中的银元
3. 装备所有权从玩家转移至英雄
4. 返回 `SCHeroBuyEquipmentRsp`
5. 推送 `MscSyncEquipmentData(UpdateTypeUpdate/Delete)` + `MscSyncItemData`

### 2.5 银元不足时的补救：玩家捐赠

当英雄银元不足时，玩家可以在交易界面点击"加金币"按钮，捐出自己的银元给英雄。

**调用链路**：


## 三、两个流程对比

| 维度 | 英雄使用恢复建筑 | 英雄购买装备 |
|------|-----------------|-------------|
| **触发方式** | 英雄AI自动（行为树） | 玩家手动操作 |
| **银元方向** | hero银元 → **玩家银元**（产出） | **hero银元** → 系统（消耗） |
| **银元主体** | hero付款，玩家收款 | hero付款 |
| **消耗物品** | 恢复商品（玩家背包） | 装备（玩家背包） |
| **核心协议** | `RecoverHeroXxxReq` + `MakeItemReq` | `HeroBuyEquipmentReq` |
| **银元同步** | `MscSyncItemData` | `MscSyncItemData` |
| **附属操作** | 制造商品（MakeItemReq） | 捐赠银元（DonateItemToHeroReq） |
| **涉及建筑** | 102/109/110/111 | 103/107/108 |
| **银元检查位置** | HeroController.IsRecovering() | HeroEquipmentTradeDialogItemController |
| **队列机制** | 有容量上限+排队 | 无（单人单次） |
