# 项目基础结构

## 项目概览

**zb-client** 是一个 Unity 手机游戏项目，使用 **HybridCLR** 热更新框架，代码按编译方式分为两大层：**AOT（提前编译）** 和 **Hot（热更新）**。

---

## 顶层目录结构

```
Assets/Scripts/
├── AOT/              # 提前编译代码（不能热更）
├── Editor/           # Unity 编辑器工具代码
├── EditorAssembly/   # 编辑器程序集
├── Hot/              # 热更新代码（核心业务逻辑）
├── Other/            # HybridCLR 限制：需要引用 Hot 代码但用不到编辑器
├── Shader/           # Shader 相关
└── UnityTest/        # 单元测试
```

---

## 一层：AOT — 底层框架（不可热更）

| 模块 | 职责 |
|------|------|
| `AOT/Main/Init/` | 游戏入口 `GameInitObject`，初始化 HybridCLR，加载热更 DLL |
| `AOT/Main/XKit/` | 通用工具库：网络（TCP/WebSocket）、对象池、CSV 解析、I18N、相机操控、设备信息、特效、Tween |
| `AOT/Main/UI/` | AOT 层 UI 基础组件：自定义 Button/Drag/Text/Image/Toggle/Slider 等 |
| `AOT/Main/AppModule/` | 音频管理 (`AudioUtil`)、资源加载 (`AssetsModule`)、热更模块加载 (`HotModule`) |
| `AOT/Main/Channel/` | 渠道 SDK 封装（登录/支付等） |
| `AOT/Main/Update/` | 更新流程管理（`UpdateMgr`） |

---

## 二层：Hot — 热更新业务（核心代码）

### `Hot/Common/`
通用代码层，被 Hot/AutoGen 和 Hot/Logic 依赖。

### `Hot/AutoGen/`
自动生成代码，分为：
- **Gen/** — 工具自动生成的代码
- **Keep/** — 保留的手动补充代码

### `Hot/Logic/` — 核心游戏逻辑

#### 架构设计（`Architecture/`）

采用类似 **QFramework** 的架构模式，核心类是 `LogicApp`：

```
LogicApp (架构根)
  ├── System（系统层）：处理业务逻辑，由架构统一管理生命周期
  │     ├── OnEnterGameAsync()   — 进入游戏
  │     ├── OnRoleBeforeLogin()  — 登录前
  │     ├── OnRoleAfterLogin()   — 登录后（数据齐全）
  │     ├── OnUpdate()           — 每帧更新
  │     ├── OnRoleAfterLogout()  — 登出
  │     └── OnExitGame()         — 退出游戏
  │
  └── Mgr（管理器层）：单例模式，独立于架构生命周期
        ├── UIMgr       — UI 管理
        ├── ViewMgr     — 视图管理
        ├── DialogMgr   — 弹窗管理
        └── BackStackRecordMgr — 返回栈管理
```

**设计规范**：
- System **不应持有 UI 引用**，不要直接调用 UI 刷新方法
- 应该通过 **事件（Event）** 触发 UI 刷新
- 事件发送时机：服务器回包、玩家操作

#### UI 层（`UI/`）

与 System 一一对应，每个 UI 模块一个子目录：
`Activity`, `Battle`, `Building`, `Chat`, `Hero`, `Login`, `Mail`, `Main`, `Shop`, `Stage` 等，加上：
- `Common/` — 通用 UI 组件
- `Comp/` — 可复用 UI 组件
- `GM/` — GM 调试面板
- `Guide/` — 引导 UI

#### 数据层（`Model/`）

- `Cfg/` — 配置表数据
- `G/` — 全局数据
- `UserData/` — 玩家数据

## 关键技术栈

- **HybridCLR** — IL2CPP 热更新方案
- **UniTask** — 异步编程（替代 async/await）
- **DOTween** — UI 动画
- **Spine** — 2D 骨骼动画
- **自定义 UI 框架** — 基于 QFramework 架构思想
- **网络层** — TCP + WebSocket，Protobuf 协议

---

## 开发指引

这是一个典型的 **System-driven 架构** 的 Unity 手机游戏。要加新功能时：

1. 在 `Hot/Logic/System/` 下新建 System
2. 在 `Hot/Logic/UI/` 下加对应 UI
3. 通过 **Event** 解耦 System 和 UI 之间的通信
4. System 不要直接持有 UI 引用，不要直接调用 UI 刷新方法
