# 功能提示小助手 — 类型 6、类型 7 需求文档

## 需求背景

当前功能提示小助手支持类型 1（建造提示）和类型 2-5（恢复建筑缺货提示）。现需新增类型 6 和类型 7，用于锻造类建筑（103/107/108）的英雄交易提示。

---

## 锻造类建筑定义

| 建筑 ID | 枚举值 | 名称 | 装备类型 |
|---------|--------|------|----------|
| 103 | E_103 | 裁缝机 | 防具（Type 1/2/3：帽子/衣服/裤子）|
| 107 | E_107 | 锻造台 | 武器（Type 0，按英雄职业匹配）|
| 108 | E_108 | 符儡仪 | 饰品（Type 4/5：内符/外箓）|

---

## 类型 6 — 英雄前往锻造类建筑交易提示

### 基本信息

| 属性 | 值 |
|------|-----|
| 类型编号 | 6 |
| 显示位置 | 顶部行（`btnAssistant2`），与类型 2-5、7 共用 |
| 优先级 | 高于类型 2-5，低于类型 7 |
| UI 样式 | 举牌小人 + 建筑小图标（美术资源未就绪，暂时用 log 替代） |

### 触发条件

英雄开始前往锻造类建筑（建筑 ID 103、107、108 之一）时触发。

**实现方式**：新增事件 `OnHeroGoToForgeBuilding(HeroUuid, BuildingType)`，在 `HeroController.MovingToTargetBuildingPos()` 被调用且 `targetBuildingType` 为 103/107/108 时发送。

### 文本

- **文本来源**: `assistant` 配置表 Type=6，回退 `text_info` ID 900525
- **参数**: `{0}` 英雄名称，`{1}` 建筑名称
- **占位符**: 暂时使用 `Debug.Log` 打印实际文本内容，由策划/产品自行确认

### UI 表现

- **举牌小人 + 建筑图标**: 美术资源未就绪，暂时沿用现有的图片加载逻辑框架，但用 `Debug.Log` 替代实际加载
- **位置**: 与类型 2-5 共用同一位置（`btnAssistant2` 区域）

### 点击跳转

**复用项目中已有的英雄点击行为** — 即 `HeroController.OnTouchBegin()` 的逻辑：

1. 通过 `GameObjectPool.UI3D.SpawnAsync` 生成 `hero_detail_click.prefab`
2. `HeroClickDetail.Setup()` 绑定英雄，相机 orthographicSize 设为 6，每帧 LateUpdate 跟踪英雄位置
3. 显示三个操作按钮：**详情**（打开 HeroDetailDialog）、**派遣**（移动到建筑 E_106）、**关闭**

**入口**: 通过 `MapObjectsMgr.GetHeroModel(heroUuid)` 获取 HeroController，然后调用其 `OnTouchBegin` 等效逻辑（或直接模拟 D2Button 点击）。

### 生命周期

| 阶段 | 行为 |
|------|------|
| **开始** | 英雄前往锻造类建筑时显示。新英雄触发时**替换**当前（最新出发优先）|
| **进行中** | 同时只允许一个英雄的提示 |
| **结束** | 英雄到达锻造类建筑时消失（行为树 `MovingToTargetBuildingPos` 返回 Success）|
| **中断** | 英雄前往行为中途终止（已有行为树机制处理，监听 `OnHeroExitForgeBuilding` 事件）|

## 类型 7 — 英雄可交易气泡提示

### 基本信息

| 属性 | 值 |
|------|-----|
| 类型编号 | 7 |
| 显示位置 | 顶部行（`btnAssistant2`），与类型 2-6 共用 |
| 优先级 | **最高**（7 > 6 > 5 > 4 > 3 > 2）|
| UI 样式 | 举牌小人 + 建筑小图标（美术资源未就绪，暂时用 log 替代） |

### 触发条件

英雄已到达锻造类建筑（103/107/108）的交互点，且 `WaitingForExchangeEquipment()` 处于运行状态，英雄头顶已出现交易气泡框（`ButtonBubble`）时触发。

**实现方式**：新增事件 `OnHeroReadyForEquipmentTrade(HeroUuid, BuildingType)`，在 `HeroController.WaitingForExchangeEquipment()` 进入 Running 状态时发送。

### 文本

- **文本来源**: `assistant` 配置表 Type=7，回退 `text_info` ID 900526
- **参数**: 同类型 6 — `{0}` 英雄名称，`{1}` 建筑名称
- **占位符**: 同类型 6，使用 `Debug.Log` 打印

### UI 表现

- **举牌小人 + 建筑图标**: 同类型 6，美术未就绪，用 log 替代
- **位置**: 与类型 2-6 共用同一位置

### 点击跳转

点击后执行两步操作：

**第 1 步 — 定位英雄**：复用类型 6 的点击逻辑（`HeroController.OnTouchBegin()`），镜头跟踪英雄并显示操作气泡。

**第 2 步 — 引导指引手指**：
- **实现方式**: 参考 `FloatingRecoverInfo` 的生成模式（`GameObjectPool.UI3D.SpawnAsync`），在英雄头顶世界坐标处生成引导手指 prefab
- **参考代码**: `RecoverSystem_Net.cs` 中恢复完成的浮字生成逻辑
  ```
  controller = MapObjectsMgr.GetHeroModel(hero.Uuid);
  GameObjectPool.UI3D.SpawnAsync(prefabPath, 0, item => {
      item.gameObject.transform.position = controller.transform.position;
      // ... setup finger animation
  });
  ```
- **手指行为**: 在英雄头顶交易气泡框上方常驻显示
- **消失条件**: 玩家点击屏幕任意位置后消失（需要监听全局点击事件来清除）

### 生命周期

| 阶段 | 行为 |
|------|------|
| **开始** | 英雄到达并处于可交易状态时常驻 |
| **进行中** | 新英雄触发可交易状态时顶替旧提示（同时只显示一个）|
| **结束** | 英雄可交易行为消失（离开交互点、完成交易等）时，提示自动消失 |

---

## 类型 6→7 过渡行为

与类型 2-5 的互斥替换行为类似：当同一英雄从"前往中"变为"可交易"时，类型 6 直接替换为类型 7（共用同一显示位置），无需过渡动画。

---

## 优先级完整排序

```
类型 7（可交易气泡） > 类型 6（前往锻造建筑） > 类型 5（体力）> 类型 4（精力）> 类型 3（心情）> 类型 2（血量）

即：7 > 6 > 5 > 4 > 3 > 2
类型 1（建造）独立显示在底部行，不参与优先级竞争
```

## 新增事件定义

在 `EventBuilding.cs` 中新增：

| 事件 | 携带数据 | 用途 |
|------|----------|------|
| `OnHeroGoToForgeBuilding` | `HeroUuid`, `BuildingType` | 触发类型 6 提示 |
| `OnHeroReadyForEquipmentTrade` | `HeroUuid`, `BuildingType` | 触发类型 7 提示 |
| `OnHeroExitForgeBuilding` | `HeroUuid` | 结束类型 6/7 提示 |

## 代码改动清单

### 1. EventBuilding.cs — 新增 3 个事件类

```
OnHeroGoToForgeBuilding { long HeroUuid; CfgEnum.BuildingType BuildingType; }
OnHeroReadyForEquipmentTrade { long HeroUuid; CfgEnum.BuildingType BuildingType; }
OnHeroExitForgeBuilding { long HeroUuid; }
```

### 2. HeroController.cs — 发送事件

- 在 `MovingToTargetBuildingPos()` 中：当目标建筑为 103/107/108 时，发送 `OnHeroGoToForgeBuilding`
- 在 `WaitingForExchangeEquipment()` 进入 Running 时：发送 `OnHeroReadyForEquipmentTrade`
- 在行为树切换到非锻造相关节点时：发送 `OnHeroExitForgeBuilding`

### 3. MainUIController_Assistant.cs — 核心改动

| 改动项 | 说明 |
|--------|------|
| 新增字段 | `currentHintType`（当前显示的类型 6/7）、`currentHeroUuid`、`currentBuildingType` |
| 事件注册 | 注册 `OnHeroGoToForgeBuilding`、`OnHeroReadyForEquipmentTrade`、`OnHeroExitForgeBuilding` |
| `RefreshAssistantHints()` | 新增类型 7/6 判断分支（优先级在 2-5 之前） |
| `GetAssistantHintText()` | 新增 type 6→900525、type 7→900526 的 text_info 回退 |
| 新方法 | 判断类型 6 或 7 是否有有效提示，返回提示类型和参数 |
| 点击处理 | 区分类型 6（选中英雄）和类型 7（选中英雄 + 引导手指）的点击逻辑 |
| 图片加载 | 沿用 `ImageUtil.GetSpriteByPath()` 模式，暂时用 `Debug.Log` 替代实际加载 |

### 4. MainUIController.cs — 引导手指管理

- 类型 7 点击后，在英雄头顶生成引导手指（参考 `FloatingRecoverInfo` 生成模式）
- 注册全局点击监听，点击屏幕任意位置后清除手指
