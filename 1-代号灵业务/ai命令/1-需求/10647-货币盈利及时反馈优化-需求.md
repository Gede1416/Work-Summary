# 10647-货币盈利及时反馈优化-需求

## 1. 需求概述

在英雄消费银元之后，在对应的建筑上弹出飘字提示，即时反馈玩家的银元变化。

- **恢复建筑**（102参王汤/109孟茶摊/110纸扎庐/111鬼火锅）：英雄支付银元给玩家 → 弹出 "+XXX"
- **装备购买建筑**（103/107/108）：英雄支付银元购买装备 → 弹出 "-XXX"

---

## 2. 架构设计

遵循项目 System-driven 架构规范：System 层通过 Event 通知 UI 层，System 不直接持有 UI 引用。

```
System 层                      Event                     UI 层
─────────                      ─────                     ──────
RecoverSystem ──缓存价格──→ OnHeroSilverFlow ──→ BuildingSilverFloatMgr
EquipmentSystem ──直接发送─→ {BuildingType,              │
                              SilverAmount,             累加限频
                              IsGain}                   生成飘字
```

---

## 3. Event 定义

### 新增事件：`OnHeroSilverFlow`

位置：`Hot/Logic/Event/`

| 字段 | 类型 | 说明 |
|------|------|------|
| `BuildingType` | `CfgEnum.BuildingType` | 触发银元变化的建筑类型 |
| `SilverAmount` | `long` | 银元变化量（正数） |
| `IsGain` | `bool` | true=玩家获得银元，false=英雄消耗银元 |

---

## 4. System 层改动

### 4.1 RecoverSystem

**RecoverSystem_API.cs — StartRecover 方法**：

- 新增参数 `long silverReward`，表示本次恢复英雄支付的银元（即玩家将获得的银元）
- 新增字段 `Dictionary<ulong, long> _heroSilverRewards`（heroUuid → 银元价格）
- 请求成功回调中：`_heroSilverRewards[heroUuid] = silverReward`

**RecoverSystem_Net.cs — 四个 Done 处理器**：

- 在 `OnSyncRecoverHeroHpDone` / `OnSyncRecoverHeroMoodDone` / `OnSyncRecoverHeroEnergyDone` / `OnSyncRecoverHeroStrengthDone` 中：
  - 从 `_heroSilverRewards` 取出缓存价格
  - 发送 `OnHeroSilverFlow` 事件（`IsGain = true`）
  - 清除缓存条目

**HeroController.cs — IsRecovering 方法**：

- 调用 `StartRecover` 时传入已计算的 `price`

### 4.2 EquipmentSystem

**EquipmentSystem_Net.cs — RequestHeroBuyEquipment**：

- 成功回调中发送 `OnHeroSilverFlow` 事件（`IsGain = false`，`SilverAmount = 装备售价`）

### 4.3 装备交易 UI（传递 BuildingType）

**HeroEquipmentTradeDialogController.cs**：

- 新增 `BuildingType` 字段，`Init` 时赋值

**HeroEquipmentTradeDialogItemController.cs**：

- 购买成功回调中发送 `OnHeroSilverFlow` 事件（取 `parentController.BuildingType`）

## 5. UI 层改动

### 5.1 BuildingSilverFloatManager（新建）

位置：`Hot/Logic/UI/Building/BuildingSilverFloatManager.cs`

**职责**：监听 `OnHeroSilverFlow` 事件，限频累加后生成飘字。

**限频累加机制**：

每个建筑独立维护累加器：

```csharp
class BuildingSilverAccumulator {
    long AccumulatedAmount;   // 累计银元
    bool IsGain;              // 获得/消耗
    float LastShowTime;       // 上次显示时间
    bool IsPending;           // 是否有待显示数据
}
```

处理流程：

```
OnHeroSilverFlow 到达
  │
  ├─ 该建筑无待显示数据 → 记录累加值，标记 IsPending，启动定时器
  │
  └─ 该建筑已有待显示数据 → 累加 SilverAmount，重置定时器

定时器到期（间隔 T 秒）
  │
  ├─ IsGain → 显示 "+XXX"（绿色/金色）
  ├─ !IsGain → 显示 "-XXX"（红色）
  │
  └─ 获取建筑世界坐标 → 转屏幕坐标 → 生成 CurrencyFloatText 飘字
     → 清除累加器
```

**可配置参数**：

| 参数 | 建议值 | 说明 |
|------|--------|------|
| 累加间隔 | 0.5s | 间隔内同一建筑的多次银元变化合并为一次飘字 |
| 飘字动画 | 复用 `CurrencyFloatText` | 缩放弹出(0.2s) + 上浮(0.8s) + 淡出(0.3s) |

**建筑坐标映射**：

```
BuildingSystem.Ins.GetBuildingLogicPos(buildingType)
  → MapSystem.LogicToWorldPos()
  → Camera.WorldToScreenPoint()
  → UI Canvas 定位
```

## 7. 边界情况

| 情况 | 处理方式 |
|------|----------|
| 同一建筑短时间内多个英雄恢复 | 累加显示总和，如 "+450" |
| 英雄恢复排队（容量满） | 排队不触发事件，实际恢复完成时才触发 |
| 英雄银元不足被跳过 | `StartRecover` 不会被调用，无事件产生 |
| 玩家捐赠银元给英雄 | 不触发建筑飘字（非建筑消费场景） |
| 飘字生成时建筑不在屏幕内 | 跳过本次显示或等待建筑可见 |
| 场景切换/登出 | 清除所有待显示累加器 |
